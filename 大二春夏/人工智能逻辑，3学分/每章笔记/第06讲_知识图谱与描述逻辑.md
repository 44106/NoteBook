# 第6讲 知识图谱与描述逻辑

---

## 概述

知识图谱是一种用图结构表示知识的方法，通常以三元组 `(主体, 谓词, 客体)` 的形式组织数据。例如，`(爱因斯坦, 出生地, 德国)` 表示"爱因斯坦的出生地是德国"。知识图谱能集成不同来源的数据，广泛应用于搜索、推荐和问答。

描述逻辑则是知识图谱的形式化基础，提供定义概念、属性及逻辑规则的框架。它赋予知识图谱语义推理能力，支持自动分类、矛盾检测等操作。通过知识图谱，可以实现将非结构化数据转化为机器可理解的知识。进一步地，通过描述逻辑规则推导隐含知识，可以为搜索引擎、智能问答、推荐引擎等实现语义理解与决策支持。

## 6.1 语义网络与知识图谱

在基于知识的系统中，知识可以表示为一组公式集合。不过，在许多情况下，知识具有其他类型的结构，如框架和网络等。在网络结构的知识体系中，知识在一个带标签的有向图中得到表达。其中，图的节点表示各种实体，而边则表示实体之间的关系。

### 6.1.1 三元组表示法

通常，可以把两个节点及它们之间的边组成一个三元组：`prop(Ind, Prop, Val)`。其中：

- `Ind` 称为**主语** (Subject)
- `Prop` 称为**谓词** (Predicate)
- `Val` 称为**宾语** (Object)

把一个三元组类比成有三个词的简单句子。这个三元组中的谓词是一个特性。

### 6.1.2 复杂关系的三元组处理

在建模的过程中，对于一些谓词，把它转化为三元组表示时，可能太简单或太复杂，需要进行一些处理。

**案例一：一元谓词的处理**

例如，对于 `belt_road_project(a)`，意指 a 是"一带一路"国际合作高峰论坛框架下的重点项目，看起来没有合适的特性和值。通常有如下两种方法来表示：

1. **作为类名称**：把 `belt_road_project` 当作一个类名称，用三元组 `a is-a belt_road_project` 来表示：`prop(a, is-a, belt_road_project)`
2. **作为特性名称**：把 `belt_road_project` 当作一个特性名称，用三元组 `a belt_road_project true` 来表示：`prop(a, belt_road_project, true)`

**案例二：多元关系的具体化**

对于关系 `scheduled(C, S, T, R)`，意指课程 C 的章节 S 被安排在时间 T 的教室 R。例如，课程《习近平新时代中国特色社会主义思想概论》(new-era2025) 的第2讲"以中国式现代化全面推进中华民族伟大复兴"被安排在10:00，教室D2202，记作：

```
scheduled(new-era2025, 2, 1000, D2202)
```

为了把这个关系用三元组表示，可以添加一个新个体：一个"预定"。这样，把 `scheduled` 这一关系具体化为一个有关预定的个体。一个预定有多个特性，即课程名称、章节、开始时间和房间号。为了表示上述信息，可以给预定起个名字，如 `b123`，然后得到如下表示：

```
prop(b123, course, new-era2025)
prop(b123, section, 2)
prop(b123, start_time, 10:00)
prop(b123, room, D2202)
```

这种表示方式有许多优点：
- 其一，采用模块化设计；
- 其二，特性与值的对应性清晰；
- 其三，易于添加新特性。

### 6.1.3 语义网络的定义

对于上述的 `prop` 关系，可以通过有向图进行表示。具体而言，可将三元组 `prop(Ind, Prop, Val)` 表示为两个节点 `Ind` 和 `Val`，以及一条连接这两个节点、标记为 `Prop` 的边。这类图通常被称为**语义网络** (Semantic Network)。

语义网络除了可以表示个体与个体之间的关系，还可以表示概念与概念之间的关系。

**定义6.1（语义网络）**：语义网络是一个图，其节点表示对象或概念，边表示这些对象或概念之间的关系。

**例6.1（关于电脑的语义网络）**：所有华为计算机都有Huawei商标，且有灰色和银色两种颜色。所有华为笔记本电脑MateBook 14都重1.31kg。华为笔记本电脑MateBook 14是华为计算机的子类。计算机comp_123是一台华为笔记本电脑MateBook 14。

一般地，语义网络通过由节点和边组成的图形来表示知识。其中，节点表示对象或概念，边表示节点之间的语义关系。语义网络还支持**继承**机制：子类可以继承父类的属性。教材中的图6.1展示了这种带标签的有向图结构，图6.2展示了允许继承的语义网络。

### 6.1.4 知识图谱

知识图谱是一种语义网络，用于描述现实世界中的各种实体及其关系。这里的实体可以是对象、事件、情境、概念等。知识图谱的核心是如何从大数据中提取结构化知识并以语义网络的形式加以表示。

与语义网络一样，知识图谱由如下三种成分构成：**节点**、**边**和**标签**。任何对象、地点或人可以是一个节点。边定义了这些节点之间的关系。

**核心概念理解：知识图谱**

可以把知识图谱看作一个"百科知识的网络版"：

- 每个节点是一个实体（人物、地点、事件等）
- 每条边表示实体间的关系
- 三元组是最小单元

这样，计算机就能像"画关系网"一样组织知识。

**例6.2（案例：知识图谱）**：以下三元组：

- `(爱因斯坦, 出生地, 德国)`
- `(爱因斯坦, 获得, 诺贝尔奖)`
- `(诺贝尔奖, 领域, 物理学)`

这些三元组连接成一张图，便于搜索问答系统回答："爱因斯坦获得了哪个领域的奖？" -> 物理学。

## 6.2 本体与描述逻辑

本体 (Ontology) 是一种知识表示方式，用于形式化描述某一领域中的概念、关系和公理。本体提供了一个共享的、可重用的知识框架，是语义网络和知识图谱的重要组成部分。

为了运用知识图谱中的各个元素进行推理，首先必须阐明这些元素及其关系的含义。在计算机领域，通常把符号的含义建构到文档中，即在计算机中的符号与人脑中的概念之间建立映射关系。更加具体地，可以使用统一资源标识符 (URI) 来表示个体或特性。URI之所以有含义，是因为人们在使用它时隐含了相关的含义。

另一方面，由于不同的建模者有不同的术语体系及其含义定义，为了实现知识共享，通常需要建立一个统一的术语体系并阐明其中各个元素的含义。我们把这样的术语体系称为**本体**。

本体通常由如下三个部分组成，可以被理解为一种用于描述特定领域概念及其关系的形式化规范：

1. **关于事物分类的词汇表**
2. **分类的组织**：如使用 `subClassOf` 或 `subPropertyOf` 定义继承关系
3. **一组公理集合**：用于限制一些符号的定义，以更好地反映期望的含义。例如，一些特性是传递的，或论域和值域是受限的，或对特性值的数量进行限制等。

**核心概念理解：什么是本体？**

本体就像一个"知识分类表"，告诉计算机：

- 世界上有哪些类（如人、科学家、地点）
- 这些类之间的关系（科学家属于人，爱因斯坦出生在德国）
- 应该遵守哪些约束（每个人至多有一个出生地）

区别在于：本体是机器可读的形式化表示，能支持逻辑推理。

### 知识图谱、本体和描述逻辑的关系

描述逻辑是一阶逻辑的子集，可以用于本体建模。在描述逻辑中，个体、概念和角色分别对应于一阶逻辑的个体常元、一元谓词和二元谓词。

知识图谱、本体和描述逻辑三者之间形成了层次分明、相互支撑的关系：

- **知识图谱**：以图结构组织实体和关系的实例数据
- **本体**：提供概念、关系与层次结构的统一建模方式（术语层）
- **描述逻辑**：为知识图谱和本体提供形式化语义基础和推理能力

目前，有多种描述逻辑。每种描述逻辑都有自己的表示语言、表达能力和计算复杂性。其中，属性语言 AL 是一种基本的语言，可以表示：原子的否定、概念交、全称约束、受限的存在量化。

ALC 是一种有代表性的语言，它是一种属性语言，但允许有任意的概念，而不仅仅是原子概念（"C"代表"补"即Complement，即支持任意概念的否定）。

### 6.2.1 描述逻辑的基本组成元素

描述逻辑提供了一种手段来建模一个特定领域中实体关系。在描述逻辑中，有三种实体：

- **概念** (Concept)：表示个体的集合
- **角色** (Role)：表示个体之间的二元关系
- **个体名称** (Individual Name)：代表一个领域中的单个个体

可以分别将它们视为一元谓词、二元谓词和常元。

例如，一个本体建模人及其家庭关系的领域可能使用：
- 概念如 `Parent` 来代表所有父母
- 概念如 `Female` 来表示所有的女性个体
- 角色如 `parentOf` 来表示父母与其子女之间的（二元）关系
- 个体名称如 `Julia` 和 `John` 来代表两个现实中的个体

不同于数据库，本体不能完全描述特定的"情况"或"世界的状态"，而是由一系列句子（称为公理）组成。每个公理都必须在所描述的情况下为真。这些公理通常只描述本体所描述的情况的部分知识，可能存在与本体相一致的多种世界状态。

虽然从逻辑的角度来看，不同类型的公理之间没有主要区别，但传统上将它们分为三类：

#### (1) ABox公理 — 断言型公理

ABox公理可以刻画关于命名个体的知识，即它们所属的概念以及它们相互之间的关系。

**概念断言** (Concept Assertion) 是最常见的ABox公理，如：

```
Mother(julia)                    (6.1)
```

这个公理断言Julia是一位母亲，或者更具体地说，名字为Julia的个体是 `Parent` 这个概念的实例。

**角色断言** (Role Assertion) 描述个体之间的关系。例如：

```
parentOf(julia, john)            (6.2)
```

这个断言说的是：julia是john的父母，或者更准确地说，被命名为julia的个体与被命名为john的个体之间的关系是由 `parentOf` 表示的。

#### (2) TBox公理 — 术语型公理

TBox公理描述概念之间的关系。

**概念包含** (Concept Inclusion)：例如，"所有母亲都是父母"可以用概念包含表示如下：

```
Mother ⊑ Parent                  (6.3)
```

这时，我们说概念 `Mother` 被概念 `Parent` 所包含。这种知识可以用于推出关于个体的进一步事实。例如，公式(6.1)和(6.3)一起意味着Julia是父母。

**概念等价** (Concept Equivalence)：断言两个概念有相同的实例。

```
Person ≡ Human                   (6.4)
```

同义词是同等概念的一个显然的例子，但实际上人们更常常用概念等价来给复杂表达式一个名字。此外，这样的额外概念表达式可以结合等价和包含来描述更复杂的情况，例如，概念的不相交，它断言两个概念不共享任何实例。

#### (3) RBox公理 — 关系型公理

RBox公理是关于角色的特性的。描述逻辑支持角色包含公理和角色等价公理。

```
parentOf ⊑ ancestorOf            (6.5)
```

这个公理表明 `parentOf` 是 `ancestorOf` 的子角色，即由 `parentOf` 关联的每对个体也是由 `ancestorOf` 关联的个体。因此，公式(6.2)和(6.5)一起蕴涵Julia是John的祖先。

### 6.2.2 概念和角色的构造算子

上一节介绍的基本类型公理对于精确的建模是相当有限的。为了描述更复杂的情况，描述逻辑允许使用各种不同的构造算子来构建新的概念和角色。

#### 布尔概念构造算子

布尔概念构造算子提供了基本的布尔运算，这些运算与人们熟悉的集合的交集、并集和补集运算，或者与逻辑表达式的合取、析取和否定运算密切相关。

**交集（合取）**：

例如，概念包含允许我们声明所有的母亲都是女性，所有的母亲是父母，但我们真正的意思是，母亲正是女性父母。描述逻辑通过允许我们形成复杂的概念来支持这种陈述，例如交集：

```
Female ⊓ Parent                  (6.6)
```

上述公式表示一组既是女性又是父母的个体集合。复杂概念可以与原子概念一样在公理中使用。例如，可以这样表示一个概念等价：

```
Mother ≡ Female ⊓ Parent
```

**并集（析取）**：

并集是交集的对偶。例如：

```
Father ⊔ Mother                  (6.7)
```

这一概念表示一组或者是父亲或者是母亲的个体集合。同样，它可以被用于公理中，如：

```
Parent ≡ Father ⊔ Mother
```

**补集（否定）**：

有时，我们需要表达不属于某个概念的个体，例如，未结婚的女性。这时，可以用如下复杂概念来描述：

```
Female ⊓ ¬Married                (6.8)
```

#### 角色限制

到现在为止，我们已经看到了如何分别使用TBox和RBox公理来表达概念和角色之间的关系。然而，描述逻辑最有趣的特点是能够形成将概念和角色联系在一起的语句。

**存在限制** (Existential Restriction)：

例如，概念 `Parent` 和角色 `parentOf` 之间存在明显的关系，即父母是至少一个人的父母。在描述逻辑中，这种关系可以通过如下概念等价来表达：

```
Parent ≡ ∃parentOf.⊤             (6.9)
```

其中，存在限制 `∃parentOf.⊤` 是一个复杂概念，它描述了这样的个体集合：其中的每个个体都是至少一位个体（`⊤` 的实例，即论域中的任意个体）的父母。

类似地，`∃parentOf.Female` 这个概念描述了那些至少有一个女性孩子的父母，即那些有女儿的人。

**全称限制** (Universal Restriction)：

为了表达所有孩子都是女性的个体的集合，可以使用全称限制：

```
∀parentOf.Female                 (6.10)
```

关于全称限制，常见的一个错误是没有想到上式也包含那些根本没有孩子的人。更准确地说，这条公理可以用来描述所有"除了女性孩子之外没有其他孩子"的个体的集合，即"没有非女性的孩子"。

根据这一说法，上述概念实际上可以等价地表达为：

```
¬∃parentOf.¬Female
```

如果不是要表达这个意思，人们可以用概念：

```
(∃parentOf.⊤) ⊓ (∀parentOf.Female)
```

来描述至少有一个孩子并且所有孩子都是女性的个体。

### 6.2.3 描述逻辑 ALC 的语法

本节开始介绍一种基本的描述逻辑 ALC。形式上，每个描述逻辑本体都基于三组有穷的符号集合：`N_C`、`N_R` 和 `N_O`，它们分别表示**概念名集合**、**角色名集合**和**个体名集合**。这样，三元组 `(N_C, N_R, N_O)` 构成描述逻辑的名字表 (Signature)。

**定义6.2（概念）**：ALC 概念集合是满足如下条件的极小集合：

- `⊤` 和 `⊥` 是概念（顶概念和底概念）
- `A ∈ N_C`：每个概念名是概念
- 如果 `C` 和 `D` 是概念，且 `R ∈ N_R` 是角色名，那么如下是概念：
  - `C ⊓ D`：两个概念的交 (Conjunction/Intersection)
  - `C ⊔ D`：两个概念的并 (Disjunction/Union)
  - `¬C`：一个概念的补 (Negation/Complement)
  - `∀R.C`：由一个角色对一个概念的全称限制 (Universal Restriction / Value Restriction)
  - `∃R.C`：由一个角色对一个概念的存在限制 (Existential Restriction)

每个角色名指称一组对象元组的集合。典型的例子：`hasChild` 和 `loves`。概念 `⊤` 表示领域中所有对象的集合。概念建构符 `⊓` 和 `∃` 分别表示集合交和存在约束。

**例6.3（有关人类的概念）**：设 `Human` 和 `Female` 是概念名，`hasChild`、`gender`、`hasParents` 是角色名。那么，我们可以得到如下概念：

- `∃hasChild.⊤` — 有孩子的东西
- `Human ⊓ ∃hasChild.⊤` — 有孩子的人
- `Human ⊓ ∃hasChild.Human` — 孩子是人的人
- `Human ⊓ ∃gender.Female` — 女人
- `Human ⊓ ∃hasChild.⊤ ⊓ ∃hasParent.⊤` — 有孩子和父母的人
- `Human ⊓ ∃hasChild.∃gender.Female` — 有女儿的人
- `Human ⊓ ∃hasChild.∃hasChild.⊤` — 有孙子的人

**定义6.3（术语公理）**：普通概念包含形如 `C ⊑ D`，其中 `C` 和 `D` 是概念。当 `C ⊑ D` 且 `D ⊑ C` 时，记作 `C ≡ D`。

**TBox** 是一组术语公理的有穷集合。

**例6.4（一个TBox的例子）**：

```
Father ≡ Person ⊓ ∃gender.Male ⊓ ∃hasChild.⊤
Student ≡ Person ⊓ ∃isRegisteredAt.University
Father ⊑ Person
Father ⊑ ∃hasChild.⊤
```

**TBox的扩张**：

给定一个TBox `T`，我们把出现于 `T` 中的原子概念分为两个集合：

- **名称符号集合** `N_T`：出现于公理左侧的符号，称为**被定义的概念**
- **基符号集合** `B_T`：出现于公理右侧的符号，称为**原始概念**

我们希望可以用基符号来定义名称符号。

当一个TBox中的概念定义是无环的时，可以通过一组原子的概念来定义所有其他概念。

设 `A`、`B` 是出现在 `T` 中的原子概念。如果 `B` 出现在 `A` 的定义的右侧，则 `A` 在 `T` 中**直接使用** `B`。我们把"使用"关系看作是"直接使用"关系的传递闭包。这样，当 `T` 中存在一个使用自身的原子概念时，`T` 是**有环**的。否则，`T` 被称为**无环**的。

对于一个无环的TBox `T`，可以通过一个迭代过程来扩张 `T` 中的定义，将定义右侧的每个名称替换为该名称所代表的概念。由于定义集合中没有环，这个过程最终停止，并得到了一个TBox `T'`，它只包含形如 `A ≡ C'` 的定义，其中 `C'` 只包含基符号，不包含名称符号。我们把 `T'` 称为 `T` 的**扩张**。`T` 与 `T'` 是等价的。

**例6.5（TBox的扩张）**：给定如下TBox `T1`：

```
Woman ≡ Person ⊓ Female
Man ≡ Person ⊓ ¬Woman
Mother ≡ Woman ⊓ ∃hasChild.Person
Father ≡ Man ⊓ ∃hasChild.Person
Parent ≡ Mother ⊔ Father
Grandmother ≡ Mother ⊓ ∃hasChild.Parent
```

`T1` 的扩张 `T1'` 为：

```
Woman ≡ Person ⊓ Female
Man ≡ Person ⊓ ¬(Person ⊓ Female)
Mother ≡ (Person ⊓ Female) ⊓ ∃hasChild.Person
Father ≡ (Person ⊓ ¬(Person ⊓ Female)) ⊓ ∃hasChild.Person
Parent ≡ ((Person ⊓ Female) ⊓ ∃hasChild.Person) ⊔ ((Person ⊓ ¬(Person ⊓ Female)) ⊓ ∃hasChild.Person)
Grandmother ≡ ((Person ⊓ Female) ⊓ ∃hasChild.Person) ⊓ ∃hasChild.(((Person ⊓ Female) ⊓ ∃hasChild.Person) ⊔ ((Person ⊓ ¬(Person ⊓ Female)) ⊓ ∃hasChild.Person))
```

**定义6.4（断言公理）**：

- **概念断言**是一个形如 `a : C` 的句子，其中 `a ∈ N_O`，`C` 是一个概念
- **角色断言**是一个形如 `(a, b) : R` 的句子，其中 `a, b ∈ N_O`，`R` 是一个角色

**ABox** 是一组断言公理的有穷集合。

**定义6.5（知识库）**：在描述逻辑中，一个知识库是一个有序对 `(T, A)`，其中 `T` 是TBox，`A` 是ABox。

**例6.6（案例：描述逻辑公理）**：

```
Scientist ⊑ Person              （所有科学家都是人）
Einstein : Scientist             （爱因斯坦是科学家）
```

### 6.2.4 描述逻辑 ALC 的语义

描述逻辑的语义通过如下方式来定义：把概念解释为个体的集合，而把角色解释为个体有序对的集合。

一个解释 `I` 由一个称为 `I` 的域的集合 `Δ^I` 和一个解释函数 `·^I` 组成，该函数将：

- 每个原子概念 `A` 映射到一个集合 `A^I ⊆ Δ^I`
- 每个原子角色 `R` 映射到二元关系 `R^I ⊆ Δ^I × Δ^I`
- 每个个体名称映射到一个元素 `a^I ∈ Δ^I`

在此基础上，非原子概念和角色的语义由原子概念和角色的语义来定义。

**定义6.6（概念的解释）**：给定名字表 `(N_C, N_R, N_O)`，相应的术语解释定义为 `I = (Δ^I, ·^I)`。其中：

- 非空集合 `Δ^I` 称为**论域**
- 解释函数 `·^I` 包括如下映射：
  - 把每个个体 `a` 映射为元素 `a^I ∈ Δ^I`
  - 把每个概念映射为 `Δ^I` 的子集
  - 把每个角色名映射为 `Δ^I × Δ^I` 的子集

使得：

| 语法 | 语义（解释） |
|------|-------------|
| `⊤^I` | `Δ^I` |
| `⊥^I` | `∅` |
| `(C ⊔ D)^I` | `C^I ∪ D^I` |
| `(C ⊓ D)^I` | `C^I ∩ D^I` |
| `(¬C)^I` | `Δ^I \ C^I` |
| `(∀R.C)^I` | `{x ∈ Δ^I ∣ ∀y 如果 (x, y) ∈ R^I，则 y ∈ C^I}` |
| `(∃R.C)^I` | `{x ∈ Δ^I ∣ ∃y ∈ Δ^I : (x, y) ∈ R^I 且 y ∈ C^I}` |

**例6.7（一个概念解释的例子）**：设解释 `I = (Δ^I, ·^I)`，如教材图6.3所示，其中：

- `Δ^I = {a, b, c, d, Y, Z}`
- `Person^I = {a, b, c, d}`
- `Female^I = {Y}`
- `hasChild^I = {(a, b), (b, c)}`
- `gender^I = {(a, Y), (b, Z), (c, Y)}`

于是，可以得到如下概念的解释：

- `(Person ⊓ ∃gender.⊤)^I = {a, b, c}` — 因为 `Person^I = {a, b, c, d}`，`(∃gender.⊤)^I = {a, b, c}`
- `(Person ⊓ ∃gender.Female)^I = {a, c}` — 因为 `(∃gender.Female)^I = {a, c}`
- `(Person ⊓ ∃hasChild.Person)^I = {a, b}` — 因为 `(∃hasChild.Person)^I = {a, b}`
- `(Person ⊓ ∃hasChild.∃gender.Female)^I = {b}`
- `(Person ⊓ ∃hasChild.∃hasChild.⊤)^I = {a}`

## 6.3 描述逻辑 ALC 的推理

基于描述逻辑的知识表示系统可以进行具体形式的推理，包括：

1. 关于概念的推理
2. 关于TBox或ABox的推理
3. 关于包含TBox和ABox的知识库的推理

最后，这些推理都可以**规约到一个主要的推理类型**，即**ABox的一致性检查**。

### 6.3.1 关于概念的推理

当知识工程师为一个领域建模时，她会通过定义新的概念来构建一个术语体系，比如 `T`。在这个过程中，重要的是要弄清楚一个新定义的概念是否有意义或是否矛盾。从逻辑的角度来看，如果有某个解释满足 `T` 的公理（即 `T` 的模型），使得该概念在该解释中表示一个非空集，那么该概念对我们来说是有意义的。具有此性质的概念被称为是关于 `T` **可满足的**，否则是关于 `T` **不可满足的**。

检查概念的可满足性是一个关键的推理。关于概念的许多其他重要推理问题可以归结为可满足性问题。

**定义6.7（可满足 Satisfiability）**：设 `T` 是一个TBox。我们说概念 `C` 是关于 `T` 可满足的，如果存在 `T` 的一个解释 `I`，使得 `C^I` 非空。

**定义6.8（包含 Subsumption）**：设 `T` 是一个TBox。关于 `T`，我们说概念 `C` 是被包含于 `D`，如果对于 `T` 的每个解释 `I`，`C^I ⊆ D^I` 成立，记作 `T ⊨ C ⊑ D`。

**定义6.9（等价 Equivalence）**：设 `T` 是一个TBox。我们说概念 `C` 与 `D` 关于 `T` 等价，如果对于 `T` 的每个解释 `I`，`C^I = D^I`，记作 `T ⊨ C ≡ D`。

**定义6.10（不相交 Disjointness）**：设 `T` 是一个TBox。我们说概念 `C` 与 `D` 关于 `T` 不相交，如果对于 `T` 的每个解释 `I`，`C^I ∩ D^I = ∅`。

当上下文清楚时，有时也省去"关于 `T`"。

**包含、等价和不相交三种概念可以被规约到可满足问题**。对于概念 `C` 和 `D`，如下命题成立：

1. `C` 被 `D` 包含，当且仅当 `C ⊓ ¬D` 不可满足
2. `C` 与 `D` 等价，当且仅当 `C ⊓ ¬D` 与 `¬C ⊓ D` 都不可满足
3. `C` 与 `D` 不相交，当且仅当 `C ⊓ D` 不可满足

### 6.3.2 去除TBox

在实际应用中，概念通常以TBox的背景出现。然而，对于开发推理算法来说，通过抽象掉TBox，即使得TBox为空，可以使算法在概念上变得简单。

如果 `T` 是一个**无环**的TBox，则总是可以把关于 `T` 的推理问题规约到关于空TBox的推理问题。

例如，对于概念 `Woman ⊓ Man`，可以通过扩张（见例6.5），得到如下概念：

```
Person ⊓ Female ⊓ Person ⊓ ¬(Person ⊓ Female)    (6.11)
```

依据 `T` 把一个概念 `C` 扩张为 `C'` 后，二者在语义上是关于 `T` 等价的。因此：

- `C` 是关于 `T` 可满足的，当且仅当 `C'` 是关于 `T` 可满足的
- 然而，由于 `C'` 不包含被定义的名称，因此 `C'` 是关于 `T` 可满足的当且仅当 `C'` 是可满足的

于是，我们有：

- `C` 是关于 `T` 可满足的，当且仅当 `C'` 是可满足的
- 如果 `D` 是另一个概念，那么我们也有：`D` 是关于 `T` 可满足的，当且仅当 `D'` 是可满足的

因此，如下结论成立：

- `T ⊨ C ⊑ D` iff `⊨ C' ⊑ D'`
- `T ⊨ C ≡ D` iff `⊨ C' ≡ D'`

同理，如下结论也成立：

- `C` 和 `D` 是关于 `T` 不相交的，当且仅当 `C'` 和 `D'` 是不相交的

总之，关于无环TBox，通过扩张概念可以在推理问题中去掉TBox。

### 6.3.3 关于ABox的推理

在知识工程师设计了一个术语体系并使用她的描述逻辑系统的推理服务来检查所有概念是否都是可满足的，以及预期的包含关系是否成立后，可以向ABox填充关于个体的断言。

ABox包含两种断言：

1. 形如 `a : C` 的**概念断言**
2. 形如 `(a, b) : R` 的**角色断言**

显然，这些知识的表示必须是**一致的** (Consistent)。否则，从逻辑的角度来看，人们从中可以得出任意的结论。

与概念类似，关于一个无环的TBox，检查ABox的一致性可以规约为检查一个扩张的ABox的一致性。

关于概念的可满足性检查、ABox的一致性检查以及涉及普通包含公理的一致性检查可以用一种**表算法** (Tableau Algorithm) 来实现。关于表算法的具体介绍，详见相关文献。

## 6.4 小结

在大数据驱动的人工智能发展背景下，知识图谱成为连接符号主义与亚符号主义方法的关键桥梁，为深度学习模型提供结构化语义支持，也为逻辑推理系统注入可解释性语义结构。本讲围绕"结构化知识表示"的逻辑方法展开，聚焦于知识图谱与描述逻辑的建模与推理机制。

首先介绍了知识图谱的基本构成要素，包括实体和关系，并阐述了其基于图结构的表示方式，帮助读者理解如何以符号化图谱组织异构知识。在此基础上，引入描述知识图谱语义的核心逻辑体系——描述逻辑，并以 ALC 为代表，详细介绍了其基本语法元素、构造算子与模型语义。ALC 逻辑能够定义复杂概念与角色，刻画TBox（术语库）与ABox（断言库），并通过概念层与实例层的分离实现灵活推理。

描述逻辑在保持推理可判定性的同时，还具备良好的工程可实现性，是语义网、本体工程与知识问答系统的重要技术基础。它不仅继承了一阶逻辑的表达能力，还通过模型约束提升了推理效率，是人工智能知识建模中"语言—语义—推理"三位一体思想的典型代表。

同时，本讲也具有哲学与方法论启示。知识图谱与描述逻辑揭示了"对象—属性—关系"的形式化结构，并通过全称约束（必然性）与存在约束（可能性）的区分，提供了刻画知识限制与开放性的逻辑工具。它们不仅推动了人工智能的语义建模能力，也为哲学中关于本体论、语义学与认识论的问题提供了新的形式化分析路径。

下一讲将从"静态结构化知识"的封闭世界出发，进入非单调逻辑的开放式推理场景，探索如何在面对不完全、例外性甚至矛盾性知识时进行灵活而合理的逻辑推理。

## 学习要点与AI关联

### 知识图谱

以实体—关系—属性三元组为基本单元，用图结构组织和整合异构知识，是人工智能中大规模知识建模的核心形式。它为搜索引擎、智能问答、推荐系统、语义检索等任务提供了语义支撑。

### 描述逻辑与本体

描述逻辑是知识图谱的语义基础，本体则提供概念、关系与层次结构的统一建模方式。描述逻辑通过TBox（术语库）与ABox（断言库）的区分，实现了概念定义与实例信息的分层表达。在AI中，它支撑语义网、知识管理、语义互操作等场景。

### 推理机制（以 ALC 为例）

描述逻辑能刻画复杂概念与角色关系，提供概念包含检验、实例检查、一致性判断等推理能力。在AI应用中，这些推理任务是知识问答系统、语义一致性验证、知识库扩展的关键环节。

### AI应用价值

知识图谱与描述逻辑的结合，使智能系统能够从"数据驱动"走向"知识驱动"。它们为搜索、推荐、法律推理、医学知识库、数字人文等领域提供了形式化和可解释的知识建模与推理框架。

### 哲学与方法论启示

知识图谱与描述逻辑揭示了"对象—属性—关系"的形式化结构，并为"必然性与可能性"的区分提供了逻辑工具。它们不仅推动了AI的语义建模能力，也为哲学中的本体论、语义学与认识论问题提供了新的形式化分析路径。
