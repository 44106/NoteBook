# 第12讲 归纳推理与归纳逻辑编程

## 概述

上一讲介绍的概率理论通过定量的概率值对不确定性进行量化表示，更加适用于需要精确计算的场景（如风险评估、市场分析等）。而本章将要介绍的归纳推理是从观察事例到一般原理的推理模型。两者核心差异在于：**概率理论**以数学量化不确定性，侧重于对数据进行量化推理；**归纳推理**以逻辑推导处理不确定性，侧重于对事例进行归纳总结。本讲将介绍归纳推理的核心概念和归纳逻辑编程。

---

## 12.1 归纳推理

归纳推理是一种**证据支持的推理**。在演绎推理中，有效演绎论证的前提充分支持结论。在一个好的归纳论证中，前提的真给结论的真提供一定程度的支持。这里的支持程度通常用数字尺度来描述。

与演绎蕴涵的概念类比，归纳的支持程度的意思可以表达为：在那些逻辑上使得前提为真的可能的世界状态中，结论必须（只要）在比例r上为真。其中，r是支持强度的某种数字度量。

由于归纳论证的前提真不能确保结论真，因此归纳论证也被称为**有风险论证**。

### 例12.1（农产品的质量论证）

考虑如下场景：扶贫办计划向贫困地区居民收购一批农产品。在收购前，双方分别采用不同抽样方法对整批农产品的质量进行评估。

#### 1. 论证A（非随机抽样）

- **前提**：扶贫办工作人员从农产品批次的上层取出一件，验证其质量良好。
- **结论**："该批次所有农产品（几乎）都质量良好"。

这个论证的前提可以作为结论的证据，但不是一个很好的证据，因为上层农产品可能因堆放位置受保护，无法代表整体（如底部农产品可能受潮或发霉）。论证A不是一个有效论证。即使它的前提为真，它的结论也可为假。这是一个有风险的论证。

#### 2. 论证B（简单随机抽样）

- **前提**：贫困地区居民从批次中随机抽取一件农产品进行质量验证，发现该农产品质量良好。
- **结论**："该批次所有农产品（几乎）都质量良好"。

这个论证仍然是有风险的，但它不像论证A的风险那么大。

#### 3. 论证C（扩大随机抽样）

- **前提**：随机抽取6件农产品，其中5件质量良好，1件质量较差（如受潮或发霉）。
- **结论**："该批次大部分农产品质量良好"。

与论证B相比，论证C基于更多的数据，但它也不是一个有效的论证。即使在6件农产品中5个是好的，但也许只是此次抽取的运气使然。可能其他的多数农产品是受潮或发霉的。

### 归纳论证的基本形式

风险论证有多种。论证A、B、C都有如下基本形式：

- 从一个给定的总体中抽出的样本具有某个性质。
- 因此，总体中的所有个体具有这个性质。

归纳论证的强度可以使用概率来度量。对于有风险的归纳论证，其结论的概率具有风险性。

### 例12.2（关于概率的归纳论证——风险性）

- **前提**：这个骰子有六面，分别标记数字1, 2, 3, 4, 5, 6，在227次投掷中，得到4的次数为38。
- **结论**：因此，掷这个骰子得到4的概率大约是1/6。

**风险性分析**：频率接近期望值不代表骰子绝对均匀（可能存在物理偏差），属于典型归纳论证。即，这个论证的结论可能为假，尽管前提为真。这个骰子可能对出现4有偏差。所以，得到4的概率可能为1/8。

### 例12.3（关于概率的演绎论证——必然性）

相反，在那些使用概率进行推理的论证中，如果前提真可以确保结论真，那么这些论证在本质上是**演绎的**。它们的结论的概率具有**必然性**。

- **前提**：这个骰子有六面，分别标记数字1, 2, 3, 4, 5, 6，每一面出现的可能性相同。
- **结论**：这个骰子得到4的概率是1/6。

**性质**：由前提严格推导，属有效演绎论证。

根据概率公理，当事件是相互不包含（骰子每次只能有一面向上）且穷尽所有可能（骰子每次朝上的面只能是六种情况之一），所有情况的概率之和等于1。这个论证之所以是有效的，是因为给定概率基本规律，只要这种形式的论证的前提为真，那么结论也必然为真。

### 例12.4（复合事件的情形）

- **前提**：同上。
- **结论**：掷出3或4的概率是1/3。

**有效性**：基于概率加法原理，结论必然成立。

## 12.2 归纳逻辑编程

### 背景与动机

神经网络方法虽然可以从训练的数据集中抽取重要的特征，具有良好的预测性能，但需要耗费大量算力资源。在一些情况下，其性能等于简单的符号分类器。同时，在神经网络中，难以表征结构化知识。此外，为了提高神经网络的可解释性，可以通过建立特征与结果之间的关系，但难以刻画人类水平的知识。

与符号机器学习不同，数字神经网络是以如下方式来进行归纳学习的：通过权值来进行统计特征的表达。

例如：

- **逻辑表达**："如果乡村加大投入推动农业现代化，且没有遭遇自然灾害的干扰，那么乡村经济将实现显著增长"
- **神经网络表达**："W_modernize,grow = 0.989，W_naturalDisasters,grow = -0.966"

### 归纳学习的基本思想

学习的一种关键形式是**归纳**，即从具体观察到形成一般规则的过程。如果从袋子里抽取了10个球都是红色的，那么可以归纳出一个假说，即袋子里所有的球都是红色的。这样，可以去预测下个从袋子里取出的球的颜色。

机器学习自动化归纳过程。机器学习归纳出一个假说，也称为一个模型，该假说概括训练例子。例如，给定一组猫和狗的标注数据集，机器学习的目标是去归纳出一个假说，来预测一个未标记的图片是一只猫或狗。

### ILP的定义

**归纳逻辑编程（Inductive Logic Programming, ILP）**是一种机器学习技术，用于进行有监督的归纳概念学习。

其基本思路是：给定一组带标签的例子集合E和一个背景知识集合B，一个ILP系统试图寻找一个假说H来极小化一个具体的损失loss(B ∪ H, E)。

在ILP的语境下：
- **B**为一个逻辑程序（背景知识）
- **E**是一组目标概念的基原子集合，其中的标签是真值
- **H**是一个目标逻辑程序，ILP系统试图通过学习生成H，使其与B结合后能正确解释E中的例子

### ILP与其他机器学习方法的区别

大部分机器学习方法学习的是**函数**，而ILP学习**关系**。同时，大多数机器学习方法采用统计推断。相反，ILP依赖于逻辑推断，并使用自动推理和知识表示技术。

ILP的特点是：**少样本、可解释、可迁移**。ILP可以用复杂的关系信息进行学习，且易于与专家知识集成。

### 定义12.1（ILP的任务）

把一个ILP任务定义为三元组(B, E+, E-)。其中：
- **B**是背景知识
- **E+**是正面事例（Positive Examples）
- **E-**是反面事例（Negative Examples）

在ILP中，背景知识、例子和假说都被表示为逻辑程序。

### 例12.5（乐高建造者的快乐假说）

考虑如下ILP任务。其中：

**背景知识B**：
```prolog
lego_builder(beibei).
lego_builder(jingjing).
estate_agent(huanhuan).
estate_agent(yingying).
enjoys_lego(jingjing).
enjoys_lego(huanhuan).
```

**正例E+**：
```prolog
E+ = {happy(jingjing).}
```

**反例E-**：
```prolog
E- = {happy(beibei). happy(huanhuan).}
```

数据总结表：

| Name    | Job           | Enjoys lego | Happy |
|---------|---------------|-------------|-------|
| beibei  | lego builder  | no          | no    |
| jingjing| lego builder  | yes         | yes   |
| huanhuan| estate agent  | yes         | no    |
| yingying| estate agent  | no          | no    |

### 定义12.2（归纳逻辑编程）

给定一个三元组(B, E+, E-)，返回一个假说H，使得：

- **完全性（Completeness）**：∀e ∈ E+，H ∪ B ⊨ e（H覆盖所有正例）
- **一致性（Consistency）**：∀e ∈ E-，H ∪ B ⊭ e（H不覆盖任何反例）

### 例12.6（乐高建造者的快乐假说——续）

设有如下假说空间：
```prolog
H = {h1: happy(x) :- lego_builder(x).
     h2: happy(x) :- estate_agent(x).
     h3: happy(x) :- enjoys_lego(x).
     h4: happy(x) :- lego_builder(x), estate_agent(x).
     h5: happy(x) :- lego_builder(x), enjoys_lego(x).
     h6: happy(x) :- estate_agent(x), enjoys_lego(x).}
```

对于上述假说空间，ILP系统逐一评估每个假说：

| 假说 | 评估结果 | 问题 |
|------|----------|------|
| B ∪ {h1} | ⊨ happy(beibei) | 不一致（覆盖了反例beibei） |
| B ∪ {h2} | ⊭ happy(jingjing) | 不完全（未覆盖正例jingjing） |
| B ∪ {h3} | ⊨ happy(huanhuan) | 不一致（覆盖了反例huanhuan） |
| B ∪ {h4} | ⊭ happy(jingjing) | 不完全（未覆盖正例jingjing） |
| **B ∪ {h5}** | | **既完全又一致** |
| B ∪ {h6} | ⊭ happy(jingjing) | 不完全（未覆盖正例jingjing） |

因此，ILP返回的假说是**h5**：`happy(x) :- lego_builder(x), enjoys_lego(x).`

### ILP的三种搜索方法

#### 1. 自顶向下方法（Top-Down）

利用逻辑程序的一般性和具体性，从一个**一般子句**开始，逐步添加更多文字来减少覆盖范围直到满足预先指定的条件。这一过程称为**子句的具体化（Specialization）**。

依据普遍性关系，可以获得假说空间上的偏序关系。系统可依据该关系来提高搜索假说的效率。例如，如果一个子句不蕴涵一个肯定例子，则不需要搜索它的具体化，因为该子句的具体化不可能蕴涵该例子。

#### 2. 自底向上方法（Bottom-Up）

又称为**数据驱动策略**。以一个具体的子句开始，逐步移除文字来增加样本的覆盖范围，直到满足预先指定的条件。这一过程称为**子句的一般化（Generalization）**。

#### 3. 元级方法（Meta-Level）

与直接在假说图中搜索不同，元级方法将归纳逻辑编程程序编码为元级逻辑程序，然后求解该程序以获得最优假设。用于表达问题规范的形式化方法包括Prolog和回答集编程（ASP），使用现有的Prolog系统和回答集求解器来求解约束条件。

## 12.3 小结

本讲从归纳推理的基本理念出发，引入了人工智能中用于学习逻辑规则的形式方法——归纳逻辑编程（ILP），探讨了如何在有限观测的基础上自动构建可解释的逻辑知识体系。

**在推理模式上**，归纳推理不同于演绎推理的"从规则到结论"与溯因推理的"从结果到原因假设"，其核心任务是从事实中归纳出规则。ILP通过结合背景知识与正反例信息，搜索满足一致性与简洁性要求的逻辑规则，是"数据驱动逻辑学习"的代表方法之一，广泛应用于结构归纳、程序合成与可解释学习等任务。

**在因果建模任务中**，ILP所生成的规则结构也可辅助构建因果图的候选依赖关系，或作为因果发现方法的结构先验。虽然ILP本身并不直接支持干预运算或反事实评估，但其基于逻辑规则的建模语言在因果系统结构表达中具有一定借用价值，尤其适合组织可解释、可组合的知识结构。

**在人工智能应用方面**，ILP连接了逻辑推理与机器学习，为生物信息学（如蛋白质结构预测）、知识图谱补全、自然语言处理和自动化科学发现等领域提供了可解释的学习框架。它能够在保证形式化严谨性的同时，从数据中发现模式与规则，是推动"符号主义与统计学习融合"的重要途径。

**在哲学层面**，归纳推理和ILP具有方法论意义。它们为"经验与理性如何结合"这一经典问题提供了形式化解答，使得关于科学发现、知识获取与可解释性的哲学讨论能够在逻辑模型中得到具体呈现。因此，归纳逻辑编程不仅是AI的技术工具，也是认识论与科学哲学的重要研究支点。

## 学习要点与AI关联

- **归纳推理**：不同于演绎推理的"从规则到结论"以及溯因推理的"从结果到原因"，归纳推理的核心是从具体事实中总结出一般规律，为知识学习与发现提供逻辑途径。
- **归纳逻辑编程（ILP）**：ILP将背景知识与正反例结合，搜索满足一致性与简洁性约束的逻辑规则，是连接逻辑推理与机器学习的代表性方法。它能够自动生成形式化规则，支撑可解释的知识建模。
- **AI应用价值**：ILP广泛应用于生物信息学（蛋白质结构预测）、知识图谱补全、自然语言处理与科学发现等任务。它兼顾逻辑的形式化严谨性与学习的适应性，是符号主义与统计学习融合的重要路径。
- **哲学与方法论启示**：归纳推理与ILP回应了"经验与理性如何结合"这一经典问题，为科学发现、知识获取与可解释学习提供了逻辑化的模型。它们不仅是AI的技术工具，也为科学哲学与认识论中的经验主义与理性主义对话提供了新的方法论框架。

16. **分析题**：考虑如下ILP任务：
    ```
    B = { grandfather(X,Y) ← father(X,Z), parent(Z,Y)
          father(kebi,chenggong) ←
          mother(chenggong,hehua) ←
          mother(chenggong,heye) ← }
    E+ = { grandfather(kebi,hehua) ←, grandfather(kebi,heye) ← }
    E- = { ← grandfather(hehua,kebi), ← grandfather(heye,hehua) }
    ```
    给定假说：`H = { parent(X,Y) ← mother(X,Y) }`
    
    判断H是否是完全且一致的。

    **分析**：
    - B ∪ H ⊨ grandfather(kebi, hehua)（因为mother(chenggong, hehua)推出parent(chenggong, hehua)，再结合father(kebi, chenggong)推出grandfather(kebi, hehua)），所以覆盖了正例 ✓
    - B ∪ H ⊨ grandfather(kebi, heye)（同理），所以覆盖了正例 ✓
    - B ∪ H是否能推出grandfather(hehua, kebi)？不能，因为hehua没有孩子。所以不覆盖反例 ✓
    - B ∪ H是否能推出grandfather(heye, hehua)？不能。所以不覆盖反例 ✓
    - **结论**：H是完全且一致的。
