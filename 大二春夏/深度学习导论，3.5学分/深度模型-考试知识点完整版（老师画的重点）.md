## 一、卷积的所有计算

### 1. 单步卷积计算

卷积核在输入图像上滑动，每次取一个局部区域，做**逐元素相乘再求和**。

**数值示例**：

输入（3×3）：
```
26   120  126
56    76   99
46   106  101
```

卷积核（2×2）：
```
1    0
0    1
```

核在左上角时的计算： 26×1 + 120×0 + 56×0 + 76×1 = 102

核向右滑动一格： 120×1 + 126×0 + 76×0 + 99×1 = 219

核向下滑动一格： 56×1 + 76×0 + 46×0 + 106×1 = 162

核再向右滑动： 76×1 + 99×0 + 106×0 + 101×1 = 177

最终输出（2×2 特征图）：
```
102   219
162   177
```

### 2. 多通道卷积（RGB 三通道）

输入形状：C_in × H × W（如 RGB = 3 × 224 × 224）。**卷积核深度必须等于输入通道数**。

计算：卷积核的每个通道分别和输入对应通道做二维卷积 → 累加各通道结果 → 一个输出通道。要得到 C_out 个输出通道 → 用 C_out 个独立的卷积核。

参数量 = C_out × C_in × K_H × K_W

### 3. 输出尺寸公式（必须会手算）

$$H' = \frac{H - K + 2P}{S} + 1, \quad W' = \frac{W - K + 2P}{S} + 1$$

| 符号 | 含义 |
|------|------|
| H, W | 输入的高度和宽度 |
| K | 卷积核尺寸（假设方形 K×K） |
| P | Padding（每边补几圈 0） |
| S | Stride（步长） |

**计算示例**：输入 32×32，K=5，P=2，S=1 → H' = (32-5+4)/1+1 = 32

⚠️ 结果必须是正整数，否则卷积核无法整齐滑过。

### 4. 参数量计算

- **卷积层权重**：C_out × C_in × K_H × K_W
- **卷积层偏置**：C_out
- **全连接层**：输入神经元数 × 输出神经元数 + 输出神经元数（偏置）

**计算示例**：输入 224×224×3（150,528 维），输出 1000：
- 全连接：150,528 × 1,000 + 1,000 ≈ 1.5 亿
- 卷积（1000 个 3×3×3 核）：3 × 3 × 3 × 1,000 + 1,000 = 28,000
- 差距约 5500 倍

### 5. 感受野堆叠公式

$$R = 1 + L \times (K - 1)$$

| 堆叠方式 | 感受野 | 参数量（输入输出通道均为 C） |
|----------|:------:|:---------------------------:|
| 1 层 3×3 | 3×3 | 9C² |
| 2 层 3×3 | 5×5 | 18C² |
| 3 层 3×3 | 7×7 | 27C² |
| 1 层 7×7 | 7×7 | 49C² |

**结论**：3 个 3×3 和 1 个 7×7 感受野相同，但参数量少 45%，且中间多了 2 次 ReLU。

**计算示例**：输入 224×224，只用 3×3 卷积不用池化，需要多少层覆盖全图？1 + L×(3-1) = 224 → L = 111.5 → 至少 112 层。

### 6. 计算量（FLOPs）

- **标准卷积**：K² × C_in × C_out × H' × W'
- **深度可分离卷积（MobileNet V1）**：深度卷积 K² × C_in × H' × W' + 逐点卷积 C_in × C_out × H' × W'
- 计算量比约为 1/C_out + 1/K²（通常为标准卷积的 1/8 ~ 1/9）

### 7. 池化层输出尺寸公式

$$H' = \frac{H - K}{S} + 1$$

池化层**没有可学习参数**。

### 8. 1×1 卷积

对每个空间位置 (h,w) 的 C_in 维向量做线性变换 → 输出 C_out 维向量。

$$y_{h,w} = W_{1 \times 1} \cdot x_{h,w} + b$$

**Bottleneck 计算省参数量**（ResNet-50）：直接 3×3（256→256）：3² × 256² = 589,824。Bottleneck：1²×256×64 + 3²×64² + 1²×64×256 = 69,632（省 88%）。


## 二、mAP、IoU 等算子的计算方法

### 1. IoU（Intersection over Union，交并比）

$$IoU = \frac{|A \cap B|}{|A \cup B|} = \frac{交集面积}{A面积 + B面积 - 交集面积}$$

取值：0（完全不重叠）~ 1（完全重叠）

**Faster R-CNN 中 IoU 的阈值约定**：

| IoU 范围 | 样本类型 | 用途 |
|-----------|---------|------|
| IoU > 0.7 | 正样本（前景） | 参与分类和回归训练 |
| 0.3 < IoU < 0.7 | 忽略样本 | 不参与训练 |
| IoU < 0.3 | 负样本（背景） | 参与分类训练 |

### 2. Precision 和 Recall

$$Precision = \frac{TP}{TP + FP}$$

（所有检测为「物体」的框中，有多少是对的）

$$Recall = \frac{TP}{TP + FN}$$

（所有真正的物体中，有多少被检测出来了）

### 3. AP（Average Precision）/ mAP / mIoU

- **AP**：单类的 PR 曲线下面积。逐框按置信度降序排列 → 逐框计算 Precision/Recall → 积分。
- **mAP**：所有类别的 AP 取平均。mAP@0.5（IoU 阈值为 0.5）/ mAP@0.5:0.95（IoU 阈值从 0.5-0.95 取平均，COCO 标准）。
- **mIoU**：语义分割评估指标。所有类别的 IoU 取平均：mIoU = (1/C) Σ[TP_c/(TP_c+FP_c+FN_c)]（像素级别计算）。

### 4. NMS（Non-Maximum Suppression，非极大值抑制）

用于去除同一物体的重复检测框。将所有框按置信度降序排列 → 取最高分框 A → 删除所有与 A 的 IoU > 阈值的框 → 对剩余框重复 → 直到排序表中无框存活。


## 三、R-CNN 到 YOLO 的核心改进

### 整体脉络

```
R-CNN → Fast R-CNN → Faster R-CNN → YOLO / SSD → RetinaNet → DETR
两阶段   共享特征图   RPN替代         单阶段      Focal Loss   Transformer
                      Selective Search 实时检测    解决样本不平衡
```

### 1. R-CNN

**流程**：Selective Search 生成 ~2000 个 RoI → 每个 RoI 缩放至固定尺寸 → 每个 RoI 独立通过 CNN 提取特征 → SVM 分类 → 线性回归器精调框 (dx, dy, dw, dh)。

**问题**：每张图 2000 次 CNN 前向传播，大量 RoI 空间重叠造成重复计算；CNN、SVM、回归器分开训练，不是端到端。

### 2. Fast R-CNN

**核心改进**：
- **共享特征图**：整张图只走一次 CNN，所有 RoI 从特征图上取特征
- **RoI Pooling**：将任意尺寸 RoI 特征块均匀分成固定网格（如 7×7），每个格内 Max Pooling → 固定尺寸输出
- **端到端训练**：CNN + 分类头（Softmax）+ 回归头，联合训练
- 比 R-CNN 训练快约 9 倍，推理快约 200 倍

**遗留问题**：Selective Search 仍在 CPU 上运行（~2 秒/图）。

### 3. Faster R-CNN

**核心改进：引入 RPN（Region Proposal Network）替代 Selective Search**。

- RPN 和检测网络**共享同一个 CNN 骨干特征图**。RPN 直接在特征图上预测候选区域。
- **锚点框 Anchor Boxes**：在特征图每个空间位置预设 K 个锚点框（如 3 种尺度 × 3 种长宽比 = 9 个）。
- **RPN 的两个任务**：二分类——对每个锚点框判断「前景」还是「背景」；回归——对前景锚点框预测 4 个修正量 (dx, dy, dw, dh)。
- 推理时按前景得分取前 ~300 个作为 Proposal → 检测网络对这些 Proposal 做精细分类和回归。

**意义**：实现了**完全端到端、全 GPU 上运行**的目标检测。

### 4. YOLO（You Only Look Once）

**核心理念**：一次前向传播，直接输出所有检测结果。不再有「先提议再分类」的两步走。

**工作流程**：
- 特征图尺寸 S×S（如 7×7）。物体中心落在哪个空间位置 → 由那个位置的特征向量负责预测。
- 每个空间位置输出 B 个边界框（每个框 5 个值：x, y, w, h, confidence）+ C 个类别概率。
- x, y = 相对于当前空间位置的偏移量；w, h = 整个框的宽高（用原图比例表示）。
- 总输出维度：S × S × (B×5 + C)。

**后处理**：置信度筛选（低分框丢弃）→ NMS（去重）。

**优势**：速度快，适合实时应用。**局限**：对小物体和密集物体的效果不如两阶段方法。

### 5. YOLO vs R-CNN 对比

| 维度 | R-CNN 系列 | YOLO |
|------|-----------|------|
| 检测方式 | 先提议 → 后分类（两阶段） | 一次前向全出（单阶段） |
| 候选区域 | 外部算法或 RPN 网络生成 | 不需要单独的提议阶段 |
| 速度 | 相对慢 | 快，实时 |
| 输出结构 | 可变数量 | 固定维度 S×S×(B×5+C) |
| 小物体检测 | 较好 | 相对弱 |

### 6. Fast R-CNN → Faster R-CNN → YOLO 三步进化

```
Fast R-CNN（共享卷积，但区域提议靠外部 Selective Search）
    ↓ 把区域提议也纳入网络
Faster R-CNN（RPN 替代 Selective Search，全 GPU 端到端）
    ↓ 把「提议 + 分类」两步合成一步
YOLO（直接在特征图上密集预测，不需要显式的提议阶段）
```


## 四、ResNet 架构

### 1. 退化问题

直接堆更深层 → **训练误差反而上升**（不是过拟合）。理论上，56 层网络的后面 36 层只要学恒等映射 H(x)=x，就至少不比 20 层差。但实际上**非线性层很难学会恒等映射**。

### 2. 残差连接：核心公式

$$y = F(x) + x$$

- 不直接学目标映射 H(x)，而是学残差 F(x) = H(x) - x
- 如果最优映射就是恒等映射，F(x) 只需趋近于 0——推权重趋向 0 比学精确恒等映射容易得多

### 3. 梯度流动

$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \cdot \left(1 + \frac{\partial F}{\partial x}\right)$$

关键在那个 +1。即使 F(x) 内部的梯度衰减为 0，梯度仍能通过跳跃连接无损传回。**这是 ResNet 能训练 152 层甚至 1000+ 层的本质原因。**

### 4. 残差块结构

**Basic Block**（ResNet-18/34 用）：输入 → 3×3 Conv → BN → ReLU → 3×3 Conv → BN → + 输入 → ReLU

**Bottleneck Block**（ResNet-50/101/152 用）：输入 → 1×1 Conv（降维，如 256→64）→ BN → ReLU → 3×3 Conv（空间卷积）→ BN → ReLU → 1×1 Conv（恢复维度）→ BN → + 输入 → ReLU

Bottleneck 参数量对比（输入输出均为 256 通道）：直接 3×3 = 3²×256² = 589,824；Bottleneck = 1²×256×64 + 3²×64² + 1²×64×256 = 69,632（省 88%）。

### 5. ResNet 关键数字

| 模型 | 层数 | 残差块类型 | 参数量（约） |
|------|:---:|-----------|:----------:|
| ResNet-18 | 18 | Basic | 11.7M |
| ResNet-34 | 34 | Basic | 21.8M |
| ResNet-50 | 50 | Bottleneck | 25.6M |
| ResNet-101 | 101 | Bottleneck | 44.5M |
| ResNet-152 | 152 | Bottleneck | 60.2M |

**里程碑成就**：ResNet-152 ImageNet Top-5 错误率 3.57%，**首次超越人类水平（约 5.1%）**。

### 6. 维度不匹配的解决方案

当 stride=2 或通道数改变时，F(x) 和 x 的形状不一致，不能直接相加。

- **Projection Shortcut（最常用）**：y = F(x) + W_s·x，在跳跃连接上用一个 1×1 卷积投影到正确的维度和尺寸。
- **Zero Padding**：对缺失的维度填零（更简单但效果稍差）。


## 五、上采样技术

### 为什么语义分割需要上采样

编码器通过反复下采样压缩空间尺寸以提取语义信息，特征图变小。但语义分割要求**输出和输入同样大小的像素级预测图**。需要把特征图从小放大回去。

### 五种上采样方法

| 方法 | 原理 | 可学习 | 特点 |
|------|------|:---:|------|
| **最近邻插值** | 目标像素值 = 最近源像素值 | 否 | 极快但粗糙，有锯齿 |
| **双线性插值** | 目标像素周围 4 个源像素按距离加权平均 | 否 | 平滑，计算量适中 |
| **Max Unpooling** | 用 Pooling 时记录的最大值位置将值放回原位，其余填 0 | 否 | 保留空间结构，但非最大值信息丢失 |
| **转置卷积** | 每个输入像素通过可学习的 K×K 核「扩散」到输出区域 | 是 | 可学习，⚠️ 需注意棋盘效应（使 K 能被 S 整除） |
| **反池化/0-填充** | 每个值复制到扩大后的位置，其余填 0 | 否 | 简单直接，精度较低 |

### 各模型中的上采样方式

| 模型 | 上采样方式 |
|------|-----------|
| **FCN** | 转置卷积 |
| **U-Net** | 转置卷积或双线性 + 卷积 |
| **DeepLab v1/v2** | 双线性插值 |
| **DeepLab v3+** | 双线性插值 + 解码器特征融合 |
| **SegFormer** | MLP + 双线性插值 |
| **SAM** | 转置卷积 |

## 六、三种 Transformer 架构

### 1. 总览

| 架构 | 选用部分 | 注意力掩码 | 代表模型 | 擅长 |
|------|---------|-----------|---------|------|
| Encoder-only | 只用 Encoder | **完全注意力**（双向） | BERT、RoBERTa | 理解任务 |
| Decoder-only | 只用 Decoder | **下三角因果掩码**（单向） | GPT、LLaMA | 生成任务 |
| Encoder-Decoder | Encoder + Decoder | **完全 + 下三角 + 交叉注意力** | T5、BART | 翻译/摘要等输入→输出映射 |

### 2. Encoder-only 架构

**结构**：输入编码（Token 嵌入 + 位置编码）→ 特征编码（多个 Encoder Block，每块含双向自注意力 + FFN）→ 任务处理（接分类头）。

**注意力特点**：双向完全注意力——每个 Token 可看到序列中所有其他 Token。

**代表模型——BERT**：
- BERT-Base：12 层，768 维，12 头，1.1 亿参数
- BERT-Large：24 层，1024 维，16 头，3.4 亿参数
- 预训练数据：BookCorpus + 英语维基百科 ≈ 33 亿 Token / ~15GB
- 预训练任务：**MLM**（掩码语言建模——遮 15% Token 做完形填空）+ **NSP**（下文预测——两句是否相邻）

**代表模型——RoBERTa**：架构完全同 BERT。改进：数据 15GB→160GB；训练步数 100K→500K；**动态掩码**（每次加载数据时重新遮）；移除 NSP。核心结论：训练充分性 > 架构创新。

**代表模型——ALBERT**：参数因子分解（嵌入矩阵 V×H → V×E + E×H，嵌入参数降 87%）+ 跨层参数共享（所有 Block 共享参数）。NSP 替换为 SOP（句序预测）。

**优势**：双向注意力，理解任务出色。**局限**：没有解码器无法生成；参数最大仅 ~3.5 亿。

### 3. Decoder-only 架构

**结构**：去掉 Encoder，去掉交叉注意力，只保留**因果自注意力 + 前馈网络**。

**注意力特点**：下三角因果掩码——每个 Token 只能看自己及前面，看不到未来的 Token。数学上通过下三角掩码矩阵将未来位置注意力分数设为 -∞（softmax 后权重为 0）。

**代表模型——GPT 系列五个阶段**：

| 阶段 | 模型 | 年份 | 最大参数 | 数据量 | 关键突破 |
|------|------|------|---------|--------|---------|
| 初出茅庐 | GPT-1 | 2018 | 1.17 亿 | ~5GB | 证明单向自回归预训练可行 |
| 小有所成 | GPT-2 | 2019 | 15 亿 | 40GB | 零样本任务能力 |
| 崭露头角 | GPT-3 | 2020 | 1750 亿 | 1TB | **涌现上下文学习 ICL** |
| 一鸣惊人 | ChatGPT | 2022 | 未公开 | 未公开 | RLHF + LLMaaS 服务模式 |
| 大放异彩 | GPT-4/4o | 2023-24 | 未公开 | 未公开 | 图文双模态 / 降低延迟 |

**GPT-3 的涌现能力——上下文学习（In-Context Learning, ICL）**：
- **Zero-shot**：只给任务描述，不给示例，模型直接输出答案。**Few-shot**：给任务描述 + 几个示例。
- 不需要梯度更新、不需要微调——纯粹的前向传播即可完成新任务。
- 这是「涌现能力」的典型代表——参数增加到一定规模后自然出现，不是编写代码实现的。

**代表模型——LLaMA 系列**：

| 模型 | 年份 | 最大参数 | 数据量 | 关键改进 |
|------|------|---------|--------|---------|
| LLaMA-1 | 2023 | 652 亿 | ~5TB | RoPE / Pre-Norm / SwiGLU |
| LLaMA-2 | 2023 | 700 亿 | ~7TB | +RLHF + GQA |
| LLaMA-3/3.1 | 2024 | 4050 亿 | ~50TB | 数据量 7 倍于上代 |

**LLaMA 的三项架构创新**：
- **RoPE（旋转位置编码）**：用旋转矩阵注入位置信息，注意力分数天然取决于 Token 间相对位置差异
- **Pre-Norm**：在每子层输入之**前**做层正则化（非传统输出之后），深层训练更稳定
- **SwiGLU**：替换 ReLU，结合 Swish 平滑非线性和 GLU 门控机制
- **GQA（LLaMA-2 引入）**：多个 Query 头共享同一组 Key/Value 头，减少 KV 缓存

**Decoder-only 成为主流的原因**：省掉 Encoder → 约一半参数集中用在生成侧；架构极简 → 分布式扩展更易；自回归生成天然适合开放域对话等主流应用。

### 4. Encoder-Decoder 架构

**结构**：完整 Transformer。五部分——编码输入嵌入 → 编码特征编码 → 编码器输出 → 解码输入嵌入 → 解码处理（因果自注意力 + **交叉注意力** + FFN）。

**三种注意力共存**：编码器自注意力（完全双向）、解码器自注意力（因果单向）、**交叉注意力**（Q 来自解码器，K/V 来自编码器全输出，生成每个 Token 时都能查阅完整源文语义）。

**代表模型——T5**：所有 NLP 任务统一为 Text→Text 格式（如「翻译成英文：今天天气怎么样？」→ 输出英文）。Span Corruption 掩码训练（连续段掩码，非单 Token）。C4 数据集 750GB。最大 110 亿参数。

**代表模型——BART**：去噪自编码器——输入破坏文本 → Encoder → Decoder 复原。五项破坏任务：Token 遮挡、删除、句子打乱、文档旋转、文本填充。


## 七、幻觉（Hallucination）

### 定义

大语言模型生成的内容看似合理，但实际上与事实相悖、逻辑混乱或包含不实信息。

### 七大来源

**训练数据侧（3 个）**：

1. **知识过时**：训练截止后世界的变化模型不知道。回答时可能「自信地编造」过时信息。
2. **知识边界**：训练数据不可能覆盖全部知识。低频专业信息不在训练数据中或出现极少。
3. **知识偏差**：互联网文本含大量偏见，模型不加辨别地学到并复现（如「企业家」默认用男性代词）。

**模型本身侧（4 个）**：

4. **知识长尾**：低频知识曝光极少，模型记忆不牢，容易混淆为高频相似实体。
5. **曝光偏差（Exposure Bias）**：Teacher Forcing 训练时喂正确上文，推理时喂自己生成的带误差上文，模型偏离训练分布。
6. **解码偏差**：采样的随机性可能偶然输出低概率不合理 Token，一步偏差被自回归逐级放大。
7. **对齐不当**：RLHF 标注数据偏差让模型学到「让人类喜欢」而非「说得正确」。

### 缓解方案——RAG

核心思想：**让模型回答前先「查资料」**，用外部知识库的最新权威事实约束生成。


## 八、RAG 架构

### RAG 的基本组成

RAG = 知识检索（Retrieval）+ 生成增强（Generation）

1. **知识检索模块**：对问题编码 → 从外部知识库检索相关文档 → 返回 Top-K 文档
2. **生成增强模块**：检索文档 + 原始问题 → 拼接成 Prompt → LLM 生成最终答案

### RAG 的四大架构

| 架构 | LLM参数 | 检索器 | 代表方法 | 优点 | 缺点 |
|------|:--:|:--:|------|------|------|
| 黑盒-无微调 | ✗ | ✗ | In-Context RALM | 解耦LLM，易实现，成本最低 | 缺乏交互，效果难保证 |
| 黑盒-检索器微调 | ✗ | ✓ | REPLUG | 用 PPL 反馈迎合LLM，效果好 | LLM固定，可能不适配 |
| 白盒-仅微调LLM | ✓ | ✗ | RETRO | 交叉注意力深融外部信息 | 成本高，依赖检索器性能 |
| 白盒-协同微调 | ✓ | ✓ | ATLAS | KL散度同步优化，性能最强 | 成本最高，工程最复杂 |


## 九、何时增强

### 核心原则：非必要不增强

如果 LLM 内部知识足以正确回答，强制 RAG 反而画蛇添足。

### 外部观测法

不侵入模型内部，通过外部手段判断：
- **Prompt 直问**：直接询问模型是否具备某知识（局限——模型易过度自信）
- **观察训练数据/伪数据流行度**：设定维基百科/谷歌实体访问量阈值来估计模型是否已知。低频 = 可能不知道 → 触发 RAG
- **构造伪训练数据统计量**：流行度 = 该实体在特定环境被广泛关注的频率。越「流行」知识模型掌握越好

### 内部观测法

分析 LLM 内部状态变化——处理「已知问题」和「未知问题」时，模型中间层呈现**不同的激活动态模式**。基于此训练一个**线性探针（Linear Probe）**——一个线性分类器，根据问题对应的内部表示预测该问题是否属于模型已知。


## 十、何处增强

| 位置 | 方法 | 优点 | 缺点 |
|------|------|------|------|
| **输入端（最主流）** | 检索文档拼接在 Prompt 中 | 直观易实现，无需改结构，适用黑盒 API | 长文本增加推理成本 |
| **中间层** | 交叉注意力将检索向量融入隐层（RETRO） | 深度影响模型内部，少受输入长度限制 | 需复杂结构设计，无法用于黑盒 |
| **输出端** | 先生成后校正（后验校准） | 确保一致，提高准确可靠性 | 依赖检索知识质量和相关性 |


## 十一、降本增效

### 去除冗余文本

对检索出的原文文段进行过滤。三种级别：
- **Token 级别**：去除低困惑度 Token
- **全文本级别**：从各文档中提取核心句
- **子文本级别**：对文档分块后打分，选出高分子文本组合

**代表方法——FIT-RAG**：将文档划为子文档 → LLM 偏好打分器对每个子文本打分 → 筛选出最有益的子文本组合 → **平均压缩约 50% 输入 Token，回答准确率优于全量基线**。

### 复用计算结果

**KV-cache 机制**：将生成过程中产生的 Key 和 Value 向量缓存，下次用到相同前文时直接从缓存读取。**RAGCache**：RAG 专用的多级动态缓存机制（KV 张量缓存库 + 缓存检索器 + RAG 控制器）。

### 组合使用

Prompt 压缩 + KV-cache 结合，在保持高性能的同时显著降低 Token 开销和计算延迟。

## 十二、SimCLR 的网络结构和输入

### SimCLR 的整体框架

SimCLR（ICML 2020）是自监督对比学习的里程碑——**不需要任何人工标签**，通过数据增强自动构建正负样本对来学习图像表征。

### 输入与数据增强

**输入构造方式**：
1. 取一个 batch 的 N 张原始图像（无标签）
2. 对每张图像施加**两种不同**的随机数据增强 → 得到 2N 张增强后的图像
3. 同一张原始图像的两个增强版本互为正样本对（共 N 对）
4. batch 内其他 2N-2 张图都是负样本

**关键发现**：数据增强组合对自监督学习质量起**决定性作用**。单纯随机裁剪不够——模型可通过颜色分布匹配「作弊」。**必须结合颜色抖动**（随机改变亮度、对比度、饱和度、色相）——迫使模型学习语义特征而非表面色彩统计。典型组合：随机裁剪 + 随机水平翻转 + 颜色抖动 + 高斯模糊。

### 网络结构（三个核心组件）

**1. 基础编码器 f（Base Encoder）**：
- 通常采用 ResNet（如 ResNet-50）
- 输入：增强后的图像（224×224×3）
- 输出：高维特征向量 h（如 2048 维）

**2. 投影头 g（Projector）**：
- 一个 2~3 层的 MLP（全连接网络）
- 输入：编码器输出的特征向量 h
- 输出：投影后的向量 z（如 128 维）
- **关键作用**：吸收和过滤掉对比任务有用但对通用识别没用的冗余信息（颜色、位置等），保护编码器学到更纯净的语义特征
- **训练后扔掉投影头**——只用编码器 f 的输出作为下游任务的表征

**3. 对比损失函数（NT-Xent Loss）**：

对于 batch 中一对正样本 (i, j)：

$$l_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k=1}^{2N} \mathbb{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k) / \tau)}$$

- z_i, z_j 是投影头后的特征向量（先做 L2 归一化）
- sim(z_i, z_j) = 余弦相似度
- τ 是温度参数（控制分布锐度，越小越严格）
- 分母对 batch 内所有 2N 个样本（除了 i 自己）求和

**直观理解**：分子（正样本对相似度）越大、分母（所有负样本相似度之和）越小 → 损失越小。最终总损失是 batch 内所有正样本对损失的对称和。

**下游任务**：训练后**扔掉投影头 g**，只用编码器 f 的输出。**仅用 1% 的 ImageNet 标签微调，就能超越全监督 AlexNet 的性能。**


## 十三、Skip-Gram 变体的计算公式

### Word2Vec 简介

Word2Vec（Mikolov et al., 2013）两种变体：
- **Skip-Gram**：给定中心词 c，预测上下文词 o——P(o|c)
- **CBOW**：给定上下文词，预测中心词——PP 以 Skip-Gram 为例教学。

### Skip-Gram 的目标函数

优化目标：最大化语料库中所有中心词-上下文词对的对数概率。给定长度为 T 的序列，窗口大小 m：

$$J(\theta) = \frac{1}{T} \sum_{t=1}^{T} \sum_{-m \leq j \leq m, j \neq 0} \log P(w_{t+j} | w_t; \theta)$$

损失 = -J(θ)（最小化负对数似然）。

### Skip-Gram 中 P(o|c) 的计算——Softmax 公式

每个单词 w 用**两套向量**表示（增加灵活性）：
- **中心词向量 v_w**：当 w 充当中心词时使用
- **上下文向量 u_w**：当 w 充当上下文词时使用

$$P(o|c) = \frac{\exp(u_o^T v_c)}{\sum_{w \in V} \exp(u_w^T v_c)}$$

- 分子：中心词向量和上下文词向量的内积（相似度）
- 分母：中心词向量和词表中**所有**单词的上下文向量内积之和（归一化）
- V 是词表（如五十万词）

**含义**：在所有词中，o 与 c 的相似度占比越大 → P(o|c) 越大 → c 出现后 o 出现的概率更高。

### Skip-Gram 的优化——负采样（Negative Sampling）

**问题**：Softmax 分母需要对词表中所有单词（几十万+）计算内积，极其昂贵。

**负采样解法**：
- 不计算所有词，只随机采样 K 个「负样本」（K 通常 5~20）
- 将多分类转化为多个二分类：真实上下文词→正样本，随机采词→负样本

**负采样的目标函数**：

$$J_{neg} = \log \sigma(u_o^T v_c) + \sum_{k=1}^{K} \mathbb{E}_{w_k \sim P_n(w)} [\log \sigma(-u_{w_k}^T v_c)]$$

- σ 是 sigmoid 函数
- 第一项：c 和真实上下文词 o 的内积，经 sigmoid 后应接近 1（正样本得分高）
- 第二项：c 和随机负样本 w_k 的内积取负，经 sigmoid 后应接近 0
- P_n(w) 是负采样分布，正比于词频的 0.75 次方：P_n(w) ∝ count(w)^{0.75}

**为什么用 0.75 次方**：降低高频词（「the」「a」）被采样的概率，提高中低频词被采样的概率。高频词被采样太多会主导训练，但它们本身无语义信息。

## 十四、变分自编码器（VAE）的数学原理

### VAE 的优化目标——最大似然估计

**目标**：找到最优模型参数 θ，使真实观测数据在模型分布 P_θ 中出现概率最大：

$$\theta^* = \arg\max_{\theta} \sum_{x} \log P_{\theta}(x)$$

潜变量建模：P_θ(x) = ∫ P(z) · P_θ(x|z) dz
- P(z) = 先验分布，取标准正态 N(0, I)
- P_θ(x|z) = Decoder，由神经网络参数 θ 控制

**困难**：积分需要对 z 在整个 128 维隐空间中积分——数值上不可行。

### VAE 的变分推断——引入编码器

VAE 引入**编码器网络 Q_φ(z|x)** 来近似真实的隐变量后验分布 P(z|x)（这就是「变分」的由来）：

$$Q_{\phi}(z|x) = \mathcal{N}(\mu_{\phi}(x), \sigma^2_{\phi}(x))$$

### VAE 的最终损失函数——ELBO

$$\mathcal{L}(\theta, \phi; x) = \mathbb{E}_{z \sim Q_{\phi}(z|x)}[\log P_{\theta}(x|z)] - D_{KL}(Q_{\phi}(z|x) \| P(z))$$

等价地，最小化：

$$\mathcal{L}_{VAE} = \mathcal{L}_{recon} + \mathcal{L}_{KL}$$

### 损失函数的两大部分

**第一部分——重建损失**：

$$\mathcal{L}_{recon} = \| x - \hat{x} \|^2$$

从编码器分布 Q_φ(z|x) 中采样 z，然后 Decoder 重建 x̂。P_θ 取高斯分布时等价于 MSE。约束 Decoder 能从 z 还原出 x。

**第二部分——KL 散度正则项**（有**解析解**）：

当先验 P(z) = N(0, I) 且 Q_φ(z|x) = N(μ, σ²) 时：

$$D_{KL}(\mathcal{N}(\mu, \sigma^2) \| \mathcal{N}(0, 1)) = \frac{1}{2} \sum_{d=1}^{D} \left( \mu_d^2 + \sigma_d^2 - \log \sigma_d^2 - 1 \right)$$

约束：Encoder 输出的分布不要偏离标准正态先验 N(0,I) 太远。

**总损失**：L = L_recon + λ · L_KL，λ 是平衡两方的权重系数。


## 十五、正则项损失——KL 散度的关键作用

### 为什么 KL 散度不能被去掉

**去掉 KL 散度只保留 MSE**：神经网络为最小化 MSE，会通过让方差 σ² → 0 来消除采样不确定性。方差压到 0 → 所有图像编码退化成一个确定点（σ=0）→ VAE 塌缩成传统自编码器——隐空间重新破碎，丧失生成新图像能力。

**KL 散度的作用**：
1. 强制 Encoder 输出分布向 N(0, I) 靠近
2. 防止分布收缩为点（σ → 0），维持宽度
3. 不同图像编码分布之间有重叠——填补隐空间「断层」
4. 方便生成时从 N(0, I) 采样——绝大多数采样点都落在某个训练图像编码覆盖范围

### VAE 训练的博弈

- **重建损失过强** → 隐空间离散、丧失生成多样性（分布变点）
- **KL 散度过强** → 「后验崩溃」Posterior Collapse：Encoder 对所有输入都输出 N(0, I)（KL 散度为 0），Decoder 收到的 z 和输入完全无关，生成图像是所有训练图像的平均——极其模糊
- **VAE 训练的本质 = 在重建精度和生成多样性之间找最优平衡**


## 十六、训练困境与重参数化技巧

### 训练困境——采样操作不可微

VAE 流程：Encoder 输出 μ 和 σ → 从 N(μ, σ²) 中随机采样 z → Decoder 重建。

**问题**：从高斯分布 N(μ, σ²) 中随机采样 z 是**不可微操作**。反向传播的梯度流在「采样」这一步彻底断裂。

### 解决方案——重参数化技巧（Reparameterization Trick）

**核心公式**：

$$z = \mu + \sigma \cdot \varepsilon, \quad \varepsilon \sim \mathcal{N}(0, 1)$$

- ε 从外部随机采样（N(0,1)），不被梯度追踪
- μ 和 σ 是 Encoder 输出的确定性参数
- z 对 μ 和 σ 的梯度：∂z/∂μ = 1，∂z/∂σ = ε

**梯度链畅通**：MSE → Decoder → z = μ + σ·ε → μ, σ（梯度无障碍传递）。随机性局限在外部 ε 中，可学习部分通过线性变换与 z 连接。**没有这个公式，VAE 无法端到端训练。** 这也是 VAE 从理论走向工程落地的关键数学技巧。


## 十七、去噪模型的设计（扩散模型）

### 背景

扩散模型核心是学习「去噪」——每一步去掉含噪图像中的噪声以还原清晰图像。

### 思路一：直接预测加噪前的图像

- 输入：含噪图 x_t + 时间步 t
- 输出：去噪后的清晰图像 x_{t-1}
- **问题**：图像分布极其复杂——猫、狗、人脸、风景多样性大。一个模型要预测任意类别图像在任意噪音级别下去噪后形态——学习难度极高

### 思路二（正确路径）：预测噪声而非预测图像

- 输入：含噪图 x_t + 时间步 t
- 输出：预测此步加入的噪声 ε_θ(x_t, t)
- **为什么更优**：无论原始图像是什么，加噪过程**始终是同一类高斯噪声**——独立高斯样本，服从固定简单分布。预测「哪部分是噪声」远比预测「这是什么物体」简单
- 模型不需要理解图像内容——只需要学会区分「哪里是噪点」


## 十八、扩散模型的训练流程

### 核心训练公式——单步加噪

利用高斯噪声的可加性，从原图 x_0 直接一步合成任意第 t 步的含噪图：

$$x_t = \sqrt{\bar{\alpha}_t} \cdot x_0 + \sqrt{1 - \bar{\alpha}_t} \cdot \varepsilon, \quad \varepsilon \sim \mathcal{N}(0, I)$$

- ᾱ_t 是噪声强度参数（随 t 增加从 1 到 0），控制原图信息 vs 噪声信息比例
- 单步加噪的推导：多步独立高斯噪声叠加 = 等价于一个等效合并噪声高斯——任意时间步可一步直接计算

### 训练的完整流程

1. **随机采样**：取一张真实图像 x_0；随机取时间步 t ~ Uniform(1, T)；随机生成噪声 ε ~ N(0, I)
2. **一步合成含噪图**：用公式计算 x_t = √(ᾱ_t)·x_0 + √(1-ᾱ_t)·ε
3. **噪声预测**：将 x_t 和 t 送入噪声预测网络 ε_θ，预测真实噪声 ε
4. **计算损失**：L = || ε_θ(x_t, t) - ε ||²（噪声预测值与真值的 MSE）
5. **反向传播更新参数**

### 训练策略——随机采样时间步

**为什么不能按顺序一步步训练**：若按顺序 1→2→3→...→T 逐步训练——前期只学低噪、后期只学高噪——模型出现灾难性遗忘（学会了后面的忘记前面的）。深度学习训练需要样本随机打乱。

**解决方案——随机采样时间步**：每个迭代随机抽取一个 t，马上用单步公式生成 x_t 送进训练。模型在每个 batch 能同时学到从轻噪到重噪各种条件——全局稳定无遗忘。


## 十九、扩散模型的推理（生成）流程

### 推理两步走

**第一步——给定时间 t 及该时刻含噪图 x_t，预测噪声并减去**：
- 输入：时间步 t、含噪图 x_t（或隐变量状态）
- 输出：预测当前噪声 ε̂ = ε_θ(x_t, t)
- 利用预测噪声去除该步混入的噪音分量 → 得近似 x_{t-1}

**第二步——给去噪后结果加入少量随机噪声**：
- 生成 x_{t-1} 的公式需从 N(0, I) 采样一小份随机噪声加到降噪输出上
- **作用**：增加生成多样性（每次结果不完全相同）；打破局部预测误差累积链

### 完整 T 步迭代

从纯噪声 x_T ~ N(0, I) 开始 → for t = T → 1：预测噪声 → 利用预测噪声和随机噪声计算 x_{t-1} → 最终 x_0 = 清晰图像。

T 步数的选择（如 50~1000）代表细度 vs 速度 trade-off——多步更细更准但慢；少步更快但可能残留噪影。

## 附录：核心公式速查表

### VAE 核心公式

| 公式 | 含义 |
|------|------|
| z = μ + σ·ε | 重参数化：随机性转移给外部噪声 ε |
| L = MSE(x, x̂) + KL(N(μ,σ²) ‖ N(0,I)) | VAE 总损失 = 重建 + KL 正则 |
| KL = ½ Σ(μ² + σ² - log σ² - 1) | KL 散度解析解 |
| P(x) = ∫ P(z) P(x\|z) dz | 边际似然（潜变量积分） |

### 扩散模型核心公式

| 公式 | 含义 |
|------|------|
| x_t = √(ᾱ_t)·x_0 + √(1-ᾱ_t)·ε | 单步加噪合成含噪图 |
| L = ‖ε_θ(x_t, t) - ε‖² | 噪声预测损失（MSE） |

### SimCLR / Skip-Gram 核心公式

| 公式 | 含义 |
|------|------|
| l_{i,j} = -log(exp(sim(z_i,z_j)/τ) / Σ_k exp(sim(z_i,z_k)/τ)) | SimCLR NT-Xent 对比损失 |
| P(o\|c) = exp(u_o^T v_c) / Σ_w exp(u_w^T v_c) | Skip-Gram Softmax 条件概率 |
| J_neg = log σ(u_o^T v_c) + Σ_k log σ(-u_k^T v_c) | Skip-Gram 负采样目标 |

### ResNet / 卷积核心公式

| 公式 | 含义 |
|------|------|

| H' = (H - K + 2P) / S + 1 | 卷积输出尺寸 |
| R = 1 + L × (K - 1) | 感受野堆叠 |
