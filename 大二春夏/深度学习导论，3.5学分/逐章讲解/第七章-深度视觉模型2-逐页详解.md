# 第一板块：目标检测


## 第 1 页 · 封面页

PPT 内容：
- 课程名称：深度学习导论
- 第七章：深度视觉模型（二）
- 主讲教师：王文冠

### 讲解

上一章（第六章）讲的是「深度视觉模型（一）」，核心是图像分类——给你一张图，你判断它是猫还是狗。这一章进入更复杂的视觉任务。分类只需要回答一个「是什么」的问题；而检测要同时回答「是什么」和「在哪里」；分割要精细到每一个像素属于什么；视频理解还要加上时间维度。

这一章的内容量很大，但主线非常清晰：从分类出发，一步步增加输出信息的丰富程度——从一张图一个标签，到一张图多个框，到每个像素一个标签，再到每个实例一个掩码，最后到一段视频一个动作类别。


## 第 2 页 · 什么是目标检测

PPT 内容：
目标检测（Object Detection）：相比于只能判断整张图像类别的分类任务，目标检测更进一步——不仅能识别图像中存在哪些对象，还能用边界框精确定位每个对象的位置，实现分类与定位的结合。广泛应用于自动驾驶、异常行为检测、医疗影像分析等领域。

### 讲解

⭐ 这一页定义了本章第一个核心任务。

图像分类回答的是：「这张图里有猫吗？」→ 输出一个类别标签。

目标检测回答的是：「这张图里有什么？每个东西分别在哪里？」→ 输出多个物体的类别和位置。

位置用什么表示？用边界框 Bounding Box，通常用四个数字表示：

(x, y, w, h)

- (x, y)：边界框的中心坐标（或者左上角坐标，取决于约定）
- w：边界框的宽度
- h：边界框的高度

一张图可能输出多个边界框，每个框有自己的类别标签。比如一张街景照片可能需要输出：
- 行人 1：框 (120, 300, 80, 200)
- 行人 2：框 (400, 280, 75, 190)
- 汽车 1：框 (200, 350, 300, 150)
- 交通灯：框 (500, 100, 30, 60)

分类是「一句话描述整张图」，检测是「图上画框，每个框写标签」。


## 第 3 页 · 单目标检测

PPT 内容：
单目标检测是指图像中只存在或只需检测一个目标对象的情况。网络结构：共享的卷积特征提取器 + 两个并行的全连接头。分类头输出类别得分，使用 Softmax Loss。回归头输出边界框坐标 (x, y, w, h)，使用 L2 Loss。总损失 = 分类损失 + lambda * 定位损失。

### 讲解

这一页从最简单的检测任务讲起——假设图里只有一个物体，你只需要找出它在哪、是什么。

网络结构非常直观：

```
输入图像
  → CNN 特征提取器（如 AlexNet 的卷积部分）
  → 特征向量（如 4096 维）
  → 分成两个分支：
      ├── 分类头：FC(4096 → 1000) → Softmax → 类别概率
      └── 回归头：FC(4096 → 4) → 输出 (x, y, w, h)
```

两个头并行工作，共享同一个特征提取器。

损失函数是两个任务损失的加权求和。分类用 Softmax Loss（交叉熵），和第六章的分类任务完全一样。定位用 L2 Loss（均方误差）：

L_reg = (x_pred - x_true)^2 + (y_pred - y_true)^2 + (w_pred - w_true)^2 + (h_pred - h_true)^2

总损失 L = L_cls + lambda * L_reg

lambda 是一个超参数，用于平衡分类和定位两个任务的重要性。如果 lambda 太小，模型可能只学好了分类但框不准；如果太大，可能框很准但类别搞错了。

💡 这种「共享特征提取器 + 多个任务头」的设计模式在后面的 Fast R-CNN、Faster R-CNN、Mask R-CNN 中反复出现——提取特征是通用的，具体任务各有各的头。


## 第 4 页 · 多目标检测的核心挑战

PPT 内容：
多目标检测需要在同一图像中同时识别和定位多个目标，这些目标可能属于相同或不同类别，且常伴随重叠、尺度变化等复杂场景。核心挑战：每张图片需要不同数量的输出，而深度学习模型通常期望输入和输出具有固定的维度。

### 讲解

这是从单目标到多目标面临的根本性困难。

第六章的图像分类输出固定——永远是 1000 类的一个概率分布。单目标检测输出也固定——永远是 4 个坐标 + 1 个类别。

但多目标检测呢？一张图上可能有 1 个物体，也可能有 20 个物体。你不能让网络永远输出 1000 个框（太多），也不能只输出 5 个（不够）。

PPT 用图展示了这个困境：左上角有 4 个物体需要 4×4=16 个输出值（4 个框 × 4 个坐标），右下角密密麻麻很多物体需要很多很多个输出。

**深度学习模型的结构（层数、神经元数）是固定的，所以前向传播的输出维度也是固定的。但每张图需要检测的物体数量不固定。**

这就是后面所有 R-CNN、YOLO 系列要解决的核心问题。


## 第 5 页 · 滑动窗口的朴素思路及其致命缺陷

PPT 内容：
朴素思路：在图像上滑动不同尺寸和长宽比的窗口，对每个窗口应用一个 CNN 分类器，判断其是否包含物体及物体类别。
问题：需要将 CNN 应用于海量位置、尺度和宽高比，计算成本非常高！

### 讲解

最直观的多目标检测思路就是滑动窗口：用一个固定大小的窗口在图像上从左到右、从上到下滑动。每停一次，把窗口里的图像块裁下来送进 CNN 做分类——「这个窗口里有狗吗？有猫吗？还是什么都没有（背景）？」

然后换一个更大的窗口，再滑一遍。再换一个不同长宽比的窗口，再滑一遍。

假设一张 500×500 的图像：
- 滑动位置：约 250,000 个像素位置
- 窗口尺寸：至少 5 种（64×64, 128×128, 256×256, ...）
- 长宽比：至少 3 种（1:1, 2:1, 1:2）
- 每个位置每种组合跑一次 CNN 前向传播

总计至少 250,000 × 5 × 3 = 3,750,000 次 CNN 前向传播！这根本不可能在合理时间内完成。即使 GPU 再快，这个数量级也是不可接受的。

💡 但滑动窗口的直觉没有错——需要扫描全图。问题在于效率。后面所有的改进都在回答一个问题：怎么聪明地扫描，而不是暴力扫描？


## 第 6 页 · 区域提议

PPT 内容：
解决方案——区域提议 Region Proposals：先快速找出图像中「可能包含物体」的候选区域。只需对这几百到几千个候选区域进行精细分类和定位，大幅减少计算量。
经典方法 Selective Search：基于颜色、纹理、大小、形状等相似性，自底向上合并超像素，生成约 2000 个「blobby」区域。

### 讲解

⭐ 这一页引入了目标检测的核心思想：两阶段策略。

与其在整张图上无脑滑动窗口，不如先用一个快速算法找出「哪些地方看起来可能有物体」。这些候选区域叫 Region Proposals 或 RoI——Region of Interest 感兴趣区域。

Selective Search 是这个阶段最经典的方法，工作流程为：

1. 用 Felzenszwalb 算法把图像分割成大量超像素——小的、颜色纹理均匀的区域
2. 计算每个超像素和相邻超像素的相似度——基于颜色、纹理、大小、形状的综合评分
3. 把最相似的两个区域合并成一个更大的区域
4. 重复 2-3 步
5. 所有合并过程中产生的区域（从小到大、从细到粗）全部保留作为候选 RoI

最终每张图大约产生 **2000 个**候选区域。相比 375 万次滑动窗口，2000 个候选区域只需要 2000 次 CNN 前向传播——虽然还是很多，但已经从「完全不可行」变成「勉强可行」了。

Selective Search 的特点是「宁可多报，不能漏报」。它倾向于产生大量包含可能物体的区域，宁可把背景也圈进来，也不能放过真正的物体。


## 第 7-9 页 · R-CNN

PPT 内容（连续 3 页）：
R-CNN（Region-based CNN）：利用 Selective Search 生成候选区域，经 CNN 提取特征后分类并回归边界框，首次实现了高精度的多目标检测。

流程：1. Selective Search 生成约 2000 个 RoI → 2. 每个 RoI 缩放到固定尺寸 224×224 → 3. 每个 RoI 独立通过 CNN 提取特征 → 4. 训练 SVM 分类器 → 5. 训练线性回归器预测修正量 (dx, dy, dw, dh)。

效率瓶颈：每张图需要约 2000 次独立的 CNN 前向传播，计算高度冗余（大部分 RoI 空间上重叠）。

### 讲解

⭐ R-CNN 是目标检测历史上的里程碑——它第一次把 CNN 用于多目标检测并取得了显著突破。

R-CNN 的完整流程可以分为两大阶段：特征提取（第 7 页）和分类与精调（第 8 页）。

#### 阶段一：特征提取

Step 1：用 Selective Search 在一张图上生成大约 2000 个候选区域。这些区域大小不一、形状各异。

Step 2：把每个候选区域从原图上裁下来，缩放到固定尺寸 224×224。为什么是 224×224？因为当时最好的预训练 CNN（AlexNet）的输入尺寸就是 224×224。每个区域必须变成这个尺寸才能送进 CNN。

Step 3：每个 224×224 的区域独立地通过 CNN 提取特征。2000 个区域 = 2000 次 CNN 前向传播。

#### 阶段二：分类与精调

Step 4：对每个 RoI 的 CNN 特征向量（4096 维），训练一个线性 SVM 分类器。SVM 输出这个区域的类别——是背景、猫、狗、还是其他什么。

Step 5：对每个 RoI 的 CNN 特征向量，再训练一个线性回归器。回归器输出 4 个修正值 (dx, dy, dw, dh)。这四个值描述的是：**从当前的 RoI 框，需要平移和缩放多少才能精确框住物体**。

为什么需要修正？因为 Selective Search 给出的框只是「大概在这里」，不够精确。回归器的作用就是精调。

#### R-CNN 的效率瓶颈（第 9 页）

2000 个区域各自独立通过 CNN，这里面有巨大的计算冗余。看 PPT 的图：2000 个候选区域大多数在空间上是高度重叠的——它们的重叠区域被 CNN 反复计算了无数次。这就像把同一本书的同一页反复复印了 2000 次，而不是复印一次后把需要的那部分单独裁下来。

而且 R-CNN 的训练是多阶段的：先训练 CNN，再训练 SVM，再训练回归器——不是一个端到端的整体训练。


## 第 10-13 页 · Fast R-CNN

PPT 内容（连续 4 页）：
Fast R-CNN：先将整张图像通过 ConvNet 提取一个共享的稠密特征图，再将候选区域映射到该特征图上提取固定长度特征。

核心改进：
- 整图只过一次 CNN，得到一张共享特征图
- 将 Selective Search 生成的 RoI 映射到特征图上
- RoI Pooling：将任意尺寸的 RoI 特征块池化成固定尺寸
- 两个并行输出头：Softmax 分类器 + 边界框回归器
- 端到端训练

优势与遗留问题：速度大幅提升（共享卷积计算），端到端训练，精度高。但 Selective Search 仍然是一个外部、独立、耗时的步骤（在 CPU 上运行），成为整个系统的瓶颈。

### 讲解

⭐ Fast R-CNN 的核心改进可以用一句话概括：**整张图只跑一次 CNN，产生一张特征图，2000 个 RoI 都从这张特征图上取特征，而不是各自重新跑 CNN。**

#### 改进一：共享卷积计算

R-CNN：图 A → CNN → 特征 A；裁切区域 1 → CNN → 特征 1；裁切区域 2 → CNN → 特征 2......每个区域独立跑一次 CNN，2000 个区域 = 2000 次 CNN。

Fast R-CNN：图 A → CNN → 一张共享特征图。然后把 Selective Search 在**原图**上找到的 2000 个 RoI，按比例映射到**特征图**上。比如原图上一个 100×200 的区域，如果 CNN 下采样了 16 倍，映射到特征图上就是 6.25×12.5 的特征块。从这张特征图上「裁」下这 2000 个特征块，每个特征块代表原图中对应区域的高层语义特征。

#### 改进二：RoI Pooling 池化

映射过来的特征块尺寸各不相同——有的区域大，有的区域小。但全连接层要求固定尺寸的输入。怎么办？

RoI Pooling：把任意尺寸的 H×W 特征块，均匀划分成固定的网格（比如 7×7），每个网格内做 Max Pooling。无论输入多大，输出永远是 7×7。这就打通了从任意尺寸 RoI 到固定尺寸全连接层的桥梁。

#### 改进三：端到端训练 + 统一损失

R-CNN 里 CNN、SVM、回归器是分别训练的。Fast R-CNN 把它们统一成一个网络：
- CNN 骨干网络
- RoI Pooling
- 两个并行的全连接输出头：一个做 Softmax 分类，一个做边界框回归
- 总损失 = 分类损失 + 回归损失

所有参数一起用反向传播训练。分类损失和回归损失的梯度同时反馈给共享的 CNN，让 CNN 学到的特征对两个任务都有用。

#### 遗留问题（第 13 页）

Fast R-CNN 已经非常高效了——训练比 R-CNN 快约 9 倍，推理快约 200 倍。但有一个环节仍然是外部独立的：**Selective Search**。

Selective Search 在 CPU 上运行（不做 GPU 加速），处理一张图需要大约 2 秒。而 CNN 在 GPU 上只跑几十毫秒。所以整个系统的瓶颈是 CPU 上的区域提议，而不是 GPU 上的 CNN。


## 第 14-18 页 · Faster R-CNN

PPT 内容（连续 5 页）：
Faster R-CNN：引入区域提议网络 RPN，与检测网络共享底层卷积特征。RPN 取代了 Selective Search，实现完全在 GPU 上的端到端训练和推理。

RPN 锚点机制：在卷积特征图的每个空间位置上，预设 K 个不同尺度和长宽比的锚点框 Anchor Boxes。

RPN 二分类任务：对于每个锚点框，预测一个「物体性」Objectness 得分——该锚点框是前景还是背景。

RPN 边界框回归：对于预测为「前景」的锚点框，同时预测一个 4 维的修正向量 (dx, dy, dw, dh)，将原始锚点框精调到更接近真实物体的位置和尺寸。

训练与推理：将与真实框 IoU 较高的锚点视为正样本。推理时对所有锚点框根据 Objectness 得分排序，选取得分最高的约 300 个作为最终的区域提议。

### 讲解

⭐ Faster R-CNN 把目标检测的最后一个外部环节（区域提议）也纳入神经网络内部，实现了**完全端到端的目标检测**。

#### 整体架构

Faster R-CNN = RPN（区域提议网络） + Fast R-CNN（检测网络）。

两个网络共享同一个 CNN 骨干网络提取的特征图。RPN 在这个特征图上预测哪些位置可能有物体；检测网络在 RPN 选出的候选位置上做精细分类和回归。

#### 锚点框 Anchor Boxes（第 15 页）

这是 Faster R-CNN 最核心的机制。在 CNN 输出的特征图的每一个空间位置上，预设 K 个「锚点框」。这些锚点框有不同的大小和形状——比如 128×128、256×256、512×512 三种尺度，每种尺度有 1:1、2:1、1:2 三种长宽比，总共 K=9 种组合。

为什么要预设？因为物体的大小和形状各不相同——行人偏瘦高（1:2），汽车偏扁平（2:1），足球就是方的（1:1）。预设多种锚点框就是覆盖各种可能的形态。

一个锚点框不是真的「画在图上」，而是**数据集中的一个坐标参考**。RPN 的任务是判断每个锚点框是否包含物体，如果包含，应该怎么修正它让它更准。

#### RPN 的二分类任务（第 16 页）

对特征图上的每个空间位置，RPN 为每种锚点框输出一个二分类分数——「这个锚点框里有没有物体？」。不需要区分物体的具体类别，只需要区分前景（有物体）和背景（空）。

输出是一个 K×H×W 的概率图——特征图有 H×W 个位置，每个位置有 K 个锚点框，每个锚点框对应一个前景概率。

#### RPN 的回归任务（第 17 页）

对于被判定为「前景」的锚点框，RPN 再输出 4 个修正值 (dx, dy, dw, dh)：

- dx, dy：锚点框的中心需要平移多少
- dw, dh：锚点框的宽高需要缩放多少

为什么需要修正？因为锚点框是预设的固定参考框，物体的真实位置和形状不可能正好等于某个锚点框。RPN 学的是「从锚点框到真实框的修正量」。

#### 训练与推理（第 18 页）

训练时怎么定义正负样本？用 **IoU（交并比）**。

IoU = (预测框 ∩ 真实框) / (预测框 ∪ 真实框)

- IoU > 0.7 的锚点框 → 正样本（有物体）
- IoU < 0.3 的锚点框 → 负样本（背景）
- 0.3 < IoU < 0.7 的锚点框 → 忽略（太模糊，不参与训练）

推理时，对所有 K×H×W 个锚点框——数量可能是好几万——按前景得分排序，取前 300 个作为最终的候选区域 Proposal，送给后面的 Fast R-CNN 部分做精细分类和回归。


## 第 19 页 · 两阶段 vs 单阶段

PPT 内容：
两阶段 → 单阶段：能否摒弃区域提议阶段，直接从图像特征中预测物体类别和位置？

### 讲解

这一页引入了目标检测的路线之争。

两阶段检测器（R-CNN → Fast R-CNN → Faster R-CNN）的逻辑是：「先找候选区域，再精细分类」。第一步找出可能有物体的位置，第二步在这些位置上做类别判断和位置精修。优点是精度高，缺点是两步走，推理速度有上限。

那能不能不要第一步，直接从图像特征预测每一个位置的物体类别和边界框？这就引出了单阶段检测器：YOLO、SSD、RetinaNet。


## 第 20 页 · 单阶段目标检测器

PPT 内容：
单阶段目标检测器将检测问题转化为一个密集预测问题——在特征图的每个位置上，直接预测多个边界框及其类别。在每个网格单元内：对每个 base box 回归出最终边界框的 5 个参数 (dx, dy, dh, dw, confidence)，预测 C 个类别（含背景类）的得分。

### 讲解

单阶段检测器的思路很像 RPN，但它不仅判断「有没有物体」，还同时判断「是什么类别的物体」。

在特征图的每个空间位置上，网络直接输出：
1. 对于每个预设的 base box：5 个回归值 (dx, dy, dw, dh, confidence)。confidence 表示这个框内有物体的置信度。
2. C 个类别的得分（包括背景类）。

不再有「先提议再分类」的两步走，而是「一次性全部预测」。

这看起来很简单直接，但很快就遇到了一个严重问题——下一页会讲。


## 第 21-22 页 · YOLO

PPT 内容（连续 2 页）：
YOLO（You Only Look Once）：将输入图像划分为 S×S 的网格，每个网格直接预测边界框坐标、置信度及类别概率，仅需一次前向传播即可完成全图检测，真正实现了端到端的实时目标检测。

YOLO 后处理：先经置信度筛选剔除低分框，再通过非极大值抑制（NMS）去除重复检测，保留最优结果。

### 讲解

YOLO 的名字就说明了它的哲学——「你只需要看一次」。不像 R-CNN 系列需要先提议再分类，YOLO 把整张图一次性前向传播，直接输出所有检测结果。

#### YOLO 的工作流程

1. 把输入图像均匀分成 S×S 的网格（比如 7×7）
2. 每个网格负责预测落在该网格内的物体
3. 每个网格输出：B 个边界框（每个框 5 个值：x, y, w, h, confidence）+ C 个类别概率
4. 总输出维度：S × S × (B×5 + C)

#### 后处理

YOLO 的原始输出有很多冗余——多个相邻网格可能检测到同一个物体，输出多个高度重叠的框。需要两步后处理：

**置信度筛选**：把所有 confidence 低于阈值（比如 0.5）的框直接丢掉。

**非极大值抑制（NMS）**：对于同一类别的重叠框，保留得分最高的那个，抑制（删除）和它高度重叠的其他框。具体做法：
1. 按得分从高到低排序
2. 取最高分的框，计算它和其余框的 IoU
3. 删除所有 IoU > 阈值（0.5）的框
4. 对剩余的框重复 2-3
5. 最终只留下不重叠的高置信度框

YOLO 的核心优势是速度——一次前向传播出所有结果，非常适合实时应用。


## 第 23-24 页 · SSD 与多尺度特征融合

PPT 内容（连续 2 页）：
SSD：利用 CNN 不同层级的特征图来检测不同尺度的物体。浅层特征图（高分辨率）用于检测小物体；深层特征图（低分辨率）用于检测大物体。

YOLO 主要依赖最后的特征图，对小物体和密集物体检测效果不佳。SSD 通过多尺度特征融合，显著提升了对小物体的检测能力。

### 讲解

YOLO 的一个明显缺陷：它只在网络的最后一层做预测。经过多次池化，最后一层特征图的空间分辨率已经非常小（比如 7×7）。大物体还好（几个 grid 就能覆盖），小物体可能只占一个 grid 甚至不到一个 grid，特征图上几乎没有它的有效表示。

SSD 的解法：**在不同深度的特征图上分别做检测**。

浅层特征图分辨率高（比如 38×38），每个 grid 对应的原始图像区域小，适合检测小物体。深层特征图分辨率低（比如 1×1），每个 grid 对应的原始图像区域大，适合检测大物体。

SSD 在每个尺度的特征图上都接一个检测头，预测这个尺度上的边界框。最后把所有尺度的检测结果汇总在一起。

💡 多尺度特征融合这一思想，后来在语义分割的 U-Net、FPN（特征金字塔网络）中得到了更广泛的应用。


## 第 25-26 页 · 正负样本不平衡与 Focal Loss

PPT 内容（连续 2 页）：
单阶段检测器的挑战——正负样本不平衡：在密集预测中，绝大多数锚点/位置都是背景（负样本）。这种极端的不平衡（长尾分布）会导致模型训练困难，梯度被大量简单的负样本主导，是阻碍单阶段检测器实现最先进精度的主要障碍。

Focal Loss 的改进：引入调制因子 (1 - p)^gamma，降低易分样本（p 接近 1）的权重。使模型在训练过程中更加关注难分样本（p 接近 0）。有效解决了正负样本不平衡问题，使单阶段检测器的精度能够媲美甚至超越两阶段检测器。

### 讲解

⭐ 这是本章最重要的概念之一，解释了为什么单阶段检测器精度曾长期落后于两阶段检测器。

#### 正负样本有多不平衡？

一张 1000×600 的图片上，Faster R-CNN 可能产生约 20,000 个锚点框。其中真正包含物体的（正样本）通常只有 5-20 个，剩下的 19,980+ 个全是背景（负样本）。

正负比 = 5 : 19,995 ≈ 1 : 4000。

这些负样本大多「很容易」——模型只看了一眼就能很确信地说「这是背景」，概率 p 接近 0（预测为背景的概率接近 1）。但数量太多了，累积起来的损失还是很大，而且梯度信号被这些简单样本淹没了，模型几乎没精力去关注那 5 个真正重要的正样本。

#### 标准交叉熵的问题

标准交叉熵 Loss = -log(p_t)，其中 p_t 是模型对正确类别的预测概率。

对于一个「容易」的负样本（模型已经非常确定是背景，p_t ≈ 0.99），损失 ≈ -log(0.99) ≈ 0.01，很小。但如果有 19995 个这样的样本，总损失 ≈ 19995 × 0.01 ≈ 200。虽然每个都很小但数量大，模型被这些无关紧要的样本牵着走。

#### Focal Loss 的解决方案

Focal Loss = -(1 - p_t)^gamma × log(p_t)

新增的调制因子 (1 - p_t)^gamma 的核心作用：
- 对于易分样本（p_t ≈ 0.99）：(1 - 0.99)^gamma ≈ 0.01^gamma，非常小 → 损失被大幅压制
- 对于难分样本（p_t ≈ 0.3）：(1 - 0.3)^gamma ≈ 0.7^gamma，相对大 → 损失几乎不变

这样，模型不再被海量简单负样本牵着走，而是集中精力攻克那些「模棱两可」「边界模糊」的难样本。

gamma 通常取 2。gamma=0 时 Focal Loss 退化为标准交叉熵；gamma 越大，对易分样本的压制越强。

💡 Focal Loss 是 RetinaNet（单阶段检测器的代表作）能够在精度上超越两阶段 Faster R-CNN 的核心原因。它解决了一个根本性的数据不平衡问题，不是靠改网络结构，而是靠改损失函数——这体现了「损失函数的设计也是深度学习的重要创新维度」。


## 第 27 页 · DETR

PPT 内容：
DETR：核心架构 CNN 主干 + Transformer 编码器-解码器。简单的目标检测流程：直接从 Transformer 输出一组边界框，无需锚点框和回归边界框变换。使用二分匹配（匈牙利算法）将预测边界框与 GT 边界框进行匹配。

### 讲解

DETR（DEtection TRansformer）是 Facebook 在 2020 年提出的工作，把 Transformer 引入了目标检测。

传统方法（Faster R-CNN、YOLO）都依赖锚点框——预设一堆参考框，再学习修正量。DETR 说：不需要锚点框。直接用 Transformer 的 Decoder 输出 N 个「物体查询」，每个查询直接预测一个框 + 一个类别。

训练时，用匈牙利算法把预测的 N 个框和真实的 M 个框进行一一匹配（N > M，多余的匹配到「背景」类），然后计算损失。

DETR 的优点：流程极其简单，没有锚点框设计，没有 NMS 后处理，真正的端到端。缺点：收敛慢，在 COCO 上需要训练 500 个 epoch（Faster R-CNN 通常只需要 12-24 个 epoch）。

后续的 Deformable DETR 等改进版显著加速了收敛，使 DETR 系列成为可以和 CNN 检测器竞争的重要路线。


# 第二板块：语义分割


## 第 28 页 · 从稀疏预测到密集预测

PPT 内容：
目前我们讨论了图像分类与目标检测，这两者主要关注图像整体或特定区域的稀疏预测，输出的是类别标签或边界框。不同于这两者仅在图像级或对象级给出离散结果，密集预测则更有挑战性——它要求模型对每一个像素都做出预测。我们将重点讨论语义分割和实例分割。

### 讲解

⭐ 这一页是整个第二板块的总起。

PPT 用一张图对比了四个任务的输出精细程度：

图像分类：一张图 → 一个标签（「猫」）。输出信息极其稀疏。

目标检测：一张图 → 若干边界框和类别标签（「框 A 里有猫」「框 B 里有狗」）。输出信息仍然稀疏，只是多个稀疏输出的集合。

语义分割：一张图 → 每个像素一个类别标签。红色的像素是「人」，绿色的像素是「车」，蓝色的像素是「道路」。输出密度 = 图像分辨率。

实例分割：语义分割 + 区分个体。不仅是「这些像素都是人」，而且要区分「这个人是张三，那个人是李四」。

从分类到检测到语义分割再到实例分割，本质上是在逐步增加输出的信息密度。


## 第 29 页 · 什么是语义分割

PPT 内容：
与图像分类任务只能判断整张图像的类别不同，语义分割能够为图像中的每一个像素点分配一个语义类别标签，从而实现对图像的精细化理解，被广泛应用于自动驾驶、医学影像分析、遥感图像分析等领域。

### 讲解

语义分割的输入是一张图，输出是一张**和输入同样大小**的分割图，每个像素的值是该像素的类别编号。比如：

- 0：背景
- 1：道路
- 2：行人
- 3：车辆
- 4：建筑物
- ……

在自动驾驶中，语义分割告诉你：「这个像素是可行驶的路面」「那个像素是必须避开的行人」「那边是路沿不能越过去」。这是一个像素级的场景理解。

语义分割的衡量指标通常是 mIoU（mean Intersection over Union）——每一类的预测区域和真实区域的交集除以并集，所有类的 IoU 取平均。


## 第 30-32 页 · 语义分割三大应用场景

PPT 内容（连续 3 页）：
应用场景 1——自动驾驶：为自动驾驶车辆提供像素级的环境感知能力，精确识别可行驶区域、检测车辆与行人等动态障碍物、理解车道线与交通标志等道路要素。示例：Cityscapes 数据集，城市街景语义分割标准数据集（19 类像素级标注）。

应用场景 2——医学影像：对 CT、MRI 等医学扫描图像进行像素级的精细标注，自动分割出心脏、肝脏、肿瘤等特定器官或病灶区域，辅助医生进行快速准确的诊断与量化分析。示例：BRATS 数据集，多模态脑部 MRI 图像，标注健康组织、水肿区、坏死区等多个区域。

应用场景 3——遥感图像：对卫星遥感影像和航空摄影进行像素级的地物分类，自动识别并提取建筑物、道路、水体、植被、农田等地理要素，实现大范围的土地覆盖分类与变化检测。

### 讲解

这三页展示了语义分割最核心的三大应用方向。

自动驾驶中的 Cityscapes 数据集是业界标准——19 个类别的精细标注，包括道路、人行道、建筑、墙壁、围栏、杆子、交通灯、交通标志、植被、地面、天空、行人、骑车人、汽车、卡车、公交车、火车、摩托车、自行车。

医学影像中语义分割的价值尤其突出。医生需要知道肿瘤的确切大小和形状以制定治疗方案。手动勾画一个肿瘤区域的边界可能需要 30 分钟，而训练好的分割模型可以在几秒内完成。

遥感图像中，一张卫星图像覆盖几十平方公里，人工标注所有道路和建筑是不现实的。语义分割可以自动化这一过程——大范围的土地覆盖分类、城市扩张监测、灾害评估都依赖这项技术。


## 第 33 页 · 语义分割的本质

PPT 内容：
如前所述，我们可以将语义分割理解为像素级的分类任务。然而，单个像素缺乏上下文，无法独立判断类别。

### 讲解

这一页揭示了语义分割的根本性困难。

如果把语义分割理解为「对每个像素单独做一次分类」，那你可以用一个滑动窗口，每次只看一个像素来做判断。但这是行不通的——一个孤立的像素没有任何语义信息。一个红色的像素可能是汽车尾灯、红绿灯、行人的红衣服、路边的一朵花。你必须看到它周围的上下文才能确定。

所以语义分割网络必须同时具备两种能力：
1. 理解全局语义：这是什么场景？有哪些物体？它们之间是什么关系？
2. 保留局部细节：物体边界在哪？这个像素到底属于物体 A 还是物体 B？

这需要网络既能看到大的感受野（理解语义），又能保持高分辨率（精确定位）。这是接下来的几页要解决的核心矛盾。


## 第 34-38 页 · 语义分割的架构思考

PPT 内容（连续 5 页）：
思路一：先对图片下采样，然后做像素级预测。问题：图像分类的下采样是为了增大感受野、理解高层语义信息即可。而我们是像素级的预测！

思路二：直接保持原分辨率做预测。问题：每张图的分辨率通常较大（512×512, 1024×1024...），这会导致计算量爆炸。不下采样也会影响上下文的理解。

解决方案：采用下采样-上采样的结构。层次化特征提取捕获多尺度上下文和语义信息；瓶颈层压缩特征维度，降低后续解码开销；解码器上采样，兼顾定位精度与语义一致性。

### 讲解

⭐ 这 5 页展示了语义分割网络设计中最根本的矛盾及其解决方案。

如果把第六章的分类 CNN（比如 VGG）直接拿来用，会发现一个问题：分类 CNN 通过反复下采样（池化）把 224×224 的图压缩到 7×7，每个 7×7 位置有 512 维特征。这对分类很好——7×7 的位置保留了最强的语义信息，感受野覆盖全图。但如果你要把这个 7×7 的特征图放大回 224×224 去做逐像素分类，你会发现边界完全糊了——下采样丢失了太多的空间细节。

那不下采样行不行？保持全程 224×224？问题是：第一，计算量巨大（全分辨率上做所有卷积）；第二，不下采样感受野扩不上去——深层神经元只能看到图像的一个小角落，缺乏全局上下文。

这就是编码器-解码器架构诞生的动机：

编码器（Encoder）：像分类 CNN 一样逐步下采样——分辨率降低、通道数增多、语义信息越来越丰富。这段负责「理解图像中有什么」。

解码器（Decoder）：逐步上采样——分辨率恢复、通道数减少。这段负责「恢复空间细节」。

瓶颈（Bottleneck）：中间层——尺寸最小、通道数最多、语义信息最浓缩。

后面的 FCN、U-Net、DeepLab、SegFormer 都是编码器-解码器架构的不同实现。


## 第 39-43 页 · 上采样技术

PPT 内容（连续 5 页）：
上采样——重建像素级预测图，将低分辨率特征图恢复至原始空间尺寸。

方法一：反池化——最近邻插值或 0-填充。

方法二：单线性插值——利用邻近已知点的加权平均估计未知点值。

方法三：双线性插值——利用目标点周围 4 个相邻像素的加权平均值来计算新像素位置的值。

方法四：Max Unpooling——利用池化时记录的位置信息，把值放回原来的位置。

方法五：转置卷积——每个输入像素通过卷积核「扩散」到输出的 3×3 区域。重叠区域求和。卷积核权重可学习。

### 讲解

这 5 页详细介绍了上采样的各种技术。下采样（池化、strided conv）把图像变小，上采样把小图放大回来。

#### 最近邻插值

最简单粗暴的方法。目标像素的值 = 离它最近的已知像素的值。7×7 想放大到 14×14，每个新像素直接抄最近的老像素的值。快，但效果粗糙，有锯齿。

#### 双线性插值

取目标像素周围 4 个最近的已知像素，根据距离加权平均。比最近邻平滑，但仍然是手工规则，不可学习。

#### Max Unpooling

训练时做 Max Pooling 的时候，除了记住最大值是多少，还额外记住最大值在 2×2 窗口里的位置（左上？右上？）。上采样时，把值精确放回原来记录的位置，其余位置填 0。比最近邻和双线性保留了更多的空间结构信息。

#### 转置卷积

这是可学习的上采样方法。它的思路是把卷积反过来：普通卷积是 3×3 的核滑过去，9 个像素变成一个；转置卷积是 1 个像素通过 3×3 的核「扩散」成 9 个像素。stride=2 时会在输出像素之间插入 0 来实现空间放大。卷积核的权重通过训练学习，让网络自己决定最好的上采样方式。

⚠️ 转置卷积有个常见问题：棋盘效应（checkerboard artifact）。当 stride 和 kernel size 不匹配时，某些输出位置的梯度累积过多，导致输出有规律性的格状图案。可以通过调整 stride 和 kernel size 的比例来缓解。


## 第 44-45 页 · FCN：全卷积网络

PPT 内容（连续 2 页）：
FCN（Fully Convolutional Network）：从图像分类到图像分割的桥梁。

核心思想：全连接层 → 卷积层。全连接层做整图分类，卷积层做像素级预测。支持任意尺寸输入，端到端训练。输出热力图而不是单一类别。

多尺度特征融合：深层特征语义强但位置模糊，浅层特征位置准但语义弱。因此融合不同层的特征，兼顾语义和定位。

### 讲解

⭐ FCN 是语义分割领域具有里程碑意义的工作（CVPR 2015, Long et al.），第一次让端到端的深度学习在语义分割上取得突破。

#### 全卷积化

第六章学过，分类 CNN（如 VGG）的末尾是几个全连接层：FC(4096) → FC(4096) → FC(1000)。FCN 做了一个简单但关键的替换：把全连接层替换成 1×1 卷积层。

FC(4096) 变成 Conv(4096, 1×1)。FC(1000) 变成 Conv(1000, 1×1)。

这个替换之后，网络不再要求固定的输入尺寸——因为卷积层可以处理任意尺寸的输入。而且输出不再是一个标量（类别标签），而是一张热力图（每个类别每个空间位置都有一个得分）。

训练时，对预测图和真实分割图逐像素计算交叉熵损失，对所有像素取平均。推理时，输入整张图，输出一张像素级的分割预测图。

#### 多尺度特征融合

直接对最后一层（32 倍下采样后）的特征图上采样到原图大小做预测——FCN-32s——效果不行。因为 32 倍下采样后位置信息丢失严重，边界模糊。

FCN 的做法是把不同层的预测融合：

FCN-16s：把倒数第二层（16 倍下采样）的预测和最后一层上采样后的预测相加，再做最终上采样。效果提升。

FCN-8s：再把倒数第三层（8 倍下采样）的也加进来。效果最好。

这就是「跳跃连接」在语义分割中的早期实践——浅层特征（位置准）和深层特征（语义强）通过相加的方式融合，兼顾定位精度和语义理解。


## 第 46-48 页 · U-Net

PPT 内容（连续 3 页）：
U-Net 通过完全对称的编码器-解码器结构和密集的跳跃连接，在恢复分辨率的同时精确保留空间细节，成为小数据医学图像分割的标杆方法。

跳跃连接：将编码器对应阶段的特征图与解码器上采样后的特征图按通道拼接。保留空间细节——浅层的边缘、纹理信息直接传递到解码端，避免上采样丢失。精确定位——解码时能准确知道位置，提升分割边界精度。多尺度融合——深层语义 + 浅层细节，让网络自己学习如何组合。

### 讲解

U-Net（2015, MICCAI, Ronneberger et al.）是医学图像分割中最经典、最广泛使用的网络架构。它的名字来源于其「U」形的架构——左边编码器逐步下采样，右边解码器逐步上采样，中间有跳跃连接把左右对应层连起来。

#### 与 FCN 的区别

FCN 的跳跃连接是「特征图相加」。U-Net 的跳跃连接是「特征图按通道拼接（Concat）」。

相加和拼接的区别：相加保留通道数不变，但要求两个特征图语义上能对齐。拼接增加了通道数，让后续的卷积层来决定如何融合浅层的空间信息和深层的语义信息——给网络更大的自由度。

#### 为什么 U-Net 适合医学影像？

医学影像分割有三个特点：
1. 数据量小：不可能有几万张标注图，几百张就算多了
2. 需要精确的边界：肿瘤边界差一毫米，手术方案完全不同
3. 物体结构相对固定：器官的位置和形状有一定规律

U-Net 的密集跳跃连接把浅层的精细空间信息「抄近道」送到解码端，避免在多次下采样中丢失。这对于边界精度至关重要。而且 U-Net 在数据增强方面下了很多功夫——弹性形变（模拟组织变形）、随机旋转和缩放——使得几百张图也能训练出一个不错的分割模型。

#### U-Net 的架构细节

编码器：每层两个 3×3 卷积 + ReLU，然后 2×2 MaxPool 降采样。通道数逐步翻倍：64 → 128 → 256 → 512 → 1024。

解码器：每层先上采样（转置卷积或双线性插值）把分辨率翻倍，然后把编码器对应层的特征图拼接过来，再做两个 3×3 卷积 + ReLU。通道数逐步减半。

最后的输出层：1×1 卷积 → 输出通道数 = 类别数，逐像素 softmax。


## 第 49-51 页 · DeepLab 系列

PPT 内容（连续 3 页）：
DeepLab 是语义分割领域的经典系列工作，通过空洞卷积扩大感受野、ASPP 模块建模多尺度上下文等设计，系统性解决了「多尺度目标」和「边界模糊」两大核心挑战。

DeepLab v1/v2：引入空洞卷积，扩大感受野，捕获更多上下文。

DeepLab v3：单一空洞率无法适应所有尺度目标。提出 ASPP（Atrous Spatial Pyramid Pooling），并行多个不同空洞率的卷积 + 全局池化，同时捕获局部细节与全局上下文。

DeepLab v3+：最终形态，改进 ASPP 加入 BN 层，实现特征图的跨 Block 融合。

### 讲解

DeepLab 系列面对的是语义分割的两个核心挑战：**多尺度目标**和**边界模糊**。

#### 空洞卷积的意义

在第六章学过，空洞卷积可以在不降低分辨率的情况下扩大感受野。这在语义分割中极其重要——你需要在保持高分辨率的同时理解大范围的上下文。一张图上有一座近处的大建筑和远处的小行人，需要同时处理不同尺度的物体。

#### ASPP：空洞空间金字塔池化

DeepLab v3 的核心创新是 ASPP 模块（Atrous Spatial Pyramid Pooling）。它并行使用多个不同空洞率的卷积：

- 1×1 卷积（空洞率 = 1，最细粒度的局部信息）
- 3×3 空洞卷积，空洞率 = 6（中等感受野）
- 3×3 空洞卷积，空洞率 = 12（较大感受野）
- 3×3 空洞卷积，空洞率 = 18（全局感受野）
- 全局平均池化 + 1×1 卷积 + 上采样（捕获图像级别的全局上下文）

5 个分支的结果拼接在一起，让后续层可以同时利用从局部到全局的各种尺度信息。

💡 这本质上就是 GoogLeNet 的 Inception 思想在语义分割中的翻版——用并行多分支结构来捕获多尺度信息。


## 第 52-54 页 · SegFormer

PPT 内容（连续 3 页）：
SegFormer：基于 Transformer 骨干网络的语义分割领域的代表性工作。通过引入层次化 Mix Transformer Backbone 和轻量级 MLP Decoder，开创了高效灵活的新范式。

架构由两部分组成：层次化 Transformer-based Encoder + 仅由几个 FC 构成的 Decoder。

MiT Backbone：层次化特征提取、重叠 Patch Embedding、混合前馈网络 Mix-FFN、相比传统 baseline 具备更大感受野。由于 MiT Backbone 具备更大感受野，解码器无需复杂结构。

### 讲解

SegFormer 是语义分割从 CNN 向 Transformer 范式转变的关键里程碑。

#### 为什么需要 Transformer？

CNN 的感受野虽然可以通过堆叠和空洞卷积扩大，但本质上还是局部连接的——每个输出像素的感受野是有限的。Transformer 的自注意力机制则可以天然地为每个像素提供全局感受野——它可以同时「看」到图像中的所有位置。

#### MiT（Mix Transformer）Backbone

不同于 ViT 的单一分辨率处理方式，MiT 采用了**层次化**设计——生成 4 个不同分辨率的多尺度特征图（类似 ResNet 的 stage 结构）。这保留了 CNN 多尺度处理的优势。

重叠 Patch Embedding：ViT 用的不重叠的 patch 切分（16×16 的块各不相干），MiT 用重叠的卷积做 patch 嵌入，相邻 patch 之间有信息重叠——更连续。

Mix-FFN：在前馈网络中使用 3×3 深度卷积（而非标准的 MLP），引入局部位置信息。普通的 FFN（全连接）完全没有空间信息，Mix-FFN 通过深度卷积加入了局部空间先验。

#### 轻量级 All-MLP Decoder

这是 SegFormer 最巧妙的设计之一。因为 MiT 编码器已经赋予了每个位置足够大的感受野（通过自注意力），解码器不需要像 FCN、U-Net 那样设计复杂的多层解码结构。只需要几个 MLP 层就能把多尺度特征融合起来并生成最终的分割图。

结果：SegFormer 在精度上超越了 DeepLabV3+，在推理速度上快了一个数量级。


## 第 55-57 页 · MaskFormer：掩码分类新范式

PPT 内容（连续 3 页）：
MaskFormer 提出掩码分类新范式，将问题转化为「预测一组 {掩码, 类别} 对」的集合预测问题，通过 Query 机制与匈牙利匹配实现端到端训练，无需复杂后处理。

架构：Backbone + Pixel Decoder 生成像素级嵌入，Transformer Decoder 处理 N 个 Query 生成对象级嵌入。通过计算 Query 嵌入与像素嵌入的相似度，直接生成 N 个候选掩码。并行输出类别预测和掩码预测。

Object Queries：使用 N 个可学习的 query 向量。每个 query 独立预测一个语义 mask 和一个全局类别标签。通过 Transformer 解码器并行生成所有 mask。

### 讲解

MaskFormer 是对语义分割范式的重新思考。

传统语义分割的定义是「对每个像素分类」。MaskFormer 的定义是「预测一组掩码，每个掩码有一个类别标签」。这两个定义在数学上等价，但在实现上完全不同。

传统方法：在像素空间中做交叉熵——每个像素独立分类，最终类别相同且连通的像素区域自然形成分割结果。但像素之间是独立分类的，模型并不知道「这两个像素属于同一个物体」。

MaskFormer：用 N 个可学习的 query 向量，每个 query 输出两件东西——一个二值掩码（哪些像素属于我）和一个类别标签（我是什么类别）。推理时同类掩码合并（可能多个 query 对应同一类别的不同物体）。

把 DETR 的检测思路搬过来：拼的不是位置回归和类别分类，而是「这个 query 覆盖哪些像素 + 是什么类别」。用匈牙利匹配在训练时把预测的掩码和真实掩码配对。

💡 这是语义分割从「逐像素分类范式」到「集合预测范式」的重要转变，为 SAM 等大模型铺平了道路。


## 第 58-60 页 · SAM：Segment Anything Model

PPT 内容（连续 3 页）：
SAM 是 Meta 发布的分割基础模型，通过提示式交互（点/框/文本）实现 zero-shot 通用分割，开创了「一个模型分割万物」的新范式。

SA-1B 数据集：模型 + 人工构建数据集，包含 11 亿+ 掩码、1100 万+ 图片。

可提示分割：点 Prompt、框 Prompt、文本 Prompt。

实验结果：突破闭集限制——SAM 未在多样化数据上训练，却能有效分割训练集中未充分出现的物体类别。突破单一任务限制——SAM 可在推理时充当更大系统中的组件。

### 讲解

⭐ SAM 可能是 2023 年视觉领域最出圈的工作，被很多人称为「分割领域的 GPT 时刻」。

#### SAM 解决了什么问题？

传统语义分割模型只能分割训练数据中见过的类别。如果训练集只有「人」「车」「道路」，那模型永远不会输出「消防栓」「垃圾桶」「长凳」的分割。这就是闭集限制。

SAM 的目标是：**无论来什么图，无论要分割什么物体，都能分。** 它不依赖于固定的类别列表。

#### SAM 的工作方式

SAM 不是「输入图 → 输出分割结果」（那样又回到闭集了）。

SAM 是「输入图 + 你给的提示 → 输出分割结果」。提示可以是：
- 一个点：「请分割出这个位置的东西」
- 一个框：「请分割这个矩形框里的物体」
- 文本描述：「请分割出图中所有的猫」
- 不给定提示时，SAM 也会输出所有可能的候选掩码

#### SAM 怎么训练的？

SA-1B 数据集是核心。Meta 设计了一个三阶段的数据引擎：
1. 人工标注阶段：标注人员在 12 万张图上做详细分割标注，训练初始模型
2. 半自动阶段：模型自动生成候选掩码，标注人员选择和修正
3. 全自动阶段：模型在 1100 万张图上自动生成掩码，利用多个视角验证质量

最终得到 11 亿个高质量掩码标注——训练出了能够泛化到未见类别、未见场景的通用分割模型。

SAM 的出现标志着视觉基础模型时代的到来——一个模型不再只为一个任务训练，而是作为基础设施为下游各种任务提供支持。

PPT 内容（连续 2 页）：
常见框架：MMEngine、Detectron 2。常见工具：WandB——结果曲线、Loss 曲线等监控，结果可视化对比。

这是两页比较务实的知识点，介绍了工业界和学术界常用的框架。

# 第三板块：实例分割


## 第 63 页 · 什么是实例分割

PPT 内容：
实例分割：为图像中的每一个物体实例生成一个二值掩码 Binary Mask，精确到像素级别。需要同时解决检测（定位和分类）和分割（像素级描绘）两个子任务。

图像分类 → 目标检测 → 语义分割 → 实例分割：从只知道有什么，到知道在哪但不知道轮廓，到知道每个像素是什么但不区分个体，到区分同一类别的不同个体。

### 讲解

实例分割是语义分割 + 目标检测的结合体。

语义分割把图像中的「人」的像素都标注为同一类别（比如红色），但它不管画面里有几个人——两个人接在一起的话，分割图上就是一团红色，分不清谁是谁。

实例分割必须区分：这是「人 A」的像素，那是「人 B」的像素。为每个物体实例独立输出一个二值掩码，精确到像素级别。

因此实例分割网络需要同时做两件事：找到每个物体在哪里（检测），把每个物体精确到像素地抠出来（分割）。


## 第 64-68 页 · Mask R-CNN

PPT 内容（连续 5 页）：
Mask R-CNN 核心思想：在 Faster R-CNN 的基础上，增加一个预测掩码的分支。

架构：共享主干和 RPN 与 Faster R-CNN 相同。三个并行输出头：分类头、边界框回归头、掩码头。RoI Align 替换 RoI Pooling。

问题：RoI Pooling 存在两次量化取整，导致特征与原始 RoI 位置不对齐，掩码边界错位。

RoI Align 解决方案：移除量化操作，使用双线性插值来精确地从特征图上采样 RoI 所需的值。

掩码预测策略：逐类掩码——掩码头为每个类别都输出一个二值掩码；根据分类头的预测结果，只选择对应类别的掩码通道作为最终输出。

Mask R-CNN 还可用于人体姿态估计——将关键点（如关节）视为一种特殊的「实例」，用类似掩码的方式预测。

### 讲解

⭐ Mask R-CNN 是实例分割领域的标杆方法，简洁而强大。

#### 在 Faster R-CNN 上加一个掩码头

Faster R-CNN 已经有两个输出头：分类头（这个框是什么类别）、回归头（框应该怎么修正）。Mask R-CNN 加了第三个头——掩码头。

掩码头是一个小的全卷积网络，输入是 RoI Align 输出的固定尺寸特征（比如 14×14），输出是 28×28 的二值掩码——表示「在这个 RoI 内，哪些像素属于这个物体」。

#### RoI Align：去量化

这是 Mask R-CNN 最核心的技术贡献。

Fast/Faster R-CNN 的 RoI Pooling 有两步量化：
- 把 RoI 坐标从浮点数取整到最近的整数像素位置
- 把 RoI 区域分成网格时，每个 bin 的边界也取整

对于目标检测来说，几个像素的偏移影响不大（边界框差一两个像素，IoU 仍然很高）。但对于实例分割来说，几个像素的偏移会导致掩码的边界完全错位——人的轮廓偏移 3 个像素就是明显的误差。

RoI Align 的做法：不取整，保留浮点坐标。对于需要采样的点，用双线性插值从最近的 4 个像素算出来。全程不做量化，最大化空间精度。

#### 掩码预测策略

掩码头为**每个 RoI** 输出 K 个 28×28 的二值掩码（K 是类别数）。训练时只计算真实类别对应的那个掩码通道的损失（对每个像素做 sigmoid 交叉熵）。推理时分类头先确定这个 RoI 是什么类别，然后取对应那条掩码。

这种「分类和掩码独立预测」的设计意味着：掩码不需要做多类别 softmax（不会和其他类别竞争），每个像素的值是 0-1 之间的概率——类别之间的掩码可以重叠（这在真实世界中是可能的——一个人可以部分遮挡一辆车）。

#### 多功能性

Mask R-CNN 稍微改一下就能做人体姿态估计。姿态估计本质上就是定位人体的关键点（眼睛、鼻子、肩膀、手肘、手腕……）。把这些关键点看作是「只有一个像素的掩码」（实际上是 56×56 热力图上的一个高斯峰值），用类似的掩码预测头就能预测关键点位置。


# 第四板块：视频理解


## 第 69-70 页 · 从静态图像到动态视频

PPT 内容（连续 2 页）：
从静态图像识别迈向动态视频理解。视频的本质：一连串按时间顺序排列的图像帧（视频 = 2D + 时间）。数据表示：一个 4D 张量，通常为 T × 3 × H × W，其中 T 是帧数。

核心挑战：如何有效建模和利用时间维度上的信息。

### 讲解

⭐ 本章最后一部分从 2D 图像跨入 3D（2D 空间 + 1D 时间）的视频。

视频分类的输入不是一张图，而是一段视频片段，包含 T 帧（比如 16 帧或 32 帧，约 1-2 秒）。

核心困难：怎么处理时间维度？

最简单的办法是把视频当独立图像集合——每一帧独立送进 2D CNN 分类，然后把所有帧的结果取平均。这个方法叫「单帧 CNN」，虽然简单，但在很多场景下是一个很强的基线——因为很多动作从一帧就能猜出来（「滑雪」→ 有雪、有滑雪板、有滑雪姿势）。

但这个方法的明显缺陷是：完全感知不到运动。一个静止的人站在滑雪板上，和一帧滑雪的过程，模型分不出来。真正需要理解「这个人在动」「他在往哪个方向动」，就需要在时间维度上建模。


## 第 71-72 页 · 视频分类的任务定义与数据处理

PPT 内容（连续 2 页）：
任务定义：给定一个视频片段，预测其所属的动作类别。关键区别——图像识别「物体」，视频识别「动作」或「事件」。

核心挑战：视频数据量巨大。未压缩的视频标清约 1.5 GB/分钟、高清约 10 GB/分钟。

解决方案——基于短片段训练。不在完整长视频上训练，而是采样短视频片段 clips。训练阶段模型学习对这些短片段进行分类；测试阶段对多个片段进行推理，取平均预测结果作为最终视频级预测。

### 讲解

这两页介绍了视频理解的工程现实和数据处理策略。

视频的数据量是巨量的。一张 224×224 的 RGB 图片约 150KB。一个 10 秒的高清视频（30fps × 10s = 300 帧）就已经是 300 × 150KB ≈ 45MB。在实际项目中，不可能把几分钟甚至几小时的完整视频一次性加载到 GPU 显存中。

这个现实直接催生了「短片段训练」策略：

训练时：从每个长视频中随机采样一个小片段（比如 16 帧，约 0.5 秒），送进网络训练。

测试时：从视频中平均采样多个片段（比如 10 段），每个片段独立预测，取平均作为整个视频的分类结果。

这种策略的假设是：一个动作在视频各个阶段的局部表现足以推断整体——「打篮球」这个动作不需要看 30 分钟，几秒的运球、跳跃、投篮片段就足够了。


## 第 73-78 页 · 视频特征融合的三种策略

PPT 内容（连续 6 页）：
单帧 CNN（基线方法）：将视频分类问题简化为对每一帧独立进行图像分类。优点：实现简单，常作为强有力的基线。

早期融合：将 T 帧的 RGB 通道堆叠，形成一个 (3T) × H × W 的「超图像」，送入标准 2D CNN。第一个卷积层直接比较不同帧之间的像素差异。局限：第一个卷积层就将所有时间信息压缩掉，后续网络无法再访问时间结构。

晚期融合：先独立提取每帧的高层特征，再在特征层面进行时间融合。方式 A：将所有帧的特征向量拼接送入 MLP。方式 B：在时空维度上全局平均池化后送入线性分类器。局限：每帧独立处理，难以比较帧与帧之间的底层运动信息。

### 讲解

⭐ 这 6 页展示了时间信息融合的三种基本策略，是理解视频 CNN 的基石。

#### 早期融合

把连续 T 帧的 RGB 通道堆叠在一起。如果 T=10，那输入就是 30 个通道（10 帧 × 3 通道），像一个「30 通道的超图像」送进 2D CNN。

好处：第一个卷积层可以直接看到相邻帧的像素差异——物体移动引起的像素变化。

坏处：第一个 2D 卷积后的输出通道数（比如 64）已经把所有时间维度压缩掉了——30 个通道变成一个输出通道的特征图。后续所有层都再也访问不到原始的时间结构。只有一层能建模运动，对复杂的长期动作不够。

#### 晚期融合

每帧独立走一个 2D CNN（权值共享），得到每帧的高层特征向量。然后把这些特征向量组合起来——拼接后送 MLP，或者取平均后送分类器。

好处：每帧的空间信息被充分提取了。

坏处：每帧独立处理，帧与帧之间没有底层交互——「这个像素从第 3 帧到第 4 帧往右移了 2 格」这种精细的运动信息无法被捕捉。因为 CNN 浅层能感知到的像素位移，到了高层特征里已经被压缩成抽象的语义了。


## 第 79-80 页 · 3D CNN

PPT 内容（连续 2 页）：
3D CNN：使用 3D 卷积核在整个网络中逐步、同步地融合空间和时间信息。将 2D CNN 中的卷积和池化操作全部替换为 3D 版本。每一层的特征图都是 4D 张量，保留了完整的时空结构。

3D 卷积精度更高，但是计算开销也极其高昂。

### 讲解

2D 卷积核是 K×K 的二维矩阵，只在高度和宽度方向滑动。3D 卷积核是 K×K×Kt 的三维矩阵——除了空间维度 K×K 外，还有一个时间维度 Kt（比如 3×3×3 意味着同时看 3 帧，每帧 3×3 的空间区域）。

3D 卷积在网络每一层都保留时间维度。比如输入是 16 帧的视频，经过一个 3D 卷积后时间维度可能变成 14 或 16（取决于 padding 和 stride），空间维度可能减半（如果 stride=2）。到最后一层，时间维度和空间维度分别被压缩到几个像素——最后通过全局时空池化变成一个向量送进分类器。

好处：每一层都在同时处理空间和时间信息——浅层捕获局部运动（物体边缘的微小位移），深层捕获高层动作模式（跑步 vs 走路 vs 跳跃）。

坏处：计算量巨大。2D 卷积的计算量正比于 K^2 × C_in × C_out × H × W；3D 卷积的计算量正比于 K^3 × C_in × C_out × T × H × W——多了时间维度 T 和额外的卷积维度 Kt。一个 3D ResNet-50 训练起来比 2D ResNet-50 慢 5-10 倍。


## 第 81-82 页 · 运动识别与双流网络

PPT 内容（连续 2 页）：
运动信息是视频理解中至关重要的线索，有时甚至比外观信息更重要。

度量运动——光流：光流是一个二维向量场，描述从帧 t 到帧 t+1 每个像素点的位移 (dx, dy)。水平分量 (dx) 和垂直分量 (dy) 构成运动模式。

双流网络：融合策略——在预测阶段，将两个流的 softmax 分数进行平均，或使用 SVM 进行融合。性能优势——双流网络显著超越了仅使用空间信息或早期/晚期融合的方法。

### 讲解

⭐ 这是一个深刻洞察：动作识别中，「怎么动」往往比「长什么样」更重要。

#### 光流是什么？

光流是计算机视觉中描述运动的经典技术。对于视频中相邻两帧，它计算每个像素在两帧之间「漂」了多远、往哪个方向漂。

输出的光流有两个通道：水平位移 dx 和垂直位移 dy。就像一个二维向量场——每个像素上有一个小箭头，表示它往哪移、移了多少。彩色编码的光流图看起来像流动的色彩图案。

为什么有用？因为光流直接编码了运动。一个人往上跳——光流中人的身体区域会有向上的箭头；挥手——手周围有横向快速运动的光流箭头。

#### 双流网络

Karen Simonyan 和 Andrew Zisserman 在 2014 年提出了双流网络：

**空间流**：输入单帧 RGB 图像（外观信息），用 2D CNN 提取特征。回答「画面里有什么？」
**时间流**：输入堆叠的 10 帧光流图（运动信息），用另一个 2D CNN 提取特征。回答「画面里的东西在怎么动？」

两个流的 softmax 分数平均（或加权组合）得到最终分类。空间流告诉你「这个人穿着滑雪服、有滑雪板」，时间流告诉你「他正在快速向下滑」。两者结合——「滑雪」。

实验结果：双流网络远远超越了仅用 RGB 帧或早期/晚期融合的方法，特别是对于运动特征明显、外观差异小的动作（如「挥手」「点头」和「摇头」——外观几乎一样但光流模式完全不同）。


## 第 83-88 页 · 长期时序建模

PPT 内容（连续 6 页）：
局限性：3D CNN、双流网络主要处理短片段（约 2-5 秒），只能捕获局部的时序动态。现实需求——许多复杂活动包含跨越数十秒甚至数分钟的长期依赖。

RNN/LSTM——序列建模的经典工具：使用 CNN 逐帧提取特征向量，按时间顺序送入 RNN，RNN 的最终状态用于最终预测。优势在于理论上可捕获任意长度的时序依赖。

循环卷积网络：将 RNN 的递归思想与 CNN 的权重共享思想结合。每层在时间步 t 的输出，还取决于同一层在上一步 t-1 的输出。问题：RNN 的计算是顺序依赖的，难以并行。

时空自注意力：将自注意力机制应用于 3D CNN 的特征图上，以捕获长距离时空依赖。

I3D——膨胀 2D 网络至 3D：复用在图像上取得成功的 2D CNN 架构来构建 3D 视频模型。将 2D 卷积核 (h×w) 和 2D 池化核膨胀为 3D 核 (t×h×w)。

从 CNN 到 Transformer：Vision Transformer 及其变体（如 MViT, VideoMAE）已成为当前视频理解的主流和 SOTA 架构。

### 讲解

这 6 页展示了视频理解中如何处理「长时间」的问题。

#### 为什么要做长期建模？

3D CNN 看 16 帧（约 0.5 秒）可以判断「跳跃」「挥手」等瞬时动作。但复杂活动——「做一顿早餐」（5 分钟）、「打一场篮球比赛」（30 分钟）——包含多个步骤、多次往返、远距离的事件关联。模型需要同时知道「10 秒前发生了什么」「30 秒后将要发生什么」。

#### CNN + RNN/LSTM

很自然的思路：用 CNN 提取每帧（或每几帧）的特征，按时间顺序送进 LSTM。LSTM 的记忆单元可以在理论上记住很久以前的信息。最后用 LSTM 的输出（或所有时间步输出的平均）做分类。

问题：LSTM 是顺序计算的——必须先算 t=1，再算 t=2，再算 t=3……无法并行。而且 LSTM 在实践中有有效记忆长度（约几十到几百步），太长的依赖还是记不住。

#### I3D

「膨胀 2D 网络至 3D」是一个非常聪明的设计。想法是：既然在 ImageNet 上训练的 2D 模型已经学会了很好的空间特征，为什么不直接复用？

做法：把 2D 卷积核 (3×3) 沿着时间维「膨胀」成 3D 卷积核 (3×3×3)。把预训练的 2D 权重在时间维度上复制 N 次（N=时间维度大小），然后除以 N 做归一化——这样一开始的 3D 网络在每一帧上的行为和原始的 2D 网络完全一样。

然后在这个初始化的基础上在视频数据上微调。不需要从头训练 3D 网络（那需要巨量视频数据），而是站在 ImageNet 训练好的 2D 模型的肩膀上。

#### 时空自注意力

这是当前的方向。自注意力可以让网络中的每个时空位置（在 t1 时刻位置 A 的像素）直接和网络中的任何其他时空位置（在 t200 时刻位置 B 的像素）交互——理论上可以建模任意长度的依赖。

#### 从 CNN 到 Transformer

当前的趋势是：ViT 的各种变体（MViT、VideoMAE、TimeSformer）已经成为视频理解的 SOTA。你不再需要手动设计早期融合、晚期融合还是 3D 卷积——Transformer 的自注意力机制天然就能处理时空维度的任意长度交互。


## 第 89-90 页 · 时序动作定位与时空动作检测

PPT 内容（连续 2 页）：
时序动作定位：给定一个未修剪的长视频，不仅要识别出其中发生的动作类别，还要精确定位每个动作发生的起止时间（帧）。这是视频领域的「目标检测」任务，只不过是在一维时间轴上进行。

时空动作检测：在未修剪的长视频中，同时在空间（定位人物边界框）和时间（定位动作起止）上检测并识别人物执行的原子动作。

### 讲解

这两页是视频理解的进阶任务。

视频分类回答的是：「这段剪辑是什么动作？」→ 一个标签。

时序动作定位回答的是：「这个长视频里什么时候发生了什么动作？」→ 一组 (开始时间, 结束时间, 动作类别) 的三元组。

时空动作检测更进一步：在空间上找出谁在做动作（边界框），在时间上找出什么时候在做动作（起止帧），以及做什么动作（类别标签）。

这相当于把目标检测从 2D 空间扩展到了 3D 时空——需要在一个 4D 体积（T×C×H×W）中找到物体和动作出现的区域。每条轴上的定位都需要精确到像素/帧级别。

输入：一个 5 分钟长度、未经修剪的监控视频。

输出：人 A，框 (200, 300, 100, 200)，时间 30s-45s，动作「奔跑」；人 B，框 (500, 100, 80, 180)，时间 45s-120s，动作「站立」……等等。


## 第 91 页 · 视频大语言模型

PPT 内容：
趋势：将强大的视频编码器（如 ViViT, VideoMAE）与大语言模型（LLM）相结合。
能力：实现开放词汇的视频问答、视频描述生成、基于指令的视频内容理解等。
通向通用人工智能（AGI）的重要一步！

### 讲解

这是全章的最后一页，展望了视频理解的未来方向。

视频大语言模型的思路和图像大语言模型（如 GPT-4V）类似，只不过输入从「一张图」变成了「一段视频」：

```
视频 → 视频编码器（ViViT/VideoMAE）→ 视频特征向量
用户问题：「视频中的人最后做了什么？」
                           ↓
            LLM（大语言模型）→ 文本回答：「最后一帧中，这个人打开了门并走出房间。」
```

视频大语言模型不仅可以做分类和定位，还能回答复杂的开放性问题。比如看到一段监控录像，你可以问：「这个视频你觉得有什么异常行为？」模型不再受限于固定的动作类别列表，而是可以用自然语言自由回答。

💡 这代表了计算机视觉从「解决具体任务」到「建立通用理解能力」的范式转变。也是整个深度学习领域向 AGI 迈进的重要一步。


## 全章核心回顾

整个第七章从目标检测开始，一路走到语义分割、实例分割，最后进入视频理解。每走一步，输出信息的密度就增加一层：

图像分类 → 一张图一个标签
目标检测 → 一张图多个框 + 标签（分类 + 定位）
语义分割 → 每个像素一个类别标签
实例分割 → 每个像素一个类别标签，且区分同一类的不同个体
视频理解 → 在时间维度上建模，识别动作和事件

技术演进的主线：

目标检测：R-CNN → Fast R-CNN → Faster R-CNN → YOLO/SSD → RetinaNet(Focal Loss) → DETR

语义分割：FCN → U-Net → DeepLab(ASPP) → SegFormer(Transformer) → MaskFormer → SAM

实例分割：Mask R-CNN（Faster R-CNN + 掩码头 + RoI Align）

视频理解：单帧 CNN → 早期/晚期融合 → 3D CNN → 双流网络 → CNN+LSTM → I3D → Video Transformer → 视频大语言模型

贯穿所有任务的两个核心思想：
1. 共享特征提取 + 多任务头：一个 backbone 提取通用特征，不同任务接不同的输出头
2. 从手工设计到端到端学习：选择性搜索 → RPN → 锚点框 → 无锚点 → Transformer 集合预测
