## 第 1–4 页 · 课程引入：计算机如何"看"图像？

### 1. 计算机视觉是什么？

简单来说，计算机视觉的目标就是**让机器能"看懂"图像**——给它一张图，它能告诉你图里有什么、在哪里。PPT 上举了三个典型应用：

- **人脸识别**：手机解锁、安防监控
- **自动驾驶**：识别行人、车辆、交通标志
- **医学影像**：辅助医生发现病灶

这三个场景的共同需求是：**从像素出发，理解语义**。

### 2. 人眼和机器眼的区别

这是一个很重要的基本认知：**人类看到的是连续的光影世界，计算机看到的是离散的数字矩阵。**

比如一张 224×224 的照片，在人眼里是"一只猫在草地上"，在计算机眼里就是一个 $224 \times 224 \times 3$ 的数字张量，每个格子里存着 0 到 255 之间的整数。计算机没有"猫"的概念——它只能处理数字。我们的任务，就是训练一个模型，把这个数字矩阵映射到"猫"这个语义标签。

### 3. RGB 色彩空间

RGB 是我们最常用的图像表示方式。每个像素由三个通道组成——红（Red）、绿（Green）、蓝（Blue）。每个通道的取值范围是 0 到 255，8 个比特（$2^8 = 256$ 级）。

几个直观的例子帮你建立感觉：

| 颜色 | R | G | B |
|------|---|---|---|
| 纯红 | 255 | 0 | 0 |
| 纯绿 | 0 | 255 | 0 |
| 纯蓝 | 0 | 0 | 255 |
| 黑色 | 0 | 0 | 0 |
| 白色 | 255 | 255 | 255 |

RGB 的原理来自人眼的三种视锥细胞——它们分别对红、绿、蓝光最敏感。组合三个通道，理论上可以表示约 $256^3 \approx 1670$ 万种颜色。

### 4. HSL 色彩空间

RGB 虽然适合机器存储和计算，但对人来说不够直观。比如你想把一张图的亮度调暗一点，在 RGB 里得把三个通道一起改，很麻烦。

HSL 把颜色拆成三个更符合人直觉的维度：

- **Hue（色相，H）**：这是什么颜色？红、橙、黄、绿、蓝、紫。用角度表示——0° 是红色，120° 是绿色，240° 是蓝色。
- **Saturation（饱和度，S）**：这个颜色有多"纯"？0% 就是灰色（完全褪色），100% 就是最鲜艳的纯色。
- **Lightness（明度，L）**：这个颜色有多"亮"？0% 是纯黑，100% 是纯白，50% 左右才是"正常"的颜色。

HSL 最大的好处是**把颜色本身和亮度解耦了**。在深度学习的数据增强中，我们经常调整 S 和 L 来模拟不同光照条件，而不改变物体本身的颜色属性。

### 本节要点

- 图像在计算机中就是数字矩阵（张量）
- RGB 是机器的语言，HSL 更像是人的语言
- 深度学习默认使用 RGB 作为输入

## 第 5–9 页 · 传统特征提取方法

### 1. 深度学习之前怎么做？

在 2012 年 AlexNet 出现之前，计算机视觉的标配流程是：

```
输入图像 → 手工特征提取器（人设计的） → 分类器（如 SVM）
```

特征提取器的好坏直接决定了最终效果。所以那个年代的计算机视觉研究者，大量的精力都花在"设计更好的特征"上。

### 2. SIFT：尺度不变特征变换

SIFT（Scale-Invariant Feature Transform）由 David Lowe 于 1999 年提出，是计算机视觉史上最重要的算法之一。它能在一张图中找到"关键点"，并为每个关键点生成一个 **128 维的描述向量**，用于后续的匹配和识别。

SIFT 有四个核心步骤：

**第一步：构建尺度空间。** 真实世界中的物体可能出现在任意距离——近处的猫很大，远处的猫很小。SIFT 通过"高斯金字塔"来处理尺度问题：对同一张图用不同 $\sigma$（标准差）的高斯核做模糊，得到一系列越来越模糊的图像。然后计算相邻模糊层之间的差值——这叫 DoG（Difference of Gaussian，高斯差分图像）。

为什么要用 DoG？因为 DoG 是对 LoG（Laplacian of Gaussian，高斯拉普拉斯算子）的高效近似。LoG 是一个经典的"blob 检测器"——它在一个亮斑或暗斑的中心位置产生强烈的极值响应。但 LoG 计算很贵（要做二阶导数卷积），而 DoG 只需要相邻两层相减，计算效率高得多。

在 DoG 金字塔的每一层，检查每个像素是否是局部极值——既要和自己所在层的 8 个邻居比，也要和上一层、下一层对应位置的各 9 个邻居比。总共和 26 个邻居比较。如果是最大值或最小值，就标记为候选关键点。

**第二步：关键点精确定位。** 上一步找到的候选点还不够精确。Lowe 用泰勒展开（到二阶）在候选点附近插值，精确定位到亚像素级别。同时剔除两类不好的点：对比度太低的（容易被噪声淹没）、落在边缘上的（不稳定，因为边缘上梯度在一个方向很强但正交方向很弱）。判断后者靠 Hessian 矩阵的特征值之比——如果两个特征值相差太大，说明是边缘而非角点/blob，应该剔除。

**第三步：方向分配。** 为了让描述子具有旋转不变性（图像转了，描述子不变），SIFT 计算关键点邻域内每个像素的梯度幅值和方向，统计一个 36 方向的梯度直方图（每 10° 一个 bin）。直方图的峰值方向就是关键点的"主方向"。之后所有的坐标运算都旋转到主方向上进行。

**第四步：生成 128 维描述子。** 以关键点为中心，取周围 16×16 像素的区域，旋转到主方向上，然后划分为 4×4 共 16 个子区域。每个子区域统计 8 个方向的梯度直方图。总共 $4 \times 4 \times 8 = 128$ 维。对这个向量做归一化（除以模长），再把超过 0.2 的值钳位到 0.2（抑制大梯度值的干扰），再归一化一次。这样得到的描述子对光照变化、旋转、尺度变化都具有鲁棒性。

简单总结 SIFT 的三个不变性：
- **尺度不变**：高斯金字塔
- **旋转不变**：主方向分配
- **光照不变**：梯度方向统计 + 归一化

### 3. SIFT vs HOG vs CNN vs ViT

PPT 中给出了四种方法的对比：

- **SIFT**：稀疏的（只在关键点处提取），描述局部区域的梯度方向
- **HOG（Histogram of Oriented Gradients）**：密集的（覆盖整幅图），统计每个 cell 内的梯度方向直方图。HOG + SVM 在行人检测上非常成功。
- **CNN**：端到端学习，从数据中自动学特征层次
- **ViT**：用 Transformer 的自注意力机制建模

传统方法的优势是**可解释**——你知道每一步在干什么。但表达能力的上限比较低。深度学习方法不需要人工设计特征，但像一个"黑盒"——你看不到每一层具体做了什么。这种从"知识驱动"到"数据驱动"的转变，是 2012 年以来计算机视觉领域最大的范式转换。

### 本节要点

- 传统方法靠人工设计特征，效果好但天花板低
- SIFT 通过高斯金字塔 + DoG + 方向归一化实现了三大不变性
- 深度学习让网络自己学特征，表达力更强但可解释性更差

## 第 10–12 页 · 为什么全连接网络不适合图像？

### 1. 全连接层的问题

假设我们有一张 32×32 的 RGB 图像（CIFAR-10 的尺寸）。把它展平成一个一维向量，得到 $32 \times 32 \times 3 = 3072$ 个数字。

现在把它送进一个全连接层（又叫多层感知机 / MLP）。如果第一层有 1000 个神经元，参数量就是 $3072 \times 1000 + 1000 \approx 307$ 万。

三个严重问题：

**第一，参数量爆炸。** 对于 ImageNet 级别的图像（224×224×3），仅第一层全连接就要 $224 \times 224 \times 3 \times 1000 \approx 1.5$ 亿个参数。一个隐藏层就比整个 ResNet-50（2500 万参数）还大 6 倍。

**第二，丢失空间结构。** 展平操作把图像的二维空间布局彻底破坏了。本来左上角的像素和它相邻像素之间有很强的空间关系，展平成一长串数字后，这种关系荡然无存。

**第三，没有平移不变性。** 一只猫在图像左边和右边，对全连接层来说是两个完全不同的输入模式。它无法利用"这两个位置上的猫本质上是同一个东西"这个常识。

### 2. 那该怎么办？

显然，我们需要一种新的网络结构，它能：
- 保留图像的二维空间结构
- 参数量可控，不随图像尺寸线性爆炸
- 天然具有平移等变性（图像平移了，输出也跟着平移）

这就是**卷积神经网络（CNN）**要解决的三个核心问题。

## 第 13 页 · 感受野的生物学起源

1962 年，两位神经生理学家 David Hubel 和 Torsten Wiesel 做了一个载入史册的实验。他们在哈佛大学把微电极插入麻醉猫的初级视觉皮层（V1 区域），然后给猫看各种视觉刺激——光点、光条、不同方向的边缘——记录单个神经元的活动。

他们的发现有两个层次：

**第一，每个视觉皮层神经元只对视野中一小块区域内的刺激做出反应。** 这个区域被称为该神经元的**感受野（Receptive Field）**。你在这个区域边缘闪一个光点，神经元疯狂放电；你在区域外面闪，它毫无反应。

**第二，视觉皮层存在功能层级：**

- **简单细胞（Simple Cells）**：对特定方向、特定位置的光条或边缘有强烈反应。比如某个细胞只在视野中某个位置出现"竖着的亮条"时才兴奋，横着或者斜着都不行。这直接对应了 CNN 中浅层卷积核学习的 Gabor-like 滤波器——每个卷积核检测一个特定方向的边缘。
- **复杂细胞（Complex Cells）**：也对特定方向的边缘有反应，但对位置的小偏移不敏感（有一定的空间不变性）。这对应了卷积+池化后的特征——对微小平移具有鲁棒性。
- **超复杂细胞（Hypercomplex Cells / End-stopped Cells）**：除了方向选择性，还要求刺激有特定的长度（线条太长或太短都不行）。这对应了更深层中学习到的"物体部件"级别的特征。

这一层级结构——**简单感受野 → 不变性 → 更复杂的模式**——就是卷积神经网络直接借鉴的生物学蓝图。Hubel 和 Wiesel 因为这个发现获得了 1981 年诺贝尔生理学或医学奖。

## 第 14–18 页 · 卷积运算的数值演示

好，现在我们用 PPT 上的数值例子，来逐帧看卷积到底是怎么算的。

### 单步卷积

假设我们有一个 **3×3 的输入**（比如灰度图的局部区域）：

```
 26   120   126
 56    76    99
 46   106   101
```

和一个 **2×2 的卷积核**：

```
 1    0
 0    1
```

这个核的作用是——提取当前位置对角线上的两个值的和。

### 第 1 步：核放在左上角

覆盖的四个值是 `[26, 120; 56, 76]`。

逐元素相乘再求和：
$$26 \times 1 + 120 \times 0 + 56 \times 0 + 76 \times 1 = 26 + 0 + 0 + 76 = 102$$

输出特征图的第一个元素 = **102**。

### 第 2 步：核向右滑动 1 格（stride=1）

覆盖区域右移一格：`[120, 126; 76, 99]`。

$$120 \times 1 + 126 \times 0 + 76 \times 0 + 99 \times 1 = 120 + 0 + 0 + 99 = 219$$

输出变成 `[102, 219]`。

### 第 3 步：核向下滑动 1 格

回到左边，下移一行：`[56, 76; 46, 106]`。

$$56 \times 1 + 76 \times 0 + 46 \times 0 + 106 \times 1 = 56 + 0 + 0 + 106 = 162$$

输出变为第一列 `[102, 162]^T`。

### 第 4 步：核再向右滑动

覆盖 `[76, 99; 106, 101]`。

$$76 \times 1 + 99 \times 0 + 106 \times 0 + 101 \times 1 = 76 + 0 + 0 + 101 = 177$$

最终输出：

```
102   219
162   177
```

一张 3×3 的图，经过 2×2 卷积（stride=1），变成 2×2。这就是卷积的本质——**一个局部的加权求和，在整个输入上反复滑动执行。**

## 第 19 页 · RGB 三通道怎么卷积？

对于彩色图像，输入有 3 个通道（R、G、B）。卷积核也需要有 3 个通道的深度。比如一个 $3 \times 3 \times 3$ 的卷积核。

卷积时：
- 卷积核的三个通道分别和输入的三个通道做二维卷积
- 三个通道的结果按位置相加 → 得到输出特征图上的一个通道

如果你想要输出 64 个通道呢？你就需要 64 个独立的 $3 \times 3 \times 3$ 卷积核。每个卷积核产出 1 个输出通道，64 个卷积核产出 64 个通道。

所以一个卷积层的总参数量 = $C_{out} \times C_{in} \times K_H \times K_W$。

## 第 20–21 页 · 卷积的四个关键超参数

PPT 列出了每个卷积层必须设置的四个参数：

1. **Depth / 输出通道数**：用了多少个不同的卷积核。每个核学习检测一种特征模式。
2. **Stride / 步长**：卷积核每次滑动的像素数。
3. **Padding / 填充**：输入四周补几圈零。
4. **Bias / 偏置**：每个输出通道一个可学习的常数偏移。

### 深度（Depth / Output Channels）

越深的层，输出通道数通常越大。浅层 64 个通道可能就够了，深层可能用到 512 甚至 1024 个。更多的通道意味着模型能同时检测更多的特征模式。

### 步长（Stride）

步长决定了输出的稀疏程度：
- **S=1**：细致扫描，输出几乎和输入一样大（如果用了合适的 padding）
- **S=2**：跳跃扫描，输出尺寸大约减半，起到下采样的作用

PPT 中给了输出尺寸公式（无 padding）：$(W - F)/S + 1$。比如 4×4 输入，F=2（2×2 核），S=2 → 输出 $(4-2)/2+1 = 2$，即 2×2。

### 填充（Padding）

没有 padding 时，图像边缘的像素被卷积覆盖的次数远少于中心像素——这意味着边缘信息的利用率低。Padding 就是**在输入四周补一圈零**，让边缘像素也能多参与几次计算。

PPT 用了一个具体例子：4×4 的图，P=1（四周各补一圈零）变成 6×6，再做 2×2 卷积（S=1），输出恢复为 4×4。这叫 "Same Padding"——输入输出的空间尺寸保持一致。

### 偏置（Bias）

公式：$y = W * x + b$。每个输出通道有一个 $b$，初始通常设为 0。不过，如果卷积后面紧接 Batch Normalization，bias 就设为 False，因为 BN 里的 $\beta$ 参数起到了同样的作用。

## 第 22–23 页 · 感受野堆叠公式

PPT 给了一个重要公式：

$$L_K = 1 + L \times (K-1)$$

$L$ 是层数，$K$ 是每层的卷积核大小。举个例子：堆叠 3 层，每层都是 3×3 卷积（stride=1），感受野 = $1 + 3 \times (3-1) = 7 \times 7$。

这意味着什么？**3 个 3×3 卷积的感受野 = 1 个 7×7 卷积的感受野。** 这个结论直接引出了 VGG 网络的核心设计哲学——用多个小卷积核代替一个大卷积核，保持感受野相同，但参数量更少、非线性更多。

我们来算一下：
- 3 个 3×3（假设输入输出通道都是 C）：参数量 = $3 \times 3^2 \times C^2 = 27C^2$
- 1 个 7×7：参数量 = $7^2 \times C^2 = 49C^2$

节省了约 **45%** 的参数，中间还多了 2 次 ReLU 非线性激活。这就是"深度换表达能力"的数学依据。

## 第 24 页 · 全连接 vs 卷积的参数量对决

这是 PPT 上最有震撼力的一页对比：

**场景**：输入是 224×224×3 的 RGB 图像，输出要得到 1000 维的分类结果。

**全连接层**：$224 \times 224 \times 3 \times 1000 \approx 1.5 \text{ 亿参数}$。仅仅一个隐藏层就要 1.5 亿参数——比 ResNet-152 整体还大。

**卷积层**：用 1000 个 $3 \times 3 \times 3$ 的卷积核。

参数量 = $3 \times 3 \times 3 \times 1000 = 27,000$。

**1.5 亿 vs 2.7 万**。差了约 5500 倍。

这不仅仅是参数的节省。全连接的 1.5 亿参数意味着 1.5 亿次乘法-加法运算；卷积的 2.7 万参数只对应很低的计算开销。这也是为什么深度学习在 2012 年之前无法发展——没有卷积的参数量节省，GPU 也跑不动全连接处理的大图。

## 第 25 页 · 卷积层的完整输入输出规格

PPT 给出了一个规范的总结：

| 项目 | 符号 | 说明 |
|------|------|------|
| 输入 | $C_{in} \times H \times W$ | 通道数 × 高度 × 宽度 |
| 卷积核尺寸 | $K_H \times K_W$ | 常见：1×1, 3×3, 5×5, 7×7 |
| 输出通道数 | $C_{out}$ | 卷积核个数 |
| 填充 | $P$ | 通常 0 或 1 |
| 步长 | $S$ | 通常 1 或 2 |
| 输出 | $C_{out} \times H' \times W'$ | $H' = (H - K + 2P)/S + 1$ |
| 参数量（权重） | $C_{out} \times C_{in} \times K_H \times K_W$ | — |
| 参数量（偏置） | $C_{out}$ | 可选 |

这个表是你设计任何一个卷积层时都应该核对一遍的清单。

## 第 26 页 · PyTorch 中的 nn.Conv2d

PPT 提醒我们，在 PyTorch 里一行代码就能定义卷积：

```python
nn.Conv2d(in_channels, out_channels, kernel_size, stride=1, padding=0)
```

PyTorch 默认采用 Kaiming 初始化（针对 ReLU 优化过的权重初始化策略），输入输出 shape 采用 NCHW 格式（Batch × Channels × Height × Width）。你只需要关心超参数，具体的卷积运算、梯度计算、权重更新全部由框架自动完成。

## 第 27 页 · 1×1 卷积

1×1 卷积初看很奇怪——它只看一个像素，在空间上等于什么都没做。那它有什么用？

看公式：对于输入 $C_{in} \times H \times W$，1×1 卷积核对每个空间位置 $(h,w)$ 上的 $C_{in}$ 维向量做一个线性变换，映射到 $C_{out}$ 维：

$$y_{h,w} = W \cdot x_{h,w} + b$$

这本质上就是**每个空间位置独立执行一次全连接变换**——在通道维度上做信息交互和重组，完全不改变空间分辨率。

1×1 卷积的两大用途：

1. **改变通道数（降维/升维）**：这是它的核心价值。比如把 256 通道压缩到 64 通道，后面的 3×3 卷积就在 64 维的低维空间运行，计算量大幅降低。处理完之后再用另一个 1×1 把通道数恢复到 256。这在 GoogLeNet 的 Inception 模块和 ResNet 的 Bottleneck 中都是核心操作。
2. **增加非线性**：1×1 卷积后面接 ReLU，等于给网络额外增加了一层非线性变换，增强表达力。

举个例子：ResNet-50 的 Bottleneck 块——输入 256 通道 → 1×1 压缩到 64 → 3×3 在 64 通道上运算 → 1×1 扩回 256 → 加上残差连接。如果直接做 256→256 的 3×3 卷积，参数量是 $3^2 \times 256^2 = 589,824$。用了 Bottleneck 后，参数量 = $1^2 \times 256 \times 64 + 3^2 \times 64^2 + 1^2 \times 64 \times 256 = 16,384 + 36,864 + 16,384 = 69,632$。减少了将近 **88%**。

## 第 28 页 · 扩张卷积（Dilated / Atrous Convolution）

扩张卷积是 2016 年的一篇论文《Multi-Scale Context Aggregation by Dilated Convolutions》中提出的。

普通 3×3 卷积的 9 个权重紧密排列在输入的 3×3 格点上。扩张卷积的做法是——**在这些格点之间插入"空洞"**（跳过某些像素）。

插入空洞的数量由 dilation rate（扩张率，d）决定：
- **d=1**：标准 3×3 卷积，有效感受野 $3 \times 3$
- **d=2**：每个采样点之间插入 1 个空洞，有效感受野扩大到 $5 \times 5$
- **d=3**：每个采样点之间插入 2 个空洞，有效感受野 $7 \times 7$

公式：有效核大小 = $K + (K-1) \times (d-1)$。对于 3×3 的核，d=3 时有效大小 = $3 + 2 \times (3-1) = 7$。

扩张卷积最重要的应用场景是**语义分割**。分割任务需要同时满足两个条件：(1) 保持空间分辨率（不能像分类任务那样疯狂下采样，否则丢失了像素级定位精度）；(2) 有足够大的感受野（要理解全局上下文才能做出正确的像素分类）。扩张卷积让你在不降低分辨率的情况下获得大感受野。DeepLab 系列的 ASPP 模块就用多个不同 dilation rate 并行操作来捕获多尺度上下文。

---

## 第 29 页 · 激活函数：ReLU

ReLU（Rectified Linear Unit）的公式极其简单：
$$f(x) = \max(0, x)$$

- $x > 0$ → 原样输出
- $x \le 0$ → 输出 0

### 为什么 ReLU 比 sigmoid / tanh 更好？

回顾 sigmoid：$\sigma(x) = 1/(1+e^{-x})$。它的导数 $\sigma'(x) = \sigma(x)(1-\sigma(x))$。当输入 $x$ 很大或很小时，sigmoid 进入"饱和区"，导数趋近于 0。

在反向传播中，损失对浅层参数的梯度是各层导数连乘。假设网络有 20 层，如果每层的激活函数导数是 0.25（sigmoid 的最大导数），传到第一层的梯度变成 $0.25^{20} \approx 0$。这就是**梯度消失**。

而 ReLU 在正半轴上的导数**恒等于 1**。无论堆多少层，只要激活值大部分是正的，梯度就能完整地回传。这是 AlexNet 能够在 8 层网络上成功训练的关键。

ReLU 的缺点也有——当输入总是 ≤ 0 时，该神经元梯度永远为 0，参数永远不会更新。这叫 "Dying ReLU"。后来改进的 Leaky ReLU（负半轴保留微小斜率 $\alpha=0.01$）和 PReLU（$\alpha$ 可学习）缓解了这个问题。

## 第 30 页 · 感受野的可视化

PPT 用一个直观的图展示了堆叠 3 层 3×3 卷积时感受野的扩大过程：

- 第 1 层输出中的每个值，对应输入的 3×3 区域
- 第 2 层输出中的每个值，对应输入的 5×5 区域
- 第 3 层输出中的每个值，对应输入的 7×7 区域

正是这种逐层扩大的感受野，让深层卷积网络能够逐步从局部边缘特征（低级）聚合成全局语义特征（高级）。

## 第 31–34 页 · 池化层（Pooling）

池化层的核心作用就一个词：**降维**。

### 最大池化（Max Pooling）

在一个小窗口（如 2×2）内取最大值。stride 通常和窗口大小相同（stride=2）。

```
输入 4×4：              Max Pooling 2×2, S=2:
12   20   30   0        20   30
 8   12    2   0   →   112  100
34   70   37   4
112  100  25  12
```

每个 2×2 窗口只保留最大值，其余被扔掉。这等价于"只关心最强特征，不关心它在窗口内的精确位置"。

### 平均池化（Average Pooling）

取窗口内所有值的平均数，比 Max Pooling 更平滑。

### 为什么需要池化？

1. **降低计算量**：空间分辨率减半，后续卷积的计算量变为原来的 1/4。
2. **引入平移不变性**：物体稍微移动了几个像素，只要重要特征还在窗口内，Max Pooling 的输出就不变。对分类来说，知道"有一只猫"比知道"猫在 (137, 228) 位置"更重要。
3. **增大感受野**：分辨率降低后，每个神经元的感受野相对原始图像翻倍。

输出尺寸公式：$H' = (H - K)/S + 1$

### 全局平均池化（Global Average Pooling, GAP）

GAP 是最极端的形式——窗口大小等于整个特征图，把 $H \times W$ 压成一个标量。GoogLeNet 和 ResNet 在网络末端用 GAP 替代全连接层，直接得到每个通道的全局均值，再送进 softmax。GAP 的好处是零参数（天然避免过拟合）、并对每张特征图赋予直接的语义含义（是 CAM 可解释性技术的基础）。

---

## 第 35 页 · 泛化池化

除了 Max 和 Average，PPT 还提到 L2-norm Pooling：
$$y = \sqrt{\sum x_i^2}$$

它对离群值比 Avg Pooling 更敏感，比 Max Pooling 保留了更多的分布信息。

论文《Striving for Simplicity: The All Convolutional Net》(2015) 甚至论证了**池化层本身是可有可无的**——用步长为 2 的卷积替代池化也能实现一样的下采样效果，且不需要额外的池化层设计。

## 第 36 页 · LeNet-5：CNN 的鼻祖

1990 年代，Yann LeCun 在贝尔实验室设计了 LeNet-5，用于手写数字识别（MNIST）。

架构：
```
Input 32×32 灰度图
  → Conv1: 6个 5×5, 输出 6×28×28
  → AvgPool: 2×2, S=2, 输出 6×14×14
  → Conv2: 16个 5×5, 输出 16×10×10
  → AvgPool: 2×2, S=2, 输出 16×5×5
  → Conv3: 120个 5×5, 输出 120×1×1
  → FC: 84
  → Output: 10 (0-9 的数字)
```

LeNet-5 奠定了"卷积 → 激活 → 池化 → 全连接 → 输出"这个经典范式。它的设计思想和今天任何一个现代 CNN 都同宗同源。LeNet-5 当年在美国被银行用于支票上的手写数字识别，是 CNN 的第一次大规模商业落地。

## 第 37–38 页 · ImageNet 与 ILSVRC

### ImageNet 数据集

斯坦福大学李飞飞教授团队于 2009 年发布。超过 1400 万张图像，涵盖约 2.2 万个类别。其核心理念是——**给模型足够多、足够丰富的数据，让它自己学会"看"**。当时的计算机视觉界主流专注于精妙的算法设计，没多少人相信"砸数据"的价值。历史证明李飞飞赢了。

### ILSVRC 竞赛

ILSVRC（ImageNet Large Scale Visual Recognition Challenge）是 ImageNet 的一个子集——120 万张训练图、1000 个类别。2010–2017 年间每年举办一次，是世界顶级视觉模型的竞技场。

历年冠军回顾：

| 年份 | 冠军 | Top-5 错误率 | 核心创新 |
|------|------|:---------:|---------|
| 2010 | 传统方法 | 28.2% | 手工特征 + SVM |
| 2012 | AlexNet | 15.3% | 深度学习革命爆发 |
| 2014 | GoogLeNet | 6.67% | Inception 模块 |
| 2015 | ResNet-152 | 3.57% | 残差连接，首次超越人类 |
| 2017 | SENet | 2.25% | 通道注意力机制 |

7 年间错误率从 28.2% 降到 2.25%——降低了 12.5 倍。这在任何工程领域都是惊人的。2017 年后 ILSVRC 停办——不是没人参加，而是分类任务基本上被"解决"了，研究重心转向了目标检测、实例分割等更难的任务。

## 第 39–46 页 · AlexNet：改变一切的 2012

AlexNet 由 Alex Krizhevsky、Ilya Sutskever 和 Geoffrey Hinton 于 2012 年提出，在 ILSVRC-2012 上以 15.3% Top-5 错误率夺冠（第二名只有 26.2%），拉开了深度学习革命的序幕。

### AlexNet 的架构

```
Input 3×224×224
  → Conv1: 96个 11×11, S=4   → 96×55×55   → MaxPool(3×3,S=2) → 96×27×27
  → Conv2: 256个 5×5, P=2    → 256×27×27  → MaxPool(3×3,S=2) → 256×13×13
  → Conv3: 384个 3×3, P=1    → 384×13×13
  → Conv4: 384个 3×3, P=1    → 384×13×13
  → Conv5: 256个 3×3, P=1    → 256×13×13  → MaxPool(3×3,S=2) → 256×6×6
  → FC6: 4096 (Dropout 0.5)
  → FC7: 4096 (Dropout 0.5)
  → FC8: 1000 (Softmax)
```

总参数量约 6000 万。当年训练用了两块 GTX 580 GPU，花了 5–6 天。

### AlexNet 的五大创新

PPT 总结了五点，我们逐一展开：

**1. 数据增强**

6000 万参数对 120 万张图，很容易过拟合。AlexNet 在训练时对每张图做两种增强：
- 从 256×256 的图随机裁 224×224 的区域，再加随机水平翻转。一张图能变出约 2000 个不同的训练样本。
- **PCA 颜色增强（Fancy PCA）**：对整张图的 RGB 像素做 PCA，在主成分方向上随机扰动——改变整体色调但不改变物体结构。这告诉网络：同一只狗在暖光和冷光下都是狗。

**2. ReLU 激活函数**

AlexNet 是第一个大规模使用 ReLU 的深度网络。Krizhevsky 在论文中报告：在 CIFAR-10 上，ReLU 比 tanh 达到同样训练误差的速度快了约 6 倍。因为 tanh 在深层时的梯度几乎消失，而 ReLU 在正轴的导数为 1。

**3. 局部响应归一化（LRN）**

在同一个空间位置 $(x,y)$ 上，如果某几个通道的激活特别大，就会被相邻通道"抑制"（生物学上的侧抑制机制）。公式大致是：
$$b^i_{x,y} = \frac{a^i_{x,y}}{(k + \alpha \sum_{j} (a^j_{x,y})^2)^\beta}$$

LRN 给 AlexNet 带来了约 1-2% 的提升。但 2015 年 Batch Normalization 出现后，LRN 就基本被淘汰了。

**4. Dropout**

在 FC6 和 FC7 两个全连接层中，训练时随机以 50% 的概率令某些神经元"失活"（输出置零）。每一批数据都在训练一个不同的子网络，测试时使用完整网络。为什么有效？每个神经元不能依赖特定伙伴的存在——大家随时可能被丢掉——迫使其学习更鲁棒、更独立的特征。这等价于训练指数多个子网络的隐式集成。

**5. 多 GPU 训练**

2012 年的 GTX 580 只有 3GB 显存，装不下整个 AlexNet。Krizhevsky 把模型拆分到两个 GPU 上，每个 GPU 只处理一半的卷积核和一半的通道，仅在 Conv3、FC6、FC7 等层做跨 GPU 通信。这是"模型并行"的早期经典实践，启发了今天大模型的分布式训练。

---

## 第 47–52 页 · VGG：极简之美

VGG 由牛津大学 Visual Geometry Group 于 2014 年提出，ILSVRC-2014 分类亚军。

### 核心设计哲学

**全部使用 3×3 的小卷积核，拒绝任何"花哨"的操作。** 一个词概括 VGG：规整。

所有卷积层都是 stride=1、padding=same（输出尺寸不变）；所有池化层都是 2×2、stride=2（尺寸减半）。通道数随着空间减小而翻倍：64 → 128 → 256 → 512 → 512。

### 为什么只用 3×3？

前面我们算过了——3 个 3×3 感受野等价于 1 个 7×7，但参数量少 45%，还多了 2 次 ReLU。

VGG 的版本有 VGG-11、VGG-13、VGG-16、VGG-19，数字代表层数。最常用的是 VGG-16。

### VGG 的优点和缺点

**优点**：结构规则、极易理解和实现、适合作为预训练特征提取器（backbone）。即使到今天，很多人在做快速实验时还是用 VGG-16 的卷积部分来提特征。

**缺点**：全连接层参数量太大。VGG-16 总共有约 1.38 亿参数，其中 FC6 一层就占了 1 亿。推理速度慢，内存消耗大。后续所有网络都在想办法消灭这个大 FC 层。

---

## 第 53–61 页 · GoogLeNet / Inception：多尺度智慧

GoogLeNet 是 ILSVRC-2014 的**冠军**（Top-5 错误率 6.67%），来自 Google。

### 核心问题

图像中的物体有大有小——一张图上可能有一只近处的大狗和远处的小猫。多大的卷积核合适？大核适合大物体，小核适合小物体，但你事先不知道。

Inception 模块的答案是：**不要选，全部都要。** 在同一层并行使用 1×1、3×3、5×5 三种大小的卷积 + 3×3 最大池化，把结果在通道维度拼接起来，让网络自己学习在不同分支之间如何分配权重。

### 1×1 卷积降维是关键

但直接并行操作计算量太大。GoogLeNet 的聪明之处在于：在 3×3 和 5×5 卷积之前，先用一个 1×1 卷积把通道数压下来（比如从 256 压到 64），在低维空间做昂贵的卷积，然后再恢复。这个"先压缩、再处理、再恢复"的模式就是 **Bottleneck 思想**的起源。

### 其他亮点

- **全局平均池化（GAP）**：用 GAP 替代 VGG 的大型全连接层，将 7×7×1024 的特征图压缩成 1024 维向量直接送入 softmax。完全消除了参数膨胀，总参数量仅约 500 万。
- **辅助分类器**：在网络的中间位置（Inception 4a 和 4d 之后）各加一个分类分支，辅助损失占总损失的 30%。目的是在 22 层的深层网络中向中间层注入额外的梯度信号。

### 历史意义

GoogLeNet 用 **500 万参数** 击败了 VGG 的 1.38 亿参数，参数效率差距高达 27 倍。它证明了：**精心设计的结构远胜于粗暴的堆叠。**

---

## 第 62–74 页 · ResNet：残差连接的革命

2015 年，微软亚洲研究院（MSRA）的何恺明团队提出了 ResNet，ILSVRC-2015 冠军，Top-5 错误率仅 **3.57%**，**首次超越了人类的错误率（约 5.1%）**。这篇论文至今被引用超过 20 万次。

### 退化问题

在 ResNet 之前，人们发现：单纯把网络加深（比如从 20 层加到 56 层），不仅测试误差升高，**训练误差也升高了**！这不是过拟合（过拟合应该是训练误差低、测试误差高），而是"退化"——优化算法在深层网络中失效了。

理论上这不合理：一个 56 层网络，哪怕后 36 层什么都不做、只做恒等映射（$H(x)=x$），输出也应该等价于 20 层网络。问题在于：对一堆非线性层的组合来说，学习恒等映射反而很难。

### 残差连接的解决方案

ResNet 的核心公式只有一行：
$$y = F(x, \{W_i\}) + x$$

不是让网络学习目标映射 $H(x)$，而是学习残差 $F(x) = H(x) - x$。如果最优映射就是恒等映射（$H(x) = x$），那么 $F(x)$ 应该趋近于 0——网络只需要把所有参数推向 0 附近，比学习精确的恒等映射容易得多。

### 梯度流动的魔法

反向传播时：
$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \cdot \left(1 + \frac{\partial F}{\partial x}\right)$$

注意括号里的 **"+1"**。就算 $\frac{\partial F}{\partial x}$ 接近零，梯度仍能通过恒等路径回传。这就是 ResNet 能训练 152 层甚至 1000+ 层网络的本质原因。

### Bottleneck 设计

ResNet-50/101/152 使用 Bottleneck：1×1（降维）→ 3×3 → 1×1（升维）+ 残差。前面算过了，这比直接 3×3 卷积省了约 88% 的参数和计算。

### Shortcut 连接的处理

当输入输出维度不匹配时（比如 stride=2 或通道数变了），PPT 给了两种方法：
- **Zero-Padding**：给缺的维度补零
- **Projection Shortcut（更常用）**：用 1×1 卷积 + stride=2 做投影变换：$y = F(x) + W_s x$

### 里程碑意义

- ResNet-152 的 ImageNet Top-5 错误率 **3.57%**，首次低于人类的 5.1%
- 残差连接成为所有后续深度架构的**标配**——CNN、Transformer、图神经网络无一例外
- 它的设计思想（恒等映射 + 残差修正）深刻影响了 NLP、语音、强化学习等领域

---

## 第 75–76 页 · SENet：让网络知道"该听哪个通道的"

SENet（Squeeze-and-Excitation Network）是 ILSVRC-2017 的冠军（Top-5 错误率 2.25%）。

核心是一个即插即用的 **SE Block**，为任何 CNN 添加通道注意力：

**Squeeze（压缩）**：对每个通道的 $H \times W$ 特征图做全局平均池化 → 得到每个通道的一个标量（全局描述）。

**Excitation（激励）**：把这些标量送进两个全连接层（先降维→ReLU→升维→Sigmoid），输出每个通道的 0–1 权重。

**Recalibration（重标定）**：把权重乘回各通道——"重要"的通道被放大，"不重要"的通道被抑制。

什么算"重要"？全是网络从数据中学的。识别"汽车"时，"轮子"通道的权重上升；识别"人"时，"眼睛"通道的权重提升。而且这个权重是输入依赖的——每张不同的图会激活不同的通道权重分布。SE Block 增加的参数量极小（约增加 $2C^2/r$，$r$ 通常为 16），但精度提升显著。

---

## 第 77–78 页 · DenseNet：最极致的连接

DenseNet（CVPR 2017 最佳论文）把连接做到了极致：在一个 Dense Block 内部，**每一层的输入是前面所有层的输出的拼接（Concat），而不是相加（Sum）**。

$$x_l = H_l([x_0, x_1, x_2, ..., x_{l-1}])$$

这意味着第 1 层的特征图可以直接被第 20 层使用，不需要经过任何中间层的"中转"。梯度从损失函数可以直接到达任意一层——这是最强形式的深度监督。

DenseNet 的参数量极低——DenseNet-121 只有 8M 参数，性能却接近 ResNet-50（25M）。因为特征被充分复用了：每层只需要输出很少的新特征（growth rate $k$，通常只有 32），因为前面所有的特征图都已被拼接进来。

代价则是显存消耗大——拼接导致通道数随层数线性增长。

## 第 79–82 页 · MobileNet 系列：为手机而生

### MobileNet V1：深度可分离卷积

标准 3×3 卷积同时做了两件事：空间滤波 + 通道融合。MobileNet V1 把它们拆开：

1. **Depthwise Conv（深度卷积）**：每个通道独立做 3×3 空间卷积（通道之间互不相干）
2. **Pointwise Conv（逐点卷积/1×1）**：在通道维度上做线性融合

计算量从 $K^2 \times C_{in} \times C_{out}$ 降到 $K^2 \times C_{in} + C_{in} \times C_{out}$。对于 3×3 核和较大的 $C_{out}$，计算量约为原来的 1/8 到 1/9。

### MobileNet V2：倒残差 + 线性瓶颈

V1 的深度卷积通道之间互不交流，表达能力受限。V2 做了两个改进：

1. **Linear Bottleneck**：最后一个 1×1 卷积后**不加 ReLU**。直觉：信息在低维空间时，非线性会不可逆地破坏流形结构，保持线性是安全的。
2. **Inverted Residuals**：和 ResNet 的"宽→窄→宽"相反，V2 是"窄→宽→窄"（先 1×1 扩张，中间 3×3 深度卷积在高维空间运行，最后 1×1 压缩回低维）。中间扩张的深度卷积很便宜，却能显著提升表达力。

### MobileNet V3：集大成 + 自动搜索

V3 用 NAS（神经结构搜索）自动设计架构，结合了 V1 的深度可分离卷积 + V2 的倒残差 + SE 通道注意力。还引入了 **h-swish** 激活函数——swish 的硬件友好近似，在移动芯片上运行更快。

## 第 83 页 · EfficientNet：三个维度一起缩放

EfficientNet（Google, 2019）提出了一个重要洞察：**网络的深度、宽度、输入分辨率不是独立的，应该一起缩放。**

更大的图像 → 需要更多的层来逐级处理（扩大感受野到全局）→ 也需要更宽的通道来保持"模式密度"（大图上细节更丰富，需要更多通道来记录）。

复合缩放公式：
$$d = \alpha^\phi, \quad w = \beta^\phi, \quad r = \gamma^\phi$$

$\alpha, \beta, \gamma$ 由网格搜索确定（基于 EfficientNet-B0），$\phi$ 控制整体放大倍率。从 B0 到 B7，EfficientNet-B7 以 66M 参数取得 84.3% Top-1 准确率，比之前的 SOTA 模型小 8 倍多。

## 第 84–85 页 · Vision Transformer (ViT)：Transformer 进入视觉

2020 年，Google 把 NLP 里横扫一切的 Transformer 直接搬到了计算机视觉。核心理念：**把图像视为一系列 patch（图像块）的序列，用自注意力机制建模它们之间的关系。**

ViT 的处理流程：
1. 224×224×3 的图像 → 切成 16×16 的 patch，共 $14 \times 14 = 196$ 个
2. 每个 patch 展平为 768 维向量 → 线性投影到 D 维
3. 加上 position embedding + `[CLS]` token（借鉴 BERT）
4. 送进标准 Transformer Encoder（L 层 Multi-head Self-Attention + MLP）
5. 取 `[CLS]` token 的输出 → 分类头

### CNN vs ViT 的关键差异

CNN 有内置的视觉"先验"——平移等变性（权值共享）和局部性（小卷积核）。这让 CNN 在数据较少时就能很好泛化。

ViT 的自注意力机制没有这些视觉先验——它在一开始就把所有 patch 之间的关系建模为全连接的注意力。这意味着它需要**海量数据**来学习空间关系。所以在 ImageNet-1K（120 万张）上 ViT 不如 ResNet，但在 JFT-300M（3 亿张）上 ViT 全面超越。

## 第 86 页 · 发展脉络总结

PPT 要求我们能梳理出深度视觉模型的演进树：

```
1990: LeNet-5  → CNN 奠基
2012: AlexNet  → 深度学习革命
2014: VGG / GoogLeNet → 加深 vs 加结构
2015: ResNet   → 残差连接，超越人类
2017: DenseNet / SENet → 微结构创新
2017-19: MobileNet / EfficientNet → 轻量化与效率
2020: ViT      → Transformer 进入视觉
```

两条主线并行推进：
- **精度线**：LeNet → AlexNet → VGG → GoogLeNet → ResNet → DenseNet/SENet
- **效率线**：VGG → GoogLeNet → MobileNet → EfficientNet → ViT（大数据场景）

## 第 87–90 页 · CNN 可视化与 CAM

### 我们能看到 CNN 在"看"什么吗？

浅层的卷积核可以直接可视化。你会发现它们学到了类似 Gabor 滤波器的模式——不同方向的边缘检测器、不同颜色的 blob 检测器。这是对 Hubel & Wiesel 简单细胞的人工复现。

但深层的特征图已经是几百个通道的抽象语义，单个通道很难直接看懂。

### CAM（Class Activation Map）：让 CNN 解释自己的决策

CAM 回答的核心问题是：**网络凭什么说这张图是猫？它到底看了图的哪部分？**

原理很优雅：

设最后一层卷积输出 $K$ 个特征图 $A^k$。GAP 把每张特征图变成标量 $f_k$。分类器权重 $w^c_k$ 把特征映射到类别 $c$：
$$S_c = \sum_k w^c_k \cdot \frac{1}{HW} \sum_{i,j} A^k_{i,j} = \frac{1}{HW} \sum_{i,j} \left(\sum_k w^c_k A^k_{i,j}\right)$$

括号里的部分就是 CAM 热力图——每个空间位置 $(i,j)$ 对类别 $c$ 的贡献。

把这个热力图放大叠加到原图上，红色区域就是网络最关注的位置。如果网络正确分类了"猫"，热力图最红的地方应该是猫的脸和身体，而不是背景。

CAM 不仅帮助理解模型，还是一种**弱监督定位**——只给了分类标签（没有位置标注），就能大致定位物体。

## 第 91–98 页 · 迁移学习：站在巨人的肩膀上

### 为什么需要迁移学习？

不是每个人都有 ImageNet 级别的百万标注数据。你手头可能只有几千张医学影像，或者几百张产品缺陷图。从零训练一个大型 CNN，数据远远不够。

### 迁移学习的核心发现

在 ImageNet 上训练好的 CNN，其浅层已经学会了通用的视觉特征——边缘、纹理、形状、部件。这些特征几乎适用于所有自然图像，甚至部分适用于医学影像和遥感图像。Yosinski et al. (NIPS 2014) 的实验证明：前几层特征高度可迁移，越深层的特征越任务特定。

### 三种策略

1. **固定特征提取器**：冻结 backbone 所有权重，只训练最后的分类层。适用场景：目标数据极少，任务与 ImageNet 相似。
2. **微调（Fine-tuning）**：以很小学习率训练 backbone 的上层（甚至全部层）。适用场景：数据量中等，任务有一定差异。
3. **部分冻结**：冻结底层，只微调中高层。平衡了泛化和任务适配。

### 为什么微调能见效？

预训练权重已经位于损失函数景观中的一个"好位置"（广阔的平坦区域）。随机初始化的点可能处于尖锐的局部最优（泛化差），而微调在这个好位置附近做局部搜索，收敛快、泛化好。

## 第 99–104 页 · 实践要点与全章总结

### 一些工程上的碎碎念

1. **选 backbone**：精度优先用 ResNet/EfficientNet，轻量部署用 MobileNet，大数据场景考虑 ViT/Swin Transformer。
2. **数据增强比死磕架构重要**：随机裁剪、翻转、颜色抖动、Mixup、CutMix 对精度的提升往往大于花很多时间调模型结构。
3. **Batch Normalization 不止是归一化**：它自带隐式正则化——每个 mini-batch 的统计噪声让网络不过度依赖精确的激活值，一定程度上防过拟合。
4. **学习率是最重要的超参数**：设对了学习率和衰减策略（Cosine、Step、Exponential），训练效果天差地别。

### 全章核心回顾

同学们，这一章我们走了很长的路。从 RGB 像素的表示，到传统 SIFT 手工特征，到卷积运算的每一个细节，再到 AlexNet → VGG → GoogLeNet → ResNet → DenseNet → SENet → MobileNet → EfficientNet → ViT 这条宏伟的演进脉络。

这背后的主线其实只有三条：
1. **更强的表达力**：从 5 层 LeNet 到 152 层 ResNet，网络越来越深、越来越复杂
2. **更高的效率**：从 1.38 亿参数 VGG 到 500 万参数 GoogLeNet，同等精度下模型越来越小
3. **更广的通用性**：从单一分类到检测、分割、定位，视觉模型从"能看"走向"看懂"

这六个字——**表达力、效率、通用性**——概括了整个深度视觉模型的发展史。
