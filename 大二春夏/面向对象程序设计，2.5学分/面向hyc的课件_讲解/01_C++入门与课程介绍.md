## 1. 什么是面向对象编程？为什么需要它？

### 1.1 从过程式到面向对象：售票机的故事

假设你要写一个模拟售票机的程序。用**过程式编程**的思路，你会这样做：

```
走到售票机前 → 向售票机投钱 → 售票机打印车票 → 拿到车票离开
```

写成代码就是一连串的函数调用和全局变量。程序能工作，但问题是：**代码里没有"售票机"这个东西**。所有的数据（票价、余额、总收入）散落在各处，所有操作（显示提示、收钱、打印车票）是独立的函数。当代码规模变大，数据和操作之间的关系越来越模糊，维护变得极其困难。

面向对象编程换了一种思考方式：**先找到那个"东西"**。

```
  TicketMachine
  ├── 数据: PRICE, balance, total
  └── 操作: showPrompt, getMoney, printTicket, showBalance
```

数据（属性）和操作（方法）被打包在同一个"对象"里。你不再操作散落的全局变量，而是对售票机对象说"收钱""打印车票"。**这就是封装** —— 对象管理自己的数据，外界只能通过它提供的接口来交互。

### 1.2 为什么这样做更好？

| 问题 | 过程式 | 面向对象 |
|------|--------|----------|
| 数据在哪？ | 全局变量，到处可改 | 在对象内部，受保护 |
| 函数和数据的关系？ | 靠命名约定和注释 | 语法级别绑定在一起 |
| 代码复用什么单位？ | 复制粘贴函数 | 复用整个类 |
| 大型项目如何分工？ | 困难，函数互相依赖 | 每个类独立开发测试 |

OOP 的目标不是让小程序写得更快，而是让**大型软件**可以构建、维护和演化。

## 2. 关键术语速览

课件第 10 页列出了一批核心术语。这里逐一解释"是什么"以及"为什么重要"。

| 术语 | 英文 | 含义 | 为什么重要 |
|------|------|------|-----------|
| 继承 | inheritance | 子类获得父类的所有特征，并可添加/修改 | 实现"is-a"关系，代码复用的核心机制 |
| 封装 | encapsulation | 将数据和操作捆绑在一起，隐藏实现细节 | 降低耦合，修改内部实现不影响外部代码 |
| 迭代器 | iterators | 遍历集合中每个元素的对象 | 统一访问不同容器的方式，STL 的基石 |
| 重写 | overriding | 子类重新定义父类的虚函数 | 多态的前提——同一个接口，不同行为 |
| 耦合 | coupling | 模块之间相互依赖的程度 | 耦合越低，修改越安全，测试越容易 |
| 内聚 | cohesion | 模块内部各元素之间的关联强度 | 内聚越高，模块职责越清晰，越容易理解 |
| 模板 | template | 参数化类型，写一次代码适用多种类型 | 泛型编程的关键，STL 的基础 |
| 接口 | interface | 类对外暴露的公共方法集合 | 抽象与实现的分离点 |
| 集合类 | collection classes | 存储和管理一组对象的类 | 比如 vector, list, map |
| 修改器方法 | mutator methods | 改变对象状态的方法 (setter) | 控制对象状态变化的唯一入口 |
| 多态方法调用 | polymorphic method calls | 通过基类指针调用，实际执行的是派生类的方法 | OOP 的三大特性之一，让程序在运行时决定行为 |

**记忆技巧**：耦合和内聚是一对概念。"高内聚、低耦合"是每个优秀设计的追求。可以这样记——耦合是模块间的绳子（越少越好），内聚是模块内部的胶水（越强越好）。

## 3. C++ 简史：一门语言的进化

C++ 不是凭空产生的，它的设计者有非常明确的目标。

### 3.1 时间线

| 年份 | 事件 | 意义 |
|------|------|------|
| 1978 | Bjarne Stroustrup (BS) 在剑桥用 Simula 做仿真 | 体验了类的强大，但性能太差 |
| 1979 | BS 在 AT&T 开发 "C with Classes" (Cpre) | 在 C 上加入类，保留 C 的效率 |
| 1980 | 除虚函数外的主要特性就位 | 已具备 OOP 的雏形 |
| 1983 | 虚函数加入，Rick Mascitti 命名为 **C++** | C 语言中的 `++` 暗示"C 的增强版" |
| 1985 | 《The C++ Programming Language》出版 | 经典之作，定义语言面貌 |
| 1990 | ANSI C++ 委员会成立 | 标准化开始 |
| 1998 | **C++98** (ISO/IEC 14882) | 第一个国际标准，奠定现代 C++ 基础 |
| 2003 | C++03 | 主要是对 C++98 的修正 |
| 2011 | **C++11** | 革命性更新：auto, lambda, 智能指针, move 语义…… |
| 2014 | C++14 | 小幅改进 |
| 2017 | C++17 | 结构化绑定, if constexpr, string_view…… |
| 2020 | C++20 | concepts, ranges, coroutines, modules…… |

### 3.2 为什么了解历史很重要？

- **Simula → C with Classes 这一步**说明 C++ 的 OOP 基因来自 Simula，而 C 提供了效率和灵活性。这就是 C++ 的"双重血统"。
- **1983 年虚函数加入**是决定性的。没有虚函数就没有真正的运行时多态。
- **C++11 是分水岭**：现代 C++ 和传统 C++ 的区别比很多人想象的大。你学到的 `new`/`delete` 原始指针在 C++11 之后有了更好的替代品（智能指针），但理解底层机制仍然必不可少。

### 3.3 Bjarne Stroustrup 与浙大的缘分

BS 在 2002 年和 2005 年两次访问浙江大学，课件中专门提到。这位 C++ 之父曾评价说 C++20 会"非常出色"。他的书《The Design and Evolution of C++》是理解这门语言设计哲学的最佳读物。

## 4. C++ 是什么：混合语言的本质

**C++ 不等于 "C 加上类"。**

课件第 42 页明确指出：C++ 是一门**混合语言 (hybrid language)**，它同时支持三种编程范式：

| 范式 | 核心思想 | C++ 的体现 |
|------|----------|------------|
| 过程式编程 | 函数 + 数据 | 完全兼容 C 的写法 |
| 面向对象编程 | 类 + 对象 + 继承 + 多态 | class, virtual, 继承体系 |
| 泛型编程 | 参数化类型 | template, STL |

这意味着：**你可以在同一个程序中混用这三种风格。** 比如，核心逻辑用 OOP 组织、算法用模板实现、底层 IO 用过程式 C 函数。

### 4.1 C++ 相对 C 的主要改进

C++ 不是 C 的超集（虽然曾经接近），但它确实在 C 的基础上做了大量有意义的增强：

- **数据抽象**: class, 访问控制 (public/private/protected)
- **引用**: 比指针更安全、更直观的别名机制
- **函数重载**: 同一个函数名支持不同参数组合
- **运算符重载**: 让自定义类型也能用 `+ - * / []` 等运算符
- **构造/析构**: 保证对象创建时初始化、销毁时清理
- **内存管理**: `new`/`delete` 替代 `malloc`/`free`，会调用构造和析构
- **流式 I/O**: `cin`/`cout` 替代 `scanf`/`printf`，类型安全
- **模板**: 泛型编程的基础设施
- **异常处理**: `try`/`catch`/`throw` 结构化错误处理
- **const**: (C99 也引入了) 编译期常量
- **内联函数**: (C99 也引入了) 替代宏的安全方案
- **STL**: 标准模板库——容器、算法、迭代器三位一体

## 5. 第一个 C++ 程序：逐行拆解

```cpp
#include <iostream>
using namespace std;
int main()
{
    cout << "Hello, World! I am " << 18 << " Today!" << endl;
    return 0;
}
```

### 5.1 `#include <iostream>`

这是一个**预处理指令**。`#include` 的作用是把 `<iostream>` 文件的内容**直接复制粘贴**到当前文件这个位置。`<>` 尖括号表示在系统标准头文件目录中搜索；`""` 双引号表示先在当前目录搜索。

`<iostream>` 是 C++ 标准输入输出流的头文件，它声明了 `cin`, `cout`, `endl` 等对象。

> **为什么不 `#include <iostream.h>`？** 旧式 C++ 头文件带 `.h`。C++98 标准化后改为不带 `.h`。两者的实现细节不同，新式头文件将内容放在 `std` 命名空间中。

### 5.2 `using namespace std;`

**命名空间 (namespace)** 是 C++ 用来防止命名冲突的机制。标准库的所有内容（`cout`, `cin`, `string`, `vector`...）都放在 `std` 命名空间中。

这行代码的意思是："把 `std` 命名空间中的所有名字引入当前作用域"。你也可以不写这一行，但每次使用标准库时需要加 `std::` 前缀：

```cpp
std::cout << "Hello" << std::endl;
```

**常见争议**：在 `.cpp` 文件中写 `using namespace std;` 通常可以接受，但永远不要在头文件 `.h` 中这样做——因为这会污染所有包含该头文件的编译单元。

### 5.3 `int main()`

这是程序的入口函数。C++ 标准规定 `main` 必须返回 `int`。`return 0;` 表示程序正常结束。

> **常见错误**：写 `void main()` 在一些编译器上可能通过，但不符合 C++ 标准。

### 5.4 `cout << ... << endl;`

`cout` 是标准输出流对象（console output）。`<<` 是**流插入运算符**，它是被重载的左移位运算符，可以将不同类型的数据"推入"输出流。

```cpp
cout << "Hello, World! I am " << 18 << " Today!" << endl;
//     ^^^^^^^^^^^^^^^^^^^^^^^^   ^^   ^^^^^^^^^^   ^^^^
//     字符串字面量              整数  字符串字面量  换行+刷新
```

关键理解：
- `<<` 是**链式调用**的。`cout << a << b` 等价于 `(cout << a) << b`。每次 `<<` 返回 `cout` 本身，所以可以继续调用。
- `endl` 做两件事：输出换行符 `'\n'`，然后**刷新缓冲区** (flush)。如果不需要立即刷新，用 `'\n'` 比 `endl` 更高效。

### 5.5 `cin >> number;`

```cpp
int number;
cin >> number;
```

`cin` 是标准输入流对象，`>>` 是**流提取运算符**。它从输入缓冲区中读取数据并根据右侧变量的类型解析。`cin >> number` 会跳过空白字符，尝试将输入解析为 int。

> **陷阱**：如果用户输入的不是数字（比如 "abc"），`cin` 会进入失败状态，之后的所有输入操作都会被跳过。

## 6. C 与 C++ 对比

### 6.1 C 的优缺点

| C 的优点 | 解释 |
|----------|------|
| 程序高效 | 接近硬件的抽象级别，几乎没有运行时开销 |
| 直接访问机器 | 指针操作内存，适合操作系统和嵌入式开发 |
| 灵活 | 几乎没有强制的约束，程序员有完全的自由 |

| C 的缺点 | 解释 |
|----------|------|
| 类型检查不足 | 隐式类型转换多，`void*` 可以随意赋值 |
| 不适合高层应用 | 没有类、继承、多态，大型项目组织困难 |
| 不直接支持 OOP | 即使用 struct + 函数指针模拟，也极其笨拙 |

### 6.2 关键认知

**C++ 不是 "更好的 C"，而是一门独立的语言。** 虽然它保留了 C 的大部分语法，但 C++ 的思维方式截然不同。用 C++ 写 C 风格的代码，你得到的只是 C 代码的 C++ 编译版本，并没有真正利用这门语言的威力。

课件中 Bjarne Stroustrup 设计 C++ 的目标非常清晰：
> 结合 C 的灵活性和高效性 + 对面向对象编程的支持（继承自 Simula 和 Smalltalk）

## 8. 本章要点总结

1. **OOP 的核心是把数据和操作捆绑在一起**，形成"对象"。过程式编程关注"做什么"，OOP 关注"谁来做"。
2. **封装**捆绑数据和方法，**继承**实现"is-a"关系，**多态**让同一接口有不同的行为。
3. C++ 的进化路径：Simula (类+继承) → C with Classes (效率+类) → C++ (虚函数+多态) → C++98 → C++11 (现代 C++)。
4. C++ 是**混合语言**，同时支持过程式、OOP 和泛型编程。
5. `#include` 是文本复制，`using namespace std` 引入标准库名字，`cout <<` 是链式流输出。
6. C++ 不是 C 的超集，应该作为独立语言来学习。
