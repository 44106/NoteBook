# 04 对象交互

> 封装、构造/析构、重载与 const —— 第四篇自学笔记

---

## 1. 封装与抽象：容易混淆的两个概念

### 1.1 封装 (Encapsulation)：把东西包起来

封装是技术手段——**将数据和处理数据的方法捆绑在同一个对象中，隐藏内部细节，只通过公开方法暴露有限接口。**

```cpp
class NumberDisplay {
private:
    int limit;     // 外部不可见
    int value;     // 外部不可见
public:
    void increase(); // 唯一能修改 value 的方式
    int getValue();  // 唯一能读取 value 的方式
};
```

`NumberDisplay` 的使用者不需要知道 `value` 和 `limit` 如何存储，只需要知道调用 `increase()` 会让数字增加、调用 `getValue()` 能获取当前值。即使你把内部的 `int value` 改成 `unsigned char value`，只要接口不变，使用者的代码一行都不用改。

**封装的本质是隐藏实现细节，降低耦合。**

### 1.2 抽象 (Abstraction)：忽略细节，抓住本质

抽象是思维方式——**忽略不重要的细节，将注意力集中在问题的高层次上。**

当你开车时，你操作方向盘和踏板，不需要关心发动机内部的燃烧过程。方向盘和踏板就是"驾驶"这个概念的抽象。在编程中，`NumberDisplay` 对外暴露的 `increase()` 和 `getValue()` 就是它的抽象——你只需要知道这两个操作，不需要知道内部的 `limit` 和 `value`。

**抽象的体现**：类的外部接口只展示"做什么"（what），隐藏"怎么做"（how）。

### 1.3 封装 vs 抽象对比

| | 封装 | 抽象 |
|------|------|------|
| 关注点 | 如何隐藏（技术手段） | 隐藏什么（设计哲学） |
| 实现方式 | private 成员、访问控制 | 清晰的公共接口 |
| 目标 | 降低耦合 | 管理复杂度 |
| 比喻 | 保险柜（把东西锁起来） | 地图（只标注重要的地标） |

两者相辅相成：封装是实现抽象的技术基础，抽象是封装追求的设计目标。

### 1.4 模块化 (Modularization)

抽象的整体层面就是将系统划分为明确定义的部分，这些部分可以分别构建和检查，并以明确定义的方式进行交互。课件中用时钟显示的例子说明：一个 4 位数字的显示屏，应当设计为"一个 4 位显示"还是"两个 2 位显示"？后者更好——每个 2 位显示是一个可复用的模块（`NumberDisplay`），`ClockDisplay` 则由两个 `NumberDisplay` 组合而成。

## 2. 对象如何交互：消息传递

### 2.1 消息的三要素

在 OOP 中，"调用方法"被理解为"发送消息"：

```
发送者 →（消息）→ 接收者 →（方法实现）→ 可能返回结果 / 可能产生副作用
```

一条消息由三部分组成：
- **发送者 (sender)**：谁发起的调用
- **接收者 (receiver)**：哪个对象接收调用
- **实现 (implementation)**：接收者用什么方法来响应

在 C++ 中：
```cpp
a.move(2, 2);
// 发送者：当前上下文
// 消息：move(2, 2)
// 接收者：对象 a
// 实现：Point::move(int, int)
```

### 2.2 为什么用"消息"这个比喻？

它强调对象之间的**松耦合**——发送者不需要知道接收者内部如何实现，只需要知道接收者理解什么消息。这和网络协议的设计理念一致：客户端发送 HTTP 请求，不关服务器是用 Python 还是 Java 实现的。

## 3. 构造函数：确保对象被正确初始化

### 3.1 为什么需要构造函数？

回到第 3 章的 `Point` 类，最初我们写了 `init()` 方法：

```cpp
Point a;
a.init(1, 2);  // 如果忘了调用这行会怎样？a.x 和 a.y 是垃圾值！
```

问题：**没有任何机制强制程序员调用 `init()`。** 你可以在还没初始化的情况下使用对象，得到未定义的值。构造函数解决了这个问题——**编译器保证它在对象创建时自动被调用，在程序员接触到对象之前。**

```cpp
class Point {
public:
    Point(int x, int y);  // 构造函数——名称为类名，无返回类型
private:
    int x, y;
};

Point a(1, 2);  // 构造时自动调用，不可能忘记
```

### 3.2 构造函数的工作原理

```cpp
class X {
    int i;
public:
    X();  // 构造函数声明
};

void f() {
    X a;     // 对象创建时，编译器自动插入 a.X() 的调用
}
```

当编译器看到 `X a;` 时，它在对象的内存分配之后、`a` 可用之前，自动调用 `X::X()`。程序员无法干预这个调用时机，这就是"保证初始化"的含义。

### 3.3 带参数的构造函数

```cpp
class Tree {
    int height;
public:
    Tree(int h) : height(h) {}  // 带参数的构造函数
};

Tree t(12);        // 传递构造参数
Tree t2 = 12;      // 等价的隐式转换写法（可能引发歧义）
```

> **提示**：用 `explicit` 关键字可以禁止隐式转换：`explicit Tree(int h);`

## 4. 析构函数：确保对象被正确清理

### 4.1 为什么需要析构函数？

初始化重要，清理同样重要。如果构造函数打开了文件、分配了堆内存、获取了锁，那么在对象销毁时必须释放这些资源。析构函数就是 C++ 保证清理发生的手段。

```cpp
class Buffer {
    char* data;
public:
    Buffer(int size) { data = new char[size]; }  // 构造：分配内存
    ~Buffer() { delete[] data; }                 // 析构：释放内存
};
```

**析构函数的特征**：
- 名称是 `~类名`，如 `~Buffer()`
- 永远不带参数（也不能重载——只能有一个析构函数）
- 无返回类型

### 4.2 析构函数何时被调用？

**当对象离开作用域时，析构函数会被编译器自动调用。** 这和构造函数的调用时机对称：

```cpp
void f() {
    Buffer buf(100);  // 构造
    // ...
}  // buf 离开作用域 ← 析构函数在这里自动调用
```

对于堆上的对象：
```cpp
Buffer* buf = new Buffer(100);  // 构造
delete buf;                     // 析构 + 释放内存（必须手动调用）
```

> **常见错误**：用 `new` 创建的对象如果忘了 `delete`，析构函数永远不会被调用——造成内存泄漏。

## 5. 存储分配与构造时机

### 5.1 编译器何时分配空间？

```cpp
void f() {
    // ← 编译器在作用域入口（左花括号处）分配该作用域内所有变量的空间
    X a;    // ← 但构造函数在这个定义点才被调用
    // 如果有 goto 跳过 a 的定义……
}
```

这意味着：**空间分配和构造调用是分离的**。空间在进入作用域时就分配好了（栈指针一次性移动），构造函数的调用发生在变量定义的那个精确点。如果你用 `goto` 跳过了变量的定义点，空间已经在栈上预留了，但构造不会发生——这可能引发问题。

### 5.2 聚合初始化

对于没有用户定义构造函数、没有私有/受保护成员、没有基类的简单结构，可以使用 C 风格的聚合初始化：

```cpp
int a[5] = {1, 2, 3, 4, 5};         // 数组
int b[6] = {5};                      // 第一个元素是 5，其余是 0
int c[] = {1, 2, 3, 4};             // 编译器推断大小为 4

struct X { int i; float f; char c; };
X x1 = {1, 2.2, 'c'};               // 按声明顺序初始化
X x2[3] = { {1, 1.1, 'a'}, {2, 2.2, 'b'} };  // 最后一个用默认值

struct Y { float f; int i; Y(int a); };  // 有构造函数 → 不能用 {} 初始化
Y y1[] = { Y(1), Y(2), Y(3) };          // 必须显式调用构造函数
```

**关键限制**：一旦类有了构造函数（即使是编译器自动生成的），聚合初始化就用不了了——必须用构造函数。

## 6. 默认构造函数：细节决定成败

### 6.1 什么是默认构造函数？

默认构造函数是**可以不带参数调用的构造函数**。它有两种形式：
- 没有任何参数的构造函数：`X() { }`
- 所有参数都有默认值的构造函数：`X(int a = 0) { }`

### 6.2 编译器何时自动生成？

**当且仅当**类没有任何用户定义的构造函数时，编译器才会自动创建默认构造函数。

```cpp
struct Y {
    float f;
    int i;
    Y(int a);     // 用户定义了构造函数
};
// 编译器不会为 Y 生成默认构造函数！

Y y3[7];   // 错误！需要默认构造函数但不存在
Y y4;      // 错误！同上
```

**为什么这是重要的？** 如果你写了一个带参数的构造函数但忘了写默认构造函数，那么不能创建数组、不能在不传参数的情况下创建对象。这在你想要强制初始化为某个值时是有意为之，但如果疏忽了就会成为 bug。

### 6.3 编译器自动生成的默认构造函数做了什么？

对于内置类型成员（如 int, float），它**不进行初始化**——这和很多人的直觉不同。如果你写 `X a;` 且 X 没有构造函数，`a.i` 的值是未定义的（垃圾值）。这也是为什么最好总是自己写构造函数或至少使用初始化列表的原因。

## 7. 初始化列表 vs 赋值：效率之争

这是很多新手困惑的地方。

```cpp
// 方式一：初始化列表
Student::Student(string s) : name(s) {}

// 方式二：构造函数体内赋值
Student::Student(string s) {
    name = s;
}
```

### 7.1 执行过程的差异

**初始化列表**（方式一）：`name` 直接由 `s` 初始化 —— 只调用了一次 string 的拷贝构造函数。

**赋值**（方式二）：
1. 进入构造函数体之前，`name` 先被默认构造（调用 `string()`）
2. 构造函数体内执行 `name = s`（调用 string 的赋值运算符）
3. 总共调用了两个函数。

对于 `string` 这种类类型，方式一明显更高效。对于 `int`，虽然效率相同，但有些场景必须用初始化列表。

### 7.2 必须用初始化列表的情况

```cpp
class Point {
private:
    const float x, y;  // const 成员——不能赋值，只能初始化
public:
    Point(float xa, float ya) : x(xa), y(ya) {}  // 必须用初始化列表
};
```

还有三种情况必须用初始化列表：
- **const 成员变量**
- **引用成员变量**
- **没有默认构造函数的成员对象**（你必须给它传参数）

> **初始化顺序是成员声明的顺序，不是初始化列表中的顺序！** 如果初始化列表中的顺序和声明顺序不一致，可能导致难以发现的 bug。最简单的防御方式：让初始化列表的顺序和声明顺序保持一致。

## 8. 函数重载：同名不同参

```cpp
void print(char * str, int width);  // #1
void print(double d, int width);    // #2
void print(long l, int width);      // #3
void print(int i, int width);       // #4
void print(char *str);              // #5

print("Pancakes", 15);   // 调用 #1：char* + int
print("Syrup");           // 调用 #5：只有 char*
print(1999.0, 10);        // 调用 #2：double + int
print(1999, 12);          // 调用 #4：int + int
print(1999L, 15);         // 调用 #3：long + int
```

### 8.1 重载解析的复杂性

当参数类型不完全匹配时，编译器会尝试隐式类型转换，可能产生歧义：

```cpp
void f(int i);
void f(double d);

f('a');    // 二义性！char 可以转换为 int 也可以转换为 double
f(2);      // 明确：int
f(2L);     // 二义性！long 可以转换为 int 也可以转换为 double
f(3.2f);   // 明确：float → double
```

> **经验**：如果重载的函数之间参数类型可以互相转换，就可能产生歧义。设计重载时尽量让参数类型"远离"——比如 `int` 和 `string` 的组合就不会有歧义。

## 9. 默认参数：省略即默认

### 9.1 基本语法和右到左规则

```cpp
Stash(int size, int initQuantity = 0);

Stash s(100);        // initQuantity 默认为 0
Stash s(100, 50);    // 显式指定 initQuantity
```

**必须从右向左设置默认值**：

```cpp
int harpo(int n, int m = 4, int j = 5);     // OK
int chico(int n, int m = 6, int j);          // 非法！m 有默认值但 j 没有
int groucho(int k = 1, int m = 2, int n = 3); // OK，全部有默认值
```

**为什么是右到左？** 因为函数调用时参数从左到右匹配。如果允许 `f(int a = 1, int b)`，那么 `f(5)` 这个调用就无法判断 5 是给 `a` 的还是给 `b` 的。

### 9.2 默认参数与函数重载的陷阱

```cpp
void print(int width = 10);
void print(int width);  // 错误！声明了两次同一个函数
```

默认参数本质上只是让调用时可以省略参数，并不改变函数的签名。所以不能把它和重载混淆。

> **经验**：默认参数只在声明处出现一次（通常放在 .h 文件中），不要在定义处重复默认值。

## 10. const 成员函数：承诺不修改

### 10.1 为什么需要它？

```cpp
const Currency the_raise(42, 38);
the_raise.getValue();  // 能调用吗？
```

如果 `getValue()` 没有声明为 `const`，编译器不知道它会不会修改对象——为了安全，编译器会拒绝这个调用。也就是说，**const 对象只能调用 const 成员函数。**

### 10.2 在 const 成员函数中能做什么、不能做什么

```cpp
int Date::get_day() const {
    day++;           // 错误！不能修改数据成员
    set_day(12);     // 错误！不能调用非 const 成员函数
    return day;      // OK，只读取不修改
}
```

**const 成员函数内部的约束**：
- `this` 的类型从 `Date*` 变为 `const Date*`
- 不能修改任何非 `mutable` 的数据成员
- 只能调用其他 `const` 成员函数
- 可以读取任何成员

### 10.3 使用建议

```cpp
class Date {
public:
    int get_day() const;        // 声明和定义都要加 const
    void set_day(int d);         // 修改函数不加 const
};

// 定义
int Date::get_day() const {     // const 必须重复
    return day;
}

// 使用
Date when(1, 1, 2001);
int day = when.get_day();         // OK：非 const 对象可以调用 const 函数
when.set_day(13);                 // OK：非 const 对象可以调用非 const 函数

const Date birthday(12, 25, 1994);
int day = birthday.get_day();    // OK：const 对象调用 const 函数
birthday.set_day(14);            // 错误！const 对象不能调用非 const 函数
```

> **黄金法则**：不修改对象状态的成员函数一律声明为 `const`。

## 11. 类中的编译时常量

如果需要一个在编译期就确定大小的类内数组，直接写 `const int size;` 不行——它是每个对象一个的成员，值到构造时才知道。

### 11.1 方案一：static const

```cpp
class HasArray {
    static const int size = 100;  // 整个类共享一份，编译期确定
    int array[size];              // OK！
};
```

`static const` 整型常量可以在类内初始化。它是类的属性，不属于任何具体对象。

### 11.2 方案二：匿名 enum 技巧

```cpp
class HasArray {
    enum { size = 100 };  // 匿名的 enum，size 是编译期常量
    int array[size];      // OK！
};
```

这个方法利用的是 `enum` 的值在编译期就已经确定，可以用作数组大小。它在老式编译器上比 `static const` 更可靠。

## 12. 内联函数：空间换时间

### 12.1 函数调用的开销

```
调用一个函数需要：
1. 压入参数
2. 压入返回地址
3. 跳转到函数代码
4. 执行函数
5. 准备返回值
6. 弹出所有已压入的内容
7. 跳回调用点
```

对于只有一两行的"小函数"，这个开销可能比函数体本身还大。

### 12.2 inline 的原理

```cpp
inline int f(int i) {
    return i * 2;
}

int main() {
    int a = 4;
    int b = f(a);
}

// 编译器展开后相当于：
int main() {
    int a = 4;
    int b = a * 2;  // f(a) 被直接替换
}
```

**inline 是向编译器发出的请求**（不是命令），将函数调用替换为函数体的直接展开。

### 12.3 inline vs 宏：安全性的胜利

```cpp
// 宏（不安全）
#define unsafe(i) ((i) >= 0 ? (i) : -(i))
ans = unsafe(x++);      // 危险！x++ 被计算了两次
ans = unsafe(f());      // 危险！f() 被调用了两次

// 内联函数（安全）
inline int safe(int i) {
    return i >= 0 ? i : -i;
}
ans = safe(x++);         // x++ 只计算一次，结果传入参数 i
ans = safe(f());         // f() 只调用一次
```

### 12.4 内联的使用策略

| 适合内联 | 不适合内联 |
|----------|------------|
| 小型函数（2-3 行） | 大型函数（超过 20 行） |
| 频繁调用的函数（循环内） | 递归函数 |
| getter/setter 访问函数 | 包含复杂逻辑的函数 |

```cpp
class Cup {
    int color;
public:
    int getColor() { return color; }       // 在类内定义的函数自动 inline
    void setColor(int color) {
        this->color = color;
    }
};
```

### 12.5 关于 inline 的现代认知

课件最后一页指出：现代编译器中 `inline` 关键字的真正意义更多在于**"允许多重定义"**而非"倾向于内联"。因为链接器只允许函数被定义一次，但 inline 函数通常在头文件中定义（被多个 .cpp 包含后产生多个"定义"）——`inline` 告诉链接器这些重复定义是合法的，链接器会保留一份。

## 13. 访问函数 (getters / setters)

```cpp
class Cup {
    int color;
public:
    int getColor() const { return color; }         // getter
    void setColor(int color) { this->color = color; } // setter
};
```

访问函数是对私有数据成员的受控访问通道。即使 getter 只是简单返回一个值、setter 只是简单赋值，它们也有重要价值：

- 你可以在这里加**合法性检查**（如 setColor 检查颜色范围）
- 如果内部实现变了（如 color 从 int 改为 enum），只需修改 getter/setter，外部代码不变
- 它们是**封装的实践**——数据私有，接口公有

---

## 14. 本章要点总结

1. **封装**是技术手段（隐藏数据），**抽象**是设计哲学（忽略细节抓本质）。封装为抽象服务。
2. **构造函数**在对象创建时自动调用，**保证初始化**。析构函数在对象离开作用域时自动调用，**保证清理**。
3. **初始化列表**比构造函数体内的赋值更高效（对于类类型成员），且是 const/引用成员唯一的选择。
4. **默认构造函数**只在类没有定义任何构造函数时由编译器自动生成。
5. **函数重载**通过参数列表区分同名函数；**自动类型转换**可能引发歧义。
6. **默认参数**必须从右到左设置，且只在声明处出现一次。
7. **const 成员函数**承诺不修改对象状态，const 对象只能调用 const 成员函数。
8. **内联函数**消除调用开销，比宏安全，适合小函数和 getter/setter。
9. **访问函数**提供对私有数据的受控访问，是封装的实践。
