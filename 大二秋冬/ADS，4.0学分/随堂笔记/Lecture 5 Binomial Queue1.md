# Lecture 5 Binomial Queue

**二项队列**是一种支持高效合并的优先队列实现。其核心特性与结构如下：

1. **结构定义**：
   - 它不是一个单一堆，而是一个**二项树的集合**。
   - 每个**二项树 Bₖ** 都是一个递归定义的最小堆：B₀ 是单节点；Bₖ 由两棵 Bₖ₋₁ 连接而成。
   - 在一个二项队列中，**每个度数的二项树最多存在一棵**。这使得队列的结构与整数的二进制表示一一对应。
2. **核心操作与性能**：
   - **合并**：是其核心优势。通过模拟二进制加法原理，将相同度数的树两两合并，时间复杂度为 **O(log n)**。
   - **插入**：被视为合并一个只含单节点树的队列，摊还代价可低至 **O(1)**。
   - **删除最小元**：首先找到所有树根中的最小者，移除后将其子树构成的新队列与原队列合并，时间复杂度为 **O(log n)**。

**总结**：二项队列通过维护一个由规整的二项树构成的“森林”，以 **O(log n)** 的时间复杂度实现了高效的合并操作，从而使其在需要频繁合并优先队列的场景下优于二叉堆。

二项树B~k~的节点个数为2^k^ ,深度为d的节点有$\binom{n}{k}$个，根节点的儿子有k个。

### 合并

**核心思想：模仿二进制加法。**

1. **输入**：两个二项队列（可视为两个“二进制数”，每一位对应一棵二项树）。

2. **过程**：从度数最低的树开始，像二进制加法一样逐位处理：

   - **0棵树**：该度数结果为空。
   - **1棵树**：该度数结果就是这棵树。
   - **2棵树**：将它们合并成一棵更高一度的树，作为“进位”。
   - **3棵树**：保留一棵作为结果，另外两棵合并成“进位”。

3. **输出**：一个新的二项队列，其中每个度数最多有一棵树。

4. **时间复杂度**：**O(log n)**

   ### 删除最小元

   1. **定位**：扫描所有二项树的**根节点**，找到值最小的那个根。
   2. **拆除**：删除这个最小根节点，这会将其所在的二项树 `Bₖ` 分解成 k 棵子树：`Bₖ₋₁, Bₖ₋₂, ..., B₀`。这些子树自然地形成了一个新的二项队列。
   3. **合并**：将**原队列（已移除最小树）** 与**新生成的那个队列**进行合并。
   4. **时间复杂度**：**O(log n)** (因为定位和合并都是 O(log n))
